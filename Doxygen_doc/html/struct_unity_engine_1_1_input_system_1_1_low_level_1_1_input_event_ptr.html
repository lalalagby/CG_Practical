Detailed Description

Pointer to an InputEvent. Makes it easier to work with InputEvents and hides the unsafe operations necessary to work with them.

Note that event pointers generally refer to event buffers that are continually reused. This means that event pointers should not be held on to. Instead, to hold onto event data, manually copy an event to a buffer.

◆ handled

bool UnityEngine.InputSystem.LowLevel.InputEventPtr.handled
getset

Whether the event is considered "handled" and should not be processed further.

This is used in two ways. Setting it from inside InputSystem.onEvent will cause the event to not be processed further. If it is a StateEvent or DeltaStateEvent, the InputDevice targeted by the event will not receive the state change.

Setting this flag from inside a state change monitor (see InputState.AddChangeMonitor(InputControl,IInputStateChangeMonitor,long,uint)) will prevent other monitors on the same IInputStateChangeMonitor not receiving the state change.
