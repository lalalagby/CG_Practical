enum UnityEngine.InputSystem.Users.InputUserChange

Indicates what type of change related to an InputUser occurred.

See also
InputUser.onChange
Enumerator
Added 

A new user was added to the system.

See also
InputUser.PerformPairingWithDevice
Removed 

An existing user was removed from the user.

InputUsers are only removed when explicitly requested (InputUser.UnpairDevicesAndRemoveUser).

See also
InputUser.UnpairDevicesAndRemoveUser
DevicePaired 

A user has had a device assigned to it.

See also
InputUser.PerformPairingWithDevice, InputUser.pairedDevices
DeviceUnpaired 

A user has had a device removed from it.

This is different from DeviceLost in that the removal is intentional. DeviceLost instead indicates that the device was removed due to causes outside of the control of the application (such as battery loss) whereas DeviceUnpaired indicates the device was removed from the user under the control of the application itself.

See also
InputUser.UnpairDevice, InputUser.UnpairDevicesAndRemoveUser, InputUser.pairedDevices
DeviceLost 

A device was removed while paired to the user.

This scenario happens on battery loss, for example.

Note that there is still a DevicePaired change sent when the device is subsequently removed from the user.

See also
InputUser.pairedDevices, InputUser.lostDevices
DeviceRegained 

A device that was previously lost (DeviceLost) was regained by the user.

REVIEW: should we tie this to the specific device requirement slot in the control scheme?

This can happen, for example, when a gamepad runs out of battery and is then plugged in by wire.

Note that a device may be permanently lost and instead be replaced by a different device.

See also
InputUser.lostDevices
ControlSchemeChanged 

A user switched to a different control scheme.

See also
InputUser.ActivateControlScheme(string), InputUser.ActivateControlScheme(InputControlScheme)
ControlsChanged 

A user's bound controls have changed, either because the bindings of the user have changed (for example, due to an override applied with InputActionRebindingExtensions.ApplyBindingOverride(InputAction,InputBinding)) or because the controls themselves may have changed configuration (every time the device of the controls receives an DeviceConfigurationEvent; for example, when the current keyboard layout on a Keyboard changes which in turn modifies the InputControl.displayNames of the keys on the keyboard).

◆ InputUserPairingOptions

enum UnityEngine.InputSystem.Users.InputUserPairingOptions

Options to modify the behavior on InputUser.PerformPairingWithDevice.

Enumerator
None 

Default behavior.

ForcePlatformUserAccountSelection 

Even if the device is already paired to a user account at the platform level, force the user to select an account.

This is only supported on Xbox and Switch, at the moment.

On PS4, this is ignored as account pairing is under system control. If the user wants to switch accounts, he/she does so by pressing the PS4 button on the controller.

On Xbox, this option will bring up the account picker even if the device is already paired to a user. This behavior is useful to allow the player to change accounts.

On platforms other than Xbox and Switch, this option is ignored.

ForceNoPlatformUserAccountSelection 

Suppress user account selection when supported at the platform level and a device is not currently paired to a user account.

On Xbox, if a device that does not currently have a user account logged in on it is paired to an InputUser, no account picker will come up and the device will be used without an associated user account.

On Switch, this prevents the user management applet from coming up.

UnpairCurrentDevicesFromUser 

If the user already has paired devices, unpair them first.
