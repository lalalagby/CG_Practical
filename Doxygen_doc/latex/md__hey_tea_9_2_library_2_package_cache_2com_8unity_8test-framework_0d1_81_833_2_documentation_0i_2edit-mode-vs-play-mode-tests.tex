 Let’s clarify a bit what Play Mode and Edit Mode test means from the \doxylink{namespace_unity}{Unity} Test Framework perspective\+:\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2edit-mode-vs-play-mode-tests_autotoc_md4197}{}\doxysection{\texorpdfstring{Edit Mode tests}{Edit Mode tests}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2edit-mode-vs-play-mode-tests_autotoc_md4197}
{\bfseries{Edit Mode}} tests (also known as Editor tests) are only run in the \doxylink{namespace_unity}{Unity} Editor and have access to the Editor code in addition to the game code.

With Edit Mode tests it is possible to test any of your \href{https://docs.unity3d.com/Manual/ExtendingTheEditor.html}{\texttt{ Editor extensions}} using the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_062284b282c4f87cfc2410902afc579cf}{Unity\+Test}{0} attribute. For Edit Mode tests, your test code runs in the \href{https://docs.unity3d.com/ScriptReference/EditorApplication-update.html}{\texttt{ Editor\+Application.\+update}} callback loop.

\begin{quote}
{\bfseries{Note}}\+: You can also control entering and exiting Play Mode from your Edit Mode test. This allow your test to make changes before entering Play Mode. \end{quote}


Edit Mode tests should meet one of the following conditions\+:


\begin{DoxyItemize}
\item They should have an \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_04872722a20000e06b1b28d5347f2329d}{assembly definition}{0} with reference to {\itshape nunit.\+framework.\+dll} and has only the Editor as a target platform\+:
\end{DoxyItemize}

\item Legacy condition\+: put tests in the project’s \href{https://docs.unity3d.com/Manual/SpecialFolders.html}{\texttt{ Editor}} folder.
\end{DoxyItemize}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2edit-mode-vs-play-mode-tests_autotoc_md4198}{}\doxysection{\texorpdfstring{Play Mode tests}{Play Mode tests}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2edit-mode-vs-play-mode-tests_autotoc_md4198}
You can run {\bfseries{Play Mode}} tests as a \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_067b7200309be97aaf3a215eb01c30ccc}{standalone in a Player}{0} or inside the Editor. Play Mode tests allow you to exercise your game code, as the tests run as \href{https://docs.unity3d.com/ScriptReference/Coroutine.html}{\texttt{ coroutines}} if marked with the {\ttfamily Unity\+Test} attribute.

Play Mode tests should correspond to the following conditions\+:


\begin{DoxyItemize}
\item Have an \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_04872722a20000e06b1b28d5347f2329d}{assembly definition}{0} with reference to {\itshape nunit.\+framework.\+dll}.
\item Have the test scripts located in a folder with the .asmdef file.
\item The test assembly should reference an assembly within the code that you need to test.
\end{DoxyItemize}

Use the \href{http://www.nunit.org/}{\texttt{ NUnit}} {\ttfamily Test} attribute instead of the {\ttfamily Unity\+Test} attribute, unless you need to \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0a7960e96e42640710807a331b7891678}{yield special instructions}{0}, in Edit Mode, or if you need to skip a frame or wait for a certain amount of time in Play Mode.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2edit-mode-vs-play-mode-tests_autotoc_md4201}{}\doxysubsection{\texorpdfstring{References}{References}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2edit-mode-vs-play-mode-tests_autotoc_md4201}
It is possible for your Test Assemblies to reference the test tools in {\ttfamily \doxylink{namespace_unity_engine_1_1_test_runner}{Unity\+Engine.\+Test\+Runner}} and {\ttfamily \doxylink{namespace_unity_editor_1_1_test_runner}{Unity\+Editor.\+Test\+Runner}}. The latter is only available in Edit Mode. You can specify these references in the {\ttfamily Assembly Definition References} on the Assembly Definition.
