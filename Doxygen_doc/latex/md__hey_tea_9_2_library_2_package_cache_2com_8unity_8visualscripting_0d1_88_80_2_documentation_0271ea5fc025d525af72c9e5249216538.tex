The On Mouse Drag node listens for a mouse click and hold on a specific Game\+Object in your application. It triggers the next node connected to it as long as the mouse button is held down on that Game\+Object. It doesn\textquotesingle{}t send or receive any other data.

In the following example, the On Mouse Drag node triggers a Camera Screen To World Point node. When the user clicks and holds their mouse button over the {\bfseries{Target}} Game\+Object from the On Mouse Drag node, the Script Graph gets the user\textquotesingle{}s current mouse position with an Input Get Mouse Position node. The graph uses the X and Y values from the Get Mouse Position node\textquotesingle{}s Vector 3 result to create a new Vector 3 value, with a fixed Z value. The Screen To World Point node uses the new Vector 3 and the camera saved in the {\bfseries{Main Camera}} Scene variable to set the position of the {\bfseries{Target}} Game\+Object\textquotesingle{}s transform.



The Script Graph allows the user to drag the {\bfseries{Target}} Game\+Object around the scene when hold down their mouse button.
