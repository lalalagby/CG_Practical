 When you use a Script Graph as a Subgraph, you can pass data and logic to it from its parent graph. Add and define ports on your graph to determine which logic and data you can pass to it.



To add a Trigger Input, Trigger Output, Data Input, or Data Output port to a Script Graph\+:


\begin{DoxyEnumerate}
\item If you haven\textquotesingle{}t already, \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-open-graph-edit}{open the Script Graph you want to edit in the Graph window}{0}.
\item With no nodes selected, in the Graph Inspector, select {\bfseries{Add}} (+) under the port type you want to add to your Script Graph\+: {\bfseries{Trigger Inputs}}, {\bfseries{Trigger Outputs}}, {\bfseries{Data Inputs}}, or {\bfseries{Data Outputs}}.

\mbox{[}!include\doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_05e829de251815f572fc4f0ee0cb9ec9b}{graph-\/inspector-\/tip}{0}\mbox{]}
\item In the {\bfseries{Key}} field, enter a unique key name for your port. This name can\textquotesingle{}t be the same as any existing ports on your currently selected Script Graph.
\item (Optional) In the {\bfseries{Label}} field, enter any text you want to appear as a label for the port on a Subgraph, Input, or Output node for your current Script Graph. Otherwise, Visual Scripting uses the value in the {\bfseries{Key}} field as a label.
\item (Optional) In the {\bfseries{Summary}} field, enter any text that you want to appear as a brief summary of the port in the Graph Inspector when you select a Subgraph, Input, or Output node for your current Script Graph.
\item (Optional) Select {\bfseries{Hide Label}} to hide the label for the port on any Subgraph, Input, or Output node for your current Script Graph.
\item (Data Inputs and Data Outputs Only) Select the {\bfseries{Type}} list to open the Type menu and select a type for the data your port should accept.
\item (Data Inputs Only) Select {\bfseries{Has Default Value}} to enable the {\bfseries{Default Value}} field and specify a default value for your Script Graph\textquotesingle{}s Data Input, if your graph doesn\textquotesingle{}t receive another input while it runs.
\end{DoxyEnumerate}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-add-triggers-data-graph_autotoc_md5159}{}\doxysection{\texorpdfstring{Next steps}{Next steps}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-add-triggers-data-graph_autotoc_md5159}
You can now specify triggers and data for your Script Graph when you use it as a Subgraph in another Script Graph. For more information on each port type, see \href{vs-nesting-subgraphs-state-units.md\#subgraph-inputs-and-outputs}{\texttt{ Subgraph inputs and outputs}}.
