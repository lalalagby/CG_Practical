\href{https://docs.unity3d.com/Manual/SpeedTree.html}{\texttt{ Speed\+Tree}} is a third-\/party solution that includes both ready-\/to-\/use tree assets, and modeling software for creating your own tree assets.

Shader Graph has three built-\/in Speed\+Tree Sub Graph Assets\+:


\begin{DoxyItemize}
\item Speed\+Tree8\+Color\+Alpha
\item Speed\+Tree8\+Wind
\item Speed\+Tree8\+Billboard
\end{DoxyItemize}

These Sub Graph Assets provide Speed\+Tree 8 functionality for both the \href{https://docs.unity3d.com/Packages/com.unity.render-pipelines.universal@12.0/manual/index.html}{\texttt{ Universal Render Pipeline (URP)}} and \href{https://docs.unity3d.com/Packages/com.unity.render-pipelines.high-definition@12.0/manual/index.html}{\texttt{ High Definition Render Pipeline (HDRP)}}, so that you can work with Speed\+Tree 8 assets and create your own custom Speed\+Tree 8 Shader Graphs.

{\bfseries{Note\+:}} The URP-\/specific versions of these Speed\+Tree 8 Sub Graph Assets use transparent billboard back faces instead of culling billboard back faces. These Sub Graph Assets can only replace their URP equivalents as a default once URP supports per-\/material culling overrides in Shader Graphs.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_speed_tree8-_sub_graph_assets_autotoc_md3938}{}\doxysection{\texorpdfstring{Speed\+Tree8\+Color\+Alpha \label{_SpeedTree8ColorAlpha}%

Each Speed\+Tree asset has four maps\+: a basemap (color/albedo), bump map (which provides surface normals), extra map (which provides metallic and ambient occlusion data), and subsurface map (which provides the subsurface scattering color). The basemap provides input color and alpha data.

This Sub Graph Asset applies all Speed\+Tree 8 features that modify the basemap\textquotesingle{}s color and alpha data. These features are particularly useful for the following actions\+:
\begin{DoxyItemize}
\item Tinting the basemap color
\item Varying tree hues
\item Crossfading between levels of detail (LODs)
\item Hiding geometry seams
\end{DoxyItemize}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_speed_tree8-_sub_graph_assets_autotoc_md3939}{}\doxysubsection{\texorpdfstring{Tinting the basemap color}{Tinting the basemap color}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_speed_tree8-_sub_graph_assets_autotoc_md3939}
You can use the Speed\+Tree8\+Color\+Alpha Sub Graph Asset to apply a tint to the basemap color. This is useful if, for example, you want to adjust tree colors for different seasons of the year.

Crossfading dithers between different levels of detail (\href{https://docs.unity3d.com/Manual/LevelOfDetail.html}{\texttt{ LODs}}) to minimize \href{https://en.wikipedia.org/wiki/Popping_(computer_graphics)}{\texttt{ popping}} during abrupt transitions. The Speed\+Tree8\+Color\+Alpha Sub Graph Asset uses a \href{https://docs.unity3d.com/Packages/com.unity.shadergraph@12.0/manual/Custom-Function-Node.html}{\texttt{ Custom Function Node}} for that purpose.

This Custom Function Node is not Speed\+Tree-\/specific. Enable {\bfseries{Animate Cross-\/fading}} and select an {\bfseries{LOD Fade}} setting to use it with any asset that has the LOD Group component. See \href{https://docs.unity3d.com/Manual/class-LODGroup.html\#transitions}{\texttt{ Transitioning between LOD levels}} for more information.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_speed_tree8-_sub_graph_assets_autotoc_md3942}{}\doxysubsection{\texorpdfstring{Hiding geometry seams}{Hiding geometry seams}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_speed_tree8-_sub_graph_assets_autotoc_md3942}
The Speed\+Tree8\+Color\+Alpha Sub Graph asset applies an alpha gradient to soften the transitions between geometry segments that sample different parts of the basemap.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_speed_tree8-_sub_graph_assets_autotoc_md3943}{}\doxysection{\texorpdfstring{Speed\+Tree8\+Wind \label{_SpeedTree8Wind}%
}{Speed\+Tree8\+Wind \label{_SpeedTree8Wind}%
}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_speed_tree8-_sub_graph_assets_autotoc_md3943}
The Speed\+Tree8\+Wind Sub Graph Asset uses a \href{https://docs.unity3d.com/Packages/com.unity.shadergraph@12.0/manual/Custom-Function-Node.html}{\texttt{ Custom Function Node}} to deform the vertices of Speed\+Tree 8 models in response to your application’s wind data. You can use this to make trees appear to bend in the wind.

\doxylink{namespace_unity}{Unity} applies wind data to the Speed\+Tree8\+Wind Sub Graph Asset is as follows\+:
\begin{DoxyItemize}
\item When a \href{https://docs.unity3d.com/Manual/class-WindZone.html}{\texttt{ Wind\+Zone}} affects a Speed\+Tree 8 Game\+Object that has \href{https://docs.unity3d.com/Manual/com.unity.modules.wind.html}{\texttt{ Wind}} enabled, \doxylink{namespace_unity}{Unity} generates Speed\+Tree 8 wind simulation data.
\item \doxylink{namespace_unity}{Unity} populates the Speed\+Tree\+Wind Cbuffer with that wind simulation data.
\item The Speed\+Tree8\+Wind Sub Graph Asset bases its deformation behavior on the data in the Speed\+Tree\+Wind Cbuffer.
\end{DoxyItemize}

This asset includes automated LOD vertex interpolation when {\bfseries{LOD Fade}} is set to {\bfseries{Speed\+Tree}}. However, it does not support instancing.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_speed_tree8-_sub_graph_assets_autotoc_md3944}{}\doxysection{\texorpdfstring{Speed\+Tree8\+Billboard \label{_SpeedTree8Billboard}%

The Speed\+Tree8\+Billboard Sub Graph Asset calculates billboard normals from a Speed\+Tree 8 model\textquotesingle{}s bump map, geometric tangent, and bitangent data. It includes dithering functionality to improve the appearance of billboards at view angles diagonal to the model.

The keyword toggle associated with this feature is named {\ttfamily EFFECT\+\_\+\+BILLBOARD}.This supports backwards compatibility with previous versions of Shader\+Graph, which require keywords and their toggling properties to have identical names.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_speed_tree8-_sub_graph_assets_autotoc_md3945}{}\doxysection{\texorpdfstring{Speed\+Tree 8 Interpolated\+Normals}{Speed\+Tree 8 Interpolated\+Normals}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_speed_tree8-_sub_graph_assets_autotoc_md3945}
All Speed\+Tree 8 shaders that \doxylink{namespace_unity}{Unity} provides interpolate geometric normals, tangents, and bitangents in the vertex stage, because this results in a better visual appearance than the per-\/pixel data that Shader Graph nodes provide. You do not need to use this feature if your Speed\+Tree 8 Shader Graph does not include custom interpolators.

HDRP and URP do not have identical backface normal transformation behavior. This can become a problem when you use \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_custom-_interpolators}{Custom Interpolators}{0} for geometric normal, tangent, and bitangent data. The purpose of the Speed\+Tree 8 Interpolated\+Normals Sub Graph Asset is to allow for that difference. It combines geometric normal data with bump maps in a way that is compatible with the target pipeline\textquotesingle{}s backface normal transformation behavior.
