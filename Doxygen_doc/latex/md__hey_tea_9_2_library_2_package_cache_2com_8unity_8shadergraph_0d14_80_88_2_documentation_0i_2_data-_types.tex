There are a number of {\bfseries{Data Types}} in Shader Graph. Each {\bfseries{Port}} on a \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_node}{Node}{0} has an associated {\bfseries{Data Type}} that defines what edges can be connected to it. The {\bfseries{Data Types}} have colors for usability, these colors are applied to ports and edges of that {\bfseries{Data Type}}.

Some {\bfseries{Data Types}} have associated \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_property-_types}{Property Types}{0} for exposing these values to the \href{https://docs.unity3d.com/Manual/UsingTheInspector.html}{\texttt{ Inspector}} for \href{https://docs.unity3d.com/Manual/class-Material.html}{\texttt{ Materials}} that use the shader.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_data-_types_autotoc_md3135}{}\doxysection{\texorpdfstring{Data Types}{Data Types}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_data-_types_autotoc_md3135}

All {\bfseries{Vector}} types can be promoted or truncated to match any {\bfseries{Vector}} type \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_port}{Port}{0}. This behaviour occurs only when the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_port}{Port}{0} in question is not of type {\bfseries{Dynamic Vector}}. When truncating, excess channels are simply removed. When promoting, the extra required channels are filled by default values. These values values are (0, 0, 0, 1).\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_data-_types_autotoc_md3137}{}\doxysection{\texorpdfstring{Dynamic Data Types}{Dynamic Data Types}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_data-_types_autotoc_md3137}
Some {\bfseries{Data Types}} are dynamic. This means a port using these {\bfseries{Data Types}} can change their underlying {\bfseries{Concrete Data Type}} based on what {\bfseries{Data Type}} is connected to it. By default, \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_node}{Nodes}{0} using dynamic {\bfseries{Data Types}} can only have one {\bfseries{Concrete Data Type}}, meaning that once a connected edge has applied its {\bfseries{Data Type}} to that port, all other {\bfseries{Dynamic Data Type}} slots of that \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_node}{Node}{0} will apply the same {\bfseries{Data Type}}.

One notable exception to this is the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_multiply-_node}{Multiply Node}{0} which allows both {\bfseries{Dynamic}} {\bfseries{Matrix}} and {\bfseries{Vector}} types.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_data-_types_autotoc_md3138}{}\doxysubsection{\texorpdfstring{Dynamic Vector}{Dynamic Vector}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_data-_types_autotoc_md3138}
The {\bfseries{Dynamic Vector}} type allows connected edges of any {\bfseries{Vector}} type. All connected edges are automatically truncated to the type with the lowest dimension, unless the lowest dimension is 1, in which case the {\bfseries{Float}} is promoted.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_data-_types_autotoc_md3139}{}\doxysubsection{\texorpdfstring{Dynamic Matrix}{Dynamic Matrix}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_data-_types_autotoc_md3139}
The {\bfseries{Dynamic Matrix}} type allows connected edges of any {\bfseries{Matrix}} type. All connected edges are automatically truncated to the type with the lowest dimension.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_data-_types_autotoc_md3140}{}\doxysubsection{\texorpdfstring{Dynamic}{Dynamic}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_data-_types_autotoc_md3140}
The {\bfseries{Dynamic}} type is a special case. \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_node}{Nodes}{0} that support it must define how it is validated. In the case of the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_multiply-_node}{Multiply Node}{0}, it allows connections of any {\bfseries{Vector}} or {\bfseries{Matrix}} type, ensuring the correct multiplication is applied depending on the mix of {\bfseries{Data Types}}.
