In previous versions of URP, if post-\/processing was enabled it would cause the pipeline to always require depth. We have improved the post-\/processing integration to only require depth from the pipeline when Depth of Field, Motion Blur or SMAA effects are enabled. This improves performance in many cases.

Because Cameras that use post-\/processing no longer require depth by default, you must now manually indicate that Cameras require depth if you are using it for other effects, such as soft particles.

To make all Cameras require depth, enable the the {\ttfamily Depth Texture} option in the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do6a4fd81cefe986341b093c28ca9c1f30}{Pipeline Asset}{0}. To make an individual Camera require depth, set {\ttfamily Depth Texture} option to {\ttfamily On} in the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_doc7c35c176c1ad0ea77080cb33c453ec9}{Camera Inspector}{0}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_dod7feaf10650c864ca40427e6ba472f17_autotoc_md2599}{}\doxysection{\texorpdfstring{Sampling shadows from the Main Light}{Sampling shadows from the Main Light}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_dod7feaf10650c864ca40427e6ba472f17_autotoc_md2599}
In previous versions of URP, if shadow cascades were enabled for the main Light, shadows would be resolved in a screen space pass. The pipeline now always resolves shadows while rendering opaque or transparent objects. This allows for consistency and solved many issues regarding shadows.

If have custom HLSL shaders and sample {\ttfamily \+\_\+\+Screen\+Space\+Shadowmap\+Texture} texture, you must upgrade them to sample shadows by using the {\ttfamily Get\+Main\+Light} function instead.

For example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{float4\ shadowCoord\ =\ TransformWorldToShadowCoord(positionWorldSpace);}
\DoxyCodeLine{Light\ mainLight\ =\ GetMainLight(inputData.shadowCoord);}
\DoxyCodeLine{}
\DoxyCodeLine{//\ now\ you\ can\ use\ shadow\ to\ apply\ realtime\ occlusion}
\DoxyCodeLine{half\ shadow\ =\ mainLight.shadowAttenuation;}

\end{DoxyCode}


You must also define the following in your .shader file to make sure your custom shader can receive shadows correctly\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\#pragma\ multi\_compile\ \_\ \_MAIN\_LIGHT\_SHADOWS}
\DoxyCodeLine{\#pragma\ multi\_compile\ \_\ \_MAIN\_LIGHT\_SHADOWS\_CASCADE}

\end{DoxyCode}

Transparent objects can now receive shadows when using shadow cascades. You can also optionally disable shadow receiving for transparent to improve performance. To do so, disable {\ttfamily Transparent Receive Shadows} in the Forward Renderer asset.
