 Use blending properties to specify how the \doxylink{namespace_cinemachine}{Cinemachine} Brain component performs a blend between virtual cameras.

A \doxylink{namespace_cinemachine}{Cinemachine} blend is not a fade, wipe, or dissolve. Rather, \doxylink{namespace_cinemachine}{Cinemachine} Brain performs a smooth animation of the position, rotation, and other settings of the \doxylink{namespace_unity}{Unity} camera from one Virtual Camera to the next.

For blends between specific Virtual Cameras, use the {\bfseries{Custom Blends}} list in the \doxylink{namespace_cinemachine}{Cinemachine} Brain component. Use the {\bfseries{Default Blend}} property in \doxylink{namespace_cinemachine}{Cinemachine} Brain to specify blends between Virtual Cameras that do not have custom blends.



The {\bfseries{From}} and {\bfseries{To}} settings are name-\/based, not references. This means that \doxylink{namespace_cinemachine}{Cinemachine} finds cameras by matching their names to the settings. They are not linked to specific Game\+Objects. The built-\/in dropdowns can be used to select a virtual camera from the current scene, or the name can be typed directly into the text boxes. If a name does not match any virtual camera in the current scene, the field will be highlighted in yellow.

Use the reserved name \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\+ANY CAMERA\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*} to blend from or to any Virtual Camera.

When \doxylink{namespace_cinemachine}{Cinemachine} begins a transition from one virtual camera to another, it will look in this asset for an entry that matches the upcoming transition, and apply that blend definition. ~\newline



\begin{DoxyItemize}
\item If none is found, then the Cinemachine\+Brain\textquotesingle{}s Default\+Blend setting will apply. ~\newline

\item If multiple entries in the Custom Blends asset match the upcoming transition, \doxylink{namespace_cinemachine}{Cinemachine} will choose the one with the strongest specificity. For example, if blending from vcam1 to vcam2, and the custom blends asset contains an entry for {\itshape vcam1-\/to-\/\+Any\+Camera}, and another entry for {\itshape vcam1-\/to-\/vcam2}, then the {\itshape vcam1-\/to-\/vcam2} entry will apply.
\item If multiple entries in the Custom Blends asset match the upcoming transition with equally-\/strong specificity, then the first one found will apply.
