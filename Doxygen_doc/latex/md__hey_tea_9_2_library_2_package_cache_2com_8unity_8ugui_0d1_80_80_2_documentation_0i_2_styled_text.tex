 The text for UI elements and text meshes can incorporate multiple font styles and sizes. Rich text is supported both for the UI \doxylink{namespace_system}{System} and the legacy GUI system. The Text, GUIStyle, GUIText and Text\+Mesh classes have a {\bfseries{Rich Text}} setting which instructs \doxylink{namespace_unity}{Unity} to look for markup tags within the text. The \href{ScriptRef:Debug.Log.html}{\texttt{ Debug.\+Log}} function can also use these markup tags to enhance error reports from code. The tags are not displayed but indicate style changes to be applied to the text.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2_styled_text_autotoc_md4951}{}\doxysection{\texorpdfstring{Markup format}{Markup format}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2_styled_text_autotoc_md4951}
The markup system is inspired by HTML but isn\textquotesingle{}t intended to be strictly compatible with standard HTML. The basic idea is that a section of text can be enclosed inside a pair of matching tags\+:-\/

{\ttfamily We are \texorpdfstring{$<$}{<}b\texorpdfstring{$>$}{>}not\texorpdfstring{$<$}{<}/b\texorpdfstring{$>$}{>} amused.}

As the example shows, the tags are just pieces of text inside the "{}angle bracket"{} characters, {\ttfamily \texorpdfstring{$<$}{<}} and {\ttfamily \texorpdfstring{$>$}{>}}.

You place the {\itshape opening} tag at the beginning of the section. The text inside the tag denotes its name (which in this case is just {\bfseries{b}}).

You place another tag at the end of the section. This is the {\itshape closing} tag. It has the same name as the opening tag, but the name is prefixed with a slash {\ttfamily /} character. Every opening tag must have a corresponding closing tag. If you don\textquotesingle{}t {\itshape close} an opening tag, it is rendered as regular text.

The tags are not displayed to the user directly but are interpreted as instructions for styling the text they enclose. The {\ttfamily b} tag used in the example above applies boldface to the word "{}not"{}, so the text appears ons creen as\+:-\/

We are {\bfseries{not}} amused



A marked up section of text (including the tags that enclose it) is referred to as an {\bfseries{element}}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2_styled_text_autotoc_md4952}{}\doxysubsection{\texorpdfstring{Nested elements}{Nested elements}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2_styled_text_autotoc_md4952}
It is possible to apply more than one style to a section of text by "{}nesting"{} one element inside another

{\ttfamily We are \texorpdfstring{$<$}{<}b\texorpdfstring{$>$}{>}\texorpdfstring{$<$}{<}i\texorpdfstring{$>$}{>}definitely not\texorpdfstring{$<$}{<}/i\texorpdfstring{$>$}{>}\texorpdfstring{$<$}{<}/b\texorpdfstring{$>$}{>} amused}

The {\ttfamily \texorpdfstring{$<$}{<}i\texorpdfstring{$>$}{>}} tag applies italic style, so this would be presented onscreen as

We are \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\+\_\+definitely not\+\_\+\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*} amused



Note the ordering of the closing tags, which is in reverse to that of the opening tags. The reason for this is perhaps clearer when you consider that the inner tags need not span the whole text of the outermost element

{\ttfamily We are \texorpdfstring{$<$}{<}b\texorpdfstring{$>$}{>}absolutely \texorpdfstring{$<$}{<}i\texorpdfstring{$>$}{>}definitely\texorpdfstring{$<$}{<}/i\texorpdfstring{$>$}{>} not\texorpdfstring{$<$}{<}/b\texorpdfstring{$>$}{>} amused}

which gives

We are {\bfseries{absolutely {\itshape definitely} not}} amused

\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2_styled_text_autotoc_md4953}{}\doxysubsection{\texorpdfstring{Tag parameters}{Tag parameters}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2_styled_text_autotoc_md4953}
Some tags have a simple all-\/or-\/nothing effect on the text but others might allow for variations. For example, the {\bfseries{color}} tag needs to know which color to apply. Information like this is added to tags by the use of {\bfseries{parameters}}\+:-\/

{\ttfamily We are \texorpdfstring{$<$}{<}color=green\texorpdfstring{$>$}{>}green\texorpdfstring{$<$}{<}/color\texorpdfstring{$>$}{>} with envy}

Which produces this result\+:



Note that the ending tag doesn\textquotesingle{}t include the parameter value. Optionally, the value can be surrounded by quotation marks but this isn\textquotesingle{}t required.

Tag parameters cannot include blank spaces. For example\+:

{\ttfamily We are \texorpdfstring{$<$}{<}color = green\texorpdfstring{$>$}{>}green\texorpdfstring{$<$}{<}/color\texorpdfstring{$>$}{>} with envy}

does not work because of the spaces to either side of the {\ttfamily =} character.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2_styled_text_autotoc_md4954}{}\doxysection{\texorpdfstring{Supported tags}{Supported tags}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2_styled_text_autotoc_md4954}
