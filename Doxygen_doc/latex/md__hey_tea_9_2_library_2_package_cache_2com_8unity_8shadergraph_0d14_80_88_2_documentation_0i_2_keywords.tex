Use Keywords to create different variants for your Shader Graph.

Keywords enable you to create shaders\+:
\begin{DoxyItemize}
\item With features that you can turn on or off for each Material instance.
\item With features that behave differently on certain platforms.
\item That scale in complexity based on conditions you set.
\end{DoxyItemize}

There are three types of Keywords\+: Boolean, Enum, and Built-\/in. \doxylink{namespace_unity}{Unity} defines a Keyword in the graph, shader, and optionally, the Material Inspector based on its type. See \doxylink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_keywords_BooleanKeywords}{Boolean Keyword}, \doxylink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_keywords_EnumKeywords}{Enum Keyword}, and \doxylink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_keywords_BuiltinKeywords}{Built-\/in Keyword} for more information about Keyword types. For more information about how these Keywords affect the final shader, see documentation on \href{https://docs.unity3d.com/Manual/SL-MultipleProgramVariants.html}{\texttt{ Making multiple shader program variants}}.

In Shader Graph, you first define a Keyword on the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_blackboard}{Blackboard}{0}, then use a \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_keyword-_node}{Keyword Node}{0} to create a branch in the graph.

The Editor is able to compile variants on demand when it needs them to render content. If you declare many different variants, you can end up with millions or trillions of possibilities. However, the \doxylink{class_player}{Player} needs to determine at build time which variants are in use and include them when it pre-\/compiles your shaders. To manage memory effectively, the \doxylink{class_player}{Player} strips unused variants based on their keyword and Editor settings. See the next section, Common parameters, to learn more about how you can give the \doxylink{class_player}{Player} hints about what it needs to compile and what it can ignore. Boolean Keywords are either on or off. This results in two shader variants. \doxylink{namespace_unity}{Unity} exposes Boolean Keywords in the Material Inspector if the Exposed parameter is set to is true. To enable the keyword from a script, use Enable\+Keyword on the keyword\textquotesingle{}s Reference name. Disable\+Keyword disables the keyword. To learn more about Boolean Keywords, see \href{https://docs.unity3d.com/Manual/SL-MultipleProgramVariants.html}{\texttt{ Shader variants and keywords}}.

Enum Keywords can have two or more states, which you define in the {\bfseries{Entries}} list. If you expose an Enum Keyword, the {\bfseries{Display Names}} in its {\bfseries{Entries}} list appear in a dropdown menu in the Material Inspector.

Special characters such as ( ) or ! @ are not valid in the {\bfseries{Entry Name}} of an Enum Keyword. Shader Graph converts invalid characters to underscores ( \+\_\+ ).

When you define an Enum Keyword, Shader Graph displays labels for each state consisting of a sanitized version of the Enum\textquotesingle{}s {\bfseries{Entry Name}} appended to the main {\bfseries{Reference}} name. When controlling a keyword via script with a, {\ttfamily Material.\+Enable\+Keyword} or {\ttfamily Shader.\+Enable\+Keyword function}, enter the state label in the format {\ttfamily \{REFERENCE\}\+\_\+\{REFERENCESUFFIX\}}. For example, if your reference name is MYENUM and the desired entry is OPTION1, then you would call {\ttfamily Material.\+Enable\+Keyword("{}\+MYENUM\+\_\+\+OPTION1"{})}. When you select an option, this disables the other options.

Built-\/in Keywords are always of either the Boolean or Enum type, but they behave slightly differently from Boolean or Enum Keywords that you create. The \doxylink{namespace_unity}{Unity} Editor or active Render Pipeline sets their values, and you cannot edit these.

All Built-\/in Keyword fields in the {\bfseries{Node Settings}} tab of the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_internal-_inspector}{Graph Inspector}{0} are grayed out except for the {\bfseries{Default}} field, which you can enable or disable to show the differences in Shader Graph previews. You also cannot expose Built-\/in Keywords in the Material Inspector.
