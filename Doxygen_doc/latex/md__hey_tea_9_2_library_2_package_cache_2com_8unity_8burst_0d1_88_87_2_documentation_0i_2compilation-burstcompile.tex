 To improve the performance of \doxylink{namespace_burst}{Burst}, you can change how it behaves when it compiles a job with the \href{xref:Unity.Burst.BurstCompileAttribute}{\texttt{ {\ttfamily \mbox{[}Burst\+Compile\mbox{]}}}} attribute. Use it do the following\+:


\begin{DoxyItemize}
\item Use a different accuracy for math functions (for example, sin, cos).
\item Relax the order of math computations so that \doxylink{namespace_burst}{Burst} can rearrange the floating point calculations.
\item Force a synchronous compilation of a job (only for \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8burst_0d1_88_87_2_documentation_0i_2compilation}{just-\/in-\/time compilation}{0}).
\end{DoxyItemize}

For example, you can use the {\ttfamily \mbox{[}Burst\+Compile\mbox{]}} attribute to change the \href{xref:Unity.Burst.FloatPrecision}{\texttt{ floating precision}} and \href{xref:Unity.Burst.FloatMode}{\texttt{ float mode}} of \doxylink{namespace_burst}{Burst} like so\+: \begin{DoxyVerb}[BurstCompile(FloatPrecision.Med, FloatMode.Fast)]
\end{DoxyVerb}

Use the \href{xref:Unity.Burst.FloatPrecision}{\texttt{ {\ttfamily Float\+Precision}}} enumeration to define \doxylink{namespace_burst}{Burst}\textquotesingle{}s floating precision accuracy.

Float precision is measured in ulp (unit in the last place or unit of least precision). This is the space between floating-\/point numbers\+: the value the least significant digit represents if it\textquotesingle{}s 1. {\ttfamily \doxylink{namespace_unity_1_1_burst_a869a28a1e7255c50e703becf62bd9d1c}{Unity.\+Burst.\+Float\+Precision}} provides the following accuracy\+:


\begin{DoxyItemize}
\item {\ttfamily Float\+Precision.\+Standard}\+: Default value, which is the same as {\ttfamily Float\+Precision.\+Medium}. This provides an accuracy of 3.\+5 ulp.
\item {\ttfamily Float\+Precision.\+High}\+: Provides an accuracy of 1.\+0 ulp.
\item {\ttfamily Float\+Precision.\+Medium}\+: Provides an accuracy of 3.\+5 ulp.
\item {\ttfamily Float\+Precision.\+Low}\+: Has an accuracy defined per function, and functions might specify a restricted range of valid inputs.
\end{DoxyItemize}

Use the \href{xref:Unity.Burst.FloatMode}{\texttt{ {\ttfamily Float\+Mode}}} enumeration to define \doxylink{namespace_burst}{Burst}\textquotesingle{}s floating point math mode. It provides the following modes\+:


\begin{DoxyItemize}
\item {\ttfamily Float\+Mode.\+Default}\+: Defaults to {\ttfamily Float\+Mode.\+Strict} mode.
\item {\ttfamily Float\+Mode.\+Strict}\+: \doxylink{namespace_burst}{Burst} doesn\textquotesingle{}t perform any re-\/arrangement of the calculation and respects special floating point values (such as denormals, NaN). This is the default value.
\item {\ttfamily Float\+Mode.\+Fast}\+: \doxylink{namespace_burst}{Burst} can perform instruction re-\/arrangement and use dedicated or less precise hardware SIMD instructions.
\item {\ttfamily Float\+Mode.\+Deterministic}\+: Unsupported. Deterministic mode is reserved for a future iteration of \doxylink{namespace_burst}{Burst}.
\end{DoxyItemize}

For hardware that can support Multiply and Add (e.\+g mad {\ttfamily a \texorpdfstring{$\ast$}{*} b + c}) into a single instruction, you can use {\ttfamily Float\+Mode.\+Fast} to enable this optimization. However, the reordering of these instructions might lead to a lower accuracy.

Use {\ttfamily Float\+Mode.\+Fast} for scenarios where the exact order of the calculation and the uniform handling of NaN values aren\textquotesingle{}t required.
