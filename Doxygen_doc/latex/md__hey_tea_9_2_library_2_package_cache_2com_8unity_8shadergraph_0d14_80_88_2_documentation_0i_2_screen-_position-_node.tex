Provides access to the screen position of the mesh vertex or fragment. The X and Y values represent the horizontal and vertical positions respectively. Use the {\bfseries{Mode}} dropdown control to select the mode of the output value. The available modes are as follows\+:


\begin{DoxyItemize}
\item {\bfseries{Default}} -\/ Returns X and Y values that represent the normalized {\bfseries{Screen Position}}. The normalized {\bfseries{Screen Position}} is the {\bfseries{Screen Position}} divided by the clip space position W component. The X and Y value ranges are between 0 and 1 with position {\ttfamily float2(0,0)} at the lower left corner of the screen. The Z and W values aren\textquotesingle{}t used in this mode, so they\textquotesingle{}re always 0.
\item {\bfseries{Raw}} -\/ Returns the raw {\bfseries{Screen Position}} values, which are the {\bfseries{Screen Position}} values before the clip space position W component is divided out. Position {\ttfamily float2(0,0)} is at the lower left corner of the screen. This mode is useful for projection.
\item {\bfseries{Center}} -\/ Returns X and Y values that represent the normalized {\bfseries{Screen Position}} offset so position {\ttfamily float2(0,0)} is at the center of the screen. The range of the X and Y values is â€“1 to 1. The Z and W values aren\textquotesingle{}t used in this mode, so they\textquotesingle{}re always 0.
\item {\bfseries{Tiled}} -\/ Returns {\bfseries{Screen Position}} offset so position {\ttfamily float2(0,0)} is at the center of the screen and tiled using {\ttfamily frac}.
\item {\bfseries{Pixel}} -\/ Returns {\bfseries{Screen Position}} in terms of the actual pixel width and height values of the screen. In this mode, position {\ttfamily float2(0,0)} is at the lower left corner of the screen. Whereas the range of Default mode is always 0 to 1, the range of Pixel mode depends on the screen resolution. The Z and W values aren\textquotesingle{}t used in this mode, so they\textquotesingle{}re always 0.
