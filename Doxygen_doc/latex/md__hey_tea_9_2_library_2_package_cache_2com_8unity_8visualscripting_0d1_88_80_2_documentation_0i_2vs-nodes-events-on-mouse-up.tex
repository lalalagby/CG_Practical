The On Mouse Up node listens for a user to release their mouse button after they click a Collider in your application. \mbox{[}!include\doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0410c79c94b3bae67a92801d976700de0}{nodes-\/desc-\/end}{0}\mbox{]}

The user can release their mouse button anywhere in your application to trigger the On Mouse Up node. If you want the node to trigger after the user releases the mouse button over the same Collider specified in the node\textquotesingle{}s {\bfseries{Target}}, use the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_07ec4fc19ef4fa8bac0155678344c87f8}{On Mouse Up As Button node}{0} instead.

In the following example, the On Mouse Up node adds a force to a Game\+Object based on the user\textquotesingle{}s mouse position when they release their mouse button. The On Mouse Up node triggers a Camera Screen To World Point node to get the user\textquotesingle{}s mouse position, before it sends the {\bfseries{X}} value of the mouse to a Rigidbody Add Force node to move the Game\+Object.



When the user clicks on the sphere in the middle of the scene and releases their mouse button, the sphere moves towards their mouse location.
