 Shaders written for the Built-\/in Render Pipeline are not compatible with the URP shaders.

For an overview of the mapping between built-\/in shaders and URP shaders, see Shader mappings.

Use the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do54fb4e3619bda36fb863b138677b0598}{Render Pipeline Converter}{0} to apply the shader mappings automatically.

{\bfseries{Note}}\+: The Render Pipeline Converter makes irreversible changes to the project. Back up your project before the conversion.

{\bfseries{Tip}}\+: If the preview thumbnails in the Project view are not shown correctly after the conversion, try right-\/clicking anywhere in the Project view and selecting {\bfseries{Reimport All}}.

For \href{https://docs.unity3d.com/Manual/SpeedTree.html}{\texttt{ Speed\+Tree}} Shaders, \doxylink{namespace_unity}{Unity} does not re-\/generate Materials when you re-\/import them, unless you click the {\bfseries{Generate Materials}} or {\bfseries{Apply \& Generate Materials}} button.

You cannot upgrade Custom \doxylink{namespace_unity}{Unity} shaders written for the Built-\/in Render Pipeline. Instead, custom shaders must be rewritten to work with URP or recreated in \href{https://docs.unity3d.com/Packages/com.unity.shadergraph@14.0/manual/index.html}{\texttt{ Shader\+Graph}}.

Any Materials in a Scene that use a custom shader when you upgrade a project to use URP turn pink to indicate the Material no longer works. To fix this, upgrade or change the Material\textquotesingle{}s shader to one that is compatible with URP.

{\bfseries{Note}}\+: URP does not support \href{https://docs.unity3d.com/Manual/SL-SurfaceShaders.html}{\texttt{ Surface Shaders}}.
