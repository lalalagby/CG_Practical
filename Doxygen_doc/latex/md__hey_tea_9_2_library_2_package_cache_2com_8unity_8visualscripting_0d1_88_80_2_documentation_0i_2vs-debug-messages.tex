 You can use debug nodes to see the result of a code segment inside the \doxylink{namespace_unity}{Unity} console. For example, attaching a Debug node to a Variable node outputs the value of the variable.

\begin{DoxyRemark}{Remarks}
Debugs are a useful tool when prototyping.
\end{DoxyRemark}
\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-debug-messages_autotoc_md5312}{}\doxysection{\texorpdfstring{To create a debug message}{To create a debug message}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-debug-messages_autotoc_md5312}
\begin{DoxyImportant}{Important}
Remove or disconnect the debugs from the graphs before producing the final executable.
\end{DoxyImportant}

\begin{DoxyEnumerate}
\item Add a Script Machine component to the Game\+Object.
\item Select {\bfseries{Edit Graph}}.
\item Do one of the following\+:
\begin{DoxyItemize}
\item Use the starting events that are created with the script machine.
\item Add an event node to the graph.
\end{DoxyItemize}
\item Drag and release from the node port.
\end{DoxyEnumerate}The fuzzy finder appears.
\begin{DoxyEnumerate}
\item In the finder field, enter “\+Log”.
\end{DoxyEnumerate}A list of Debug nodes appears.
\begin{DoxyEnumerate}
\item Select the relevant debug message type you want to use (for example {\bfseries{Log(\+Message)}}, {\bfseries{Log Error(\+Message)}} or {\bfseries{Log Warning(\+Message)}}).
\end{DoxyEnumerate}The select Debug node is placed in the graph and linked to the event.
\begin{DoxyEnumerate}
\item Drag and release from the Debug green (output) port.
\end{DoxyEnumerate}The fuzzy finder appears.
\begin{DoxyEnumerate}
\item In the list, select the String node.
\end{DoxyEnumerate}A String node appears on the graph, connected to the Debug node.\+v Enter the debug message in the string node\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}.\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}

\begin{DoxyRemark}{Remarks}
You can link variables or Game\+Objects other than a string to the port to see the value in the console.
\end{DoxyRemark}
Whenever the graph is run and the Event is fired, the debug node executes and the text in the String appears in the console.

\begin{DoxyNote}{Note}
The debug bar indicates the number of messages of each debug type (in the following order Message, Error, Warning).
\end{DoxyNote}
