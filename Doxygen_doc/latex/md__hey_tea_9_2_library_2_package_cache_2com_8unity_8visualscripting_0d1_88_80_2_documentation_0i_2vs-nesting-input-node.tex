 Use an Input node to control the flow of logic and data from a Script Graph\textquotesingle{}s Subgraph node. An Input node takes data from a parent graph and makes it available to a Subgraph.



For more information on Subgraphs, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0b052f136142546305a9d053d2653f106}{Subgraphs and State Units}{0} and \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-nesting-subgraph-node}{Subgraph node}{0}. The Input node is in the {\bfseries{Nesting}} category in the fuzzy finder.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-nesting-input-node_autotoc_md5410}{}\doxysection{\texorpdfstring{Available outputs}{Available outputs}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-nesting-input-node_autotoc_md5410}
By default, an Input node has no ports.

An Input node can only have output ports. Define the number and specific data type for the output ports with the \href{vs-interface-overview.md\#the-graph-inspector}{\texttt{ Graph Inspector}}. For more information on how to define ports on a Script Graph, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_06f95905303333ea92719076820d2246b}{Add a Trigger or Data port to a Script Graph}{0}.

In the following example, the {\bfseries{Character Move}} Subgraph uses an Input node to receive data from a parent graph. The Input node has one Trigger Input port and three Data Input ports. It uses the values from the parent graph and the values from two Input Get Axis nodes to create a new Vector 3 value that it sends back to its parent graph.



The parent graph sends three values from the current Game\+Object\textquotesingle{}s Transform component to the Input node. The Subgraph reduces the number of nodes in the parent graph.
