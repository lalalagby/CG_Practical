 To use the package, go to {\bfseries{Edit}} \texorpdfstring{$>$}{>} {\bfseries{Preferences}} \texorpdfstring{$>$}{>} {\bfseries{External Tools}} \texorpdfstring{$>$}{>} {\bfseries{External Script Editor}} and select the version of {\bfseries{Visual Studio}} you have installed. When you select this option, the window reloads and displays settings that control production of .csproj files.

\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ide_8visualstudio_0d2_80_820_2_documentatio41bcd64f816092ee2d6eb05e1d234092_autotoc_md1006}{}\doxysection{\texorpdfstring{Generate .csproj files}{Generate .csproj files}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ide_8visualstudio_0d2_80_820_2_documentatio41bcd64f816092ee2d6eb05e1d234092_autotoc_md1006}
Each setting in the table below enables or disables the production of .csproj files for a different type of package.\+When you click {\bfseries{Regenerate project files}}, \doxylink{namespace_unity}{Unity} updates the existing .csproj files and creates the necessary new ones based on the settings you choose.

These settings control whether to generate .csproj files for any installed packages. For more information on how to install packages, see \href{https://docs.unity3d.com/Manual/upm-ui-actions.html}{\texttt{ Adding and removing packages}}.
