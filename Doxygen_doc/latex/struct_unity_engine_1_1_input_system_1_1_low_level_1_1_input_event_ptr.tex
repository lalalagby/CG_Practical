\doxysubsection{Detailed Description}
\doxylink{class_unity_engine_1_1_input_system_1_1_pointer}{Pointer} to an \doxylink{struct_unity_engine_1_1_input_system_1_1_low_level_1_1_input_event}{Input\+Event}. Makes it easier to work with Input\+Events and hides the unsafe operations necessary to work with them. 

Note that event pointers generally refer to event buffers that are continually reused. This means that event pointers should not be held on to. Instead, to hold onto event data, manually copy an event to a buffer.

bool Unity\+Engine.\+Input\+System.\+Low\+Level.\+Input\+Event\+Ptr.\+handled\hspace{0.3cm}{\ttfamily [get]}, {\ttfamily [set]}}



Whether the event is considered "{}handled"{} and should not be processed further. 

This is used in two ways. Setting it from inside Input\+System.\+on\+Event will cause the event to not be processed further. If it is a \doxylink{struct_unity_engine_1_1_input_system_1_1_low_level_1_1_state_event}{State\+Event} or \doxylink{struct_unity_engine_1_1_input_system_1_1_low_level_1_1_delta_state_event}{Delta\+State\+Event}, the \doxylink{class_unity_engine_1_1_input_system_1_1_input_device}{Input\+Device} targeted by the event will not receive the state change.

Setting this flag from inside a state change monitor (see Input\+State.\+Add\+Change\+Monitor(\+Input\+Control,\+IInput\+State\+Change\+Monitor,long,uint)) will prevent other monitors on the same \doxylink{interface_unity_engine_1_1_input_system_1_1_low_level_1_1_i_input_state_change_monitor}{IInput\+State\+Change\+Monitor} not receiving the state change.
