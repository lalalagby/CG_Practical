\doxysubsection{Detailed Description}
Records the triggering of actions into a sequence of events that can be replayed at will. 

This is an alternate way to the callback-\/based responses (such as \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_a11c046d8fb911d300e30e67827222c1f}{Input\+Action.\+performed}) of \doxylink{class_unity_engine_1_1_input_system_1_1_input_action}{input actions}. Instead of executing response code right away whenever an action triggers, an \doxylink{class_unity_engine_1_1_input_system_1_1_utilities_1_1_input_action_trace_a54cb478410315b156285e74d840fb716}{event is recorded} which can then be queried on demand.

The recorded data will stay valid even if the bindings on the actions are changed (e.\+g. by enabling a different set of bindings through altering \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_a41caa3a44bc876781c5b1bcb04622cd2}{Input\+Action.\+binding\+Mask} or \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_map_ab32ae5eb39f8c39dcebccb5cc1bb99e9}{Input\+Action\+Map.\+devices} or when modifying the paths of bindings altogether). Note, however, that when this happens, a trace will have to make a private copy of the data that stores the binding resolution state. This means that there can be GC allocation spike when reconfiguring actions that have recorded data in traces.
