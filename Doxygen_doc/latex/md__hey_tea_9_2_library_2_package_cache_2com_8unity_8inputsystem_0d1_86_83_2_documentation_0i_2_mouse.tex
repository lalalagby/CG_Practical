On desktop platforms (Windows, Mac, Linux, and UWP), you can move the mouse cursor via code. Note that this moves the system\textquotesingle{}s actual mouse cursor, not just \doxylink{namespace_unity}{Unity}\textquotesingle{}s internally-\/stored mouse position. This means that the user sees the cursor jumping to a different position, which is generally considered to be bad UX practice. It\textquotesingle{}s advisable to only do this if the cursor is hidden (see the \href{https://docs.unity3d.com/ScriptReference/Cursor.html}{\texttt{ {\ttfamily Cursor} API documentation}} for more information).

To move the cursor to a different position, use \href{../api/UnityEngine.InputSystem.Mouse.html\#UnityEngine_InputSystem_Mouse_WarpCursorPosition_UnityEngine_Vector2_}{\texttt{ {\ttfamily Mouse.\+Warp\+Cursor\+Position}}}. The coordinates are expressed as \doxylink{namespace_unity}{Unity} screen coordinates, just like \href{../api/UnityEngine.InputSystem.Pointer.html\#UnityEngine_InputSystem_Pointer_position}{\texttt{ {\ttfamily Mouse.\+position}}}.
