 The Screen Space Shadows \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do4e3c1604c0720cb973c486c434c47f28}{Renderer Feature}{0} calculates screen-\/space shadows for opaque objects affected by the main directional light and draws them in the scene. To render screen-\/space shadows, URP requires an additional render target. This increases the amount of memory your application requires, but if your project uses forward rendering, screen-\/space shadows can benefit the runtime resource intensity. This is because if you use screen-\/space shadows, URP doesn\textquotesingle{}t need to access the cascade shadow maps multiple times. ~\newline

After you enable this Renderer Feature, URP renders screen-\/space shadows in your scene. To distinguish between shadow map shadows and screen-\/space shadows, you can view the render passes that draws the shadows in the \href{https://docs.unity3d.com/Manual/FrameDebugger.html}{\texttt{ Frame Debugger}}. ~\newline

You can compare shadows cast on opaque objects from the screen-\/space shadow texture or the cascade shadow maps. ~\newline

This Renderer Feature uses a depth texture and invokes a depth prepass before it draws opaque objects. It calculates the shadows in screen space before the {\ttfamily Draw\+Opaque\+Objects} render pass. URP doesn\textquotesingle{}t calculate or apply screen-\/space shadows for transparent objects; it uses \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do063b72965a5e74fa753731d1e69475cb}{shadow maps}{0} for transparent objects instead. \texorpdfstring{$\ast$}{*}~\newline
