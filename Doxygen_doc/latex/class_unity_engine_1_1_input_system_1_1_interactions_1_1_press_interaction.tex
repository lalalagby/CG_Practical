\doxysubsection{Detailed Description}
Performs the action at specific points in a button press-\/and-\/release sequence according top \doxylink{class_unity_engine_1_1_input_system_1_1_interactions_1_1_press_interaction_a49992f8e75518463b42367c8e4070204}{behavior}. 

By default, uses \doxylink{namespace_unity_engine_1_1_input_system_1_1_interactions_ab32b275dd7694d64e9af44106156e15baf44beb60e4a2c95a291880b333ac1f4f}{Press\+Behavior.\+Press\+Only} which performs the action as soon as the control crosses the button press threshold defined by \doxylink{class_unity_engine_1_1_input_system_1_1_interactions_1_1_press_interaction_a754f87d03a1df1e494e728bcf787bcf8}{press\+Point}. The action then will not trigger again until the control is first released.

Can be set to instead trigger on release (that is, when the control goes back below the button press threshold) using \doxylink{namespace_unity_engine_1_1_input_system_1_1_interactions_ab32b275dd7694d64e9af44106156e15ba8a4da72e7f9d025075c95b3f83d1ab92}{Press\+Behavior.\+Release\+Only} or can be set to trigger on both press and release using \doxylink{namespace_unity_engine_1_1_input_system_1_1_interactions_ab32b275dd7694d64e9af44106156e15bab573614837664b2daa76c6ee27b469e9}{Press\+Behavior.\+Press\+And\+Release}).

Note that using an explicit press interaction is only necessary if the goal is to either customize the press behavior of a button or when binding to controls that are not buttons as such (the press interaction compares magnitudes to \doxylink{class_unity_engine_1_1_input_system_1_1_interactions_1_1_press_interaction_a754f87d03a1df1e494e728bcf787bcf8}{press\+Point} and thus any type of control that can deliver a magnitude can act as a button). The default behavior available out of the box when binding \doxylink{namespace_unity_engine_1_1_input_system_a0d0d044aae0aef15123e2f31f11ea94ba87b7760f14fbff78d8819291f36ab9a0}{Input\+Action\+Type.\+Button} type actions to button-\/type controls (\doxylink{class_unity_engine_1_1_input_system_1_1_controls_1_1_button_control}{Unity\+Engine.\+Input\+System.\+Controls.\+Button\+Control}) corresponds to using a press modifier with \doxylink{class_unity_engine_1_1_input_system_1_1_interactions_1_1_press_interaction_a49992f8e75518463b42367c8e4070204}{behavior} set to \doxylink{namespace_unity_engine_1_1_input_system_1_1_interactions_ab32b275dd7694d64e9af44106156e15baf44beb60e4a2c95a291880b333ac1f4f}{Press\+Behavior.\+Press\+Only} and \doxylink{class_unity_engine_1_1_input_system_1_1_interactions_1_1_press_interaction_a754f87d03a1df1e494e728bcf787bcf8}{press\+Point} left at default.

This method is called whenever a control referenced in the binding that the interaction sits on changes value. The interaction is expected to process the value change and, if applicable, call \doxylink{struct_unity_engine_1_1_input_system_1_1_input_interaction_context_ac7e1fd7757b79512fbdd9e653e0f6863}{Input\+Interaction\+Context.\+Started} and/or its related methods to initiate a state change.

Note that if "{}control disambiguation"{} (i.\+e. the process where if multiple controls are bound to the same action, the system decides which control gets to drive the action at any one point) is in effect -- i.\+e. when either \doxylink{namespace_unity_engine_1_1_input_system_a0d0d044aae0aef15123e2f31f11ea94ba87b7760f14fbff78d8819291f36ab9a0}{Input\+Action\+Type.\+Button} or \doxylink{namespace_unity_engine_1_1_input_system_a0d0d044aae0aef15123e2f31f11ea94ba689202409e48743b914713f96d93947c}{Input\+Action\+Type.\+Value} are used but not if \doxylink{namespace_unity_engine_1_1_input_system_a0d0d044aae0aef15123e2f31f11ea94bae1ff6a7360e99cb46db6bb5d3ff8e2eb}{Input\+Action\+Type.\+Pass\+Through} is used -- inputs that the disambiguation chooses to ignore will cause this method to not be called.

Note that this method is called on the interaction even when there are multiple interactions and the interaction is not the one currently in control of the action (because another interaction that comes before it in the list had already started the action). Each interaction will get processed independently and the action will decide when to use which interaction to drive the action as a whole.
