It is possible to change the \href{https://docs.unity3d.com/ScriptReference/BuildPlayerOptions.html}{\texttt{ Build\+Player\+Options}} for the test {\bfseries{\doxylink{class_player}{Player}}}, to achieve custom behavior when running {\bfseries{Play Mode}} tests. Modifying the build options allows for changing the target location of the build as well as changing \href{https://docs.unity3d.com/ScriptReference/BuildOptions.html}{\texttt{ Build\+Options}}.

To modify the {\ttfamily Build\+Player\+Options}, do the following\+:


\begin{DoxyItemize}
\item Implement the {\ttfamily ITest\+Player\+Build\+Modifier}
\item Reference the implementation type in a {\ttfamily Test\+Player\+Build\+Modifier} attribute on an assembly level.
\end{DoxyItemize}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_087f859c00ce91a90a3fefba0f4300807_autotoc_md4240}{}\doxysubsection{\texorpdfstring{Example}{Example}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_087f859c00ce91a90a3fefba0f4300807_autotoc_md4240}

It is possible to use the \doxylink{namespace_unity}{Unity} Editor for building the \doxylink{class_player}{Player} with tests, without \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_067b7200309be97aaf3a215eb01c30ccc}{running the tests}{0}. This allows for running the \doxylink{class_player}{Player} on e.\+g. another machine. In this case, it is necessary to modify the \doxylink{class_player}{Player} to build and implement a custom handling of the test result.

By using {\ttfamily Test\+Player\+Build\+Modifier}, you can alter the {\ttfamily Build\+Options} to not start the \doxylink{class_player}{Player} after the build as well as build the \doxylink{class_player}{Player} at a specific location. Combined with \href{./reference-setup-and-cleanup.md\#prebuildsetup-and-postbuildcleanup}{\texttt{ Post\+Build\+Cleanup}}, you can automatically exit the Editor on completion of the build.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_087f859c00ce91a90a3fefba0f4300807_autotoc_md4242}{}\doxysubsection{\texorpdfstring{Example}{Example}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_087f859c00ce91a90a3fefba0f4300807_autotoc_md4242}

If the Editor is still running after the Play Mode tests have run, the \doxylink{class_player}{Player} tries to report the results back, using \href{https://docs.unity3d.com/ScriptReference/Networking.PlayerConnection.PlayerConnection.html}{\texttt{ Player\+Connection}}, which has a reference to the IP address of the Editor machine, when built.

To implement a custom way of reporting the results of the test run, let one of the assemblies in the \doxylink{class_player}{Player} include a \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_04c3d386a07f44ed21ed1c4d5c0a1b316}{Test\+Run\+Callback}{0}. At {\ttfamily Run\+Finished}, it is possible to get the full test report as XML from the \href{http://www.nunit.org/}{\texttt{ NUnit}} test result by calling {\ttfamily result.\+To\+Xml(true)}. You can save the result and then save it on the device or send it to another machine as needed.
