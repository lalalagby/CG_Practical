\doxysubsection{Detailed Description}
A two-\/axis thumbstick control that can act as both a vector and a four-\/way dpad. 

Stick controls are used to represent the thumbsticks on gamepads (see \doxylink{class_unity_engine_1_1_input_system_1_1_gamepad_a9f608a9e12af3301fb2d704e072ea024}{Gamepad.\+left\+Stick} and \doxylink{class_unity_engine_1_1_input_system_1_1_gamepad_a24a1a82fd0aab986f8e7e141dab4035f}{Gamepad.\+right\+Stick}) as well as the main stick control of joysticks (see \doxylink{class_unity_engine_1_1_input_system_1_1_joystick_aede6b1d8f1033dd09c96cc6ca8fa3a83}{Joystick.\+stick}).

Essentially, a stick is an extended {\ttfamily Vector2} control that can function either as a combined 2D vector, as independent vertical and horizontal axes, or as four individual, directional buttons. The following example demonstrates this based on the gamepad\textquotesingle{}s left stick.

\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{comment}{//\ Read\ X\ and\ Y\ axis\ of\ stick\ individually.}}
\DoxyCodeLine{Gamepad.current.leftStick.x.ReadValue();}
\DoxyCodeLine{Gamepad.current.leftStick.y.ReadValue();}
\DoxyCodeLine{}

\DoxyCodeLine{Gamepad.current.leftStick.up.ReadValue();}

\DoxyCodeLine{Gamepad.current.leftStick.left.ReadValue();}
\DoxyCodeLine{Gamepad.current.leftStick.right.ReadValue();}

\end{DoxyCode}
 

In terms of memory, a stick controls is still just from one value for the X axis and one value for the Y axis.

Unlike dpads (see \doxylink{class_unity_engine_1_1_input_system_1_1_controls_1_1_dpad_control}{Dpad\+Control}), sticks will usually have deadzone processors (see Stick\+Deadzone\+Processor) applied to them to get rid of noise around the resting point of the stick. The X and Y axis also have deadzones applied to them by default (Axis\+Deadzone\+Processor). Note, however, that the deadzoning of individual axes is different from the deadzoning applied to the stick as a whole and thus does not have to result in exactly the same values. Deadzoning of individual axes is linear (i.\+e. the result is simply clamped and normalized back into \mbox{[}0..1\mbox{]} range) whereas the deadzoning of sticks is radial (i.\+e. the length of the vector is taken into account which means that {\itshape both} the X and Y axis contribute).
