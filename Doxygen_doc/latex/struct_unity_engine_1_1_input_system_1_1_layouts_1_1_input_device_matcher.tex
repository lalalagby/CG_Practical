\doxysubsection{Detailed Description}
Specification that can be matched against an \doxylink{struct_unity_engine_1_1_input_system_1_1_layouts_1_1_input_device_description}{Input\+Device\+Description}. This is used to find which \doxylink{class_unity_engine_1_1_input_system_1_1_layouts_1_1_input_control_layout}{Input\+Control\+Layout} to create for a device when it is discovered. 

Each matcher is basically a set of key/value pairs where each value may either be a regular expression or a plain value object. The central method for testing a given matcher against an \doxylink{struct_unity_engine_1_1_input_system_1_1_layouts_1_1_input_device_description}{Input\+Device\+Description} is \doxylink{struct_unity_engine_1_1_input_system_1_1_layouts_1_1_input_device_matcher_a5e15ff278278cda90188af18f2ec4475}{Match\+Percentage}.

Various helper methods such as \doxylink{struct_unity_engine_1_1_input_system_1_1_layouts_1_1_input_device_matcher_a5a1bfe691a99a357ba7e255f31fc97c3}{With\+Interface} or \doxylink{struct_unity_engine_1_1_input_system_1_1_layouts_1_1_input_device_matcher_a888eb3272d49c57567881bad869f3d85}{With\+Capability$<$\+TValue$>$} assist with creating matchers.

A score usually in the range between 0 and 1.

The algorithm computes a score of how well the matcher matches the given description. Essentially, a matcher that matches every single property that is present (as in not {\ttfamily null} and not an empty string) in {\itshape device\+Description}  receives a score of 1, a matcher that matches none a score of 0. Matches that match only a subset receive a score in-\/between.

An exception to this are capabilities. Every single match of a capability is counted as one property match and added to the score. This means that matchers that match on multiple capabilities may actually achieve a score \texorpdfstring{$>$}{>}1.
