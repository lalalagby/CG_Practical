override void \mbox{\hyperlink{class_unity_engine_1_1_input_system_1_1_dual_shock_1_1_dual_shock_gamepad_a85d2170e70d13524a1081f22307e8b45}{Make\+Current}} ()
\begin{DoxyCompactList}\small\item\em Make this the current device of its type.

This method is called automatically by the input system when a device is added or when input is received on it. Many types of devices have {\ttfamily .current} getters that allow querying the last used device of a specific type directly (for example, see \doxylink{class_unity_engine_1_1_input_system_1_1_gamepad_a6f0bd4b71340da07b56f6ec56097da1d}{Gamepad.\+current}).

There is one special case, however, related to noise. A device that has noisy controls (i.\+e. controls for which \doxylink{class_unity_engine_1_1_input_system_1_1_input_control_afa5a8531d7c9a030e8e484560a59fe6f}{Input\+Control.\+noisy} is true) may receive input events that contain no meaningful user interaction but are simply just noise from the device. A good example of this is the PS4 gamepad which has a built-\/in gyro and may thus constantly feed events into the input system even if not being actually in use. If, for example, an Xbox gamepad and PS4 gamepad are both connected to a PC and the user is playing with the Xbox gamepad, the PS4 gamepad would still constantly make itself \doxylink{class_unity_engine_1_1_input_system_1_1_gamepad_a6f0bd4b71340da07b56f6ec56097da1d}{Gamepad.\+current} by simply flooding the system with events. Hence why by default, noise on {\ttfamily .current} getters will be filtered out and a device will only see {\ttfamily Make\+Current} getting called if there input was detected on non-\/noisy controls.  \end{DoxyCompactList}\item

{\footnotesize\ttfamily \label{class_unity_engine_1_1_input_system_1_1_dual_shock_1_1_dual_shock_gamepad_a85d2170e70d13524a1081f22307e8b45} 
override void Unity\+Engine.\+Input\+System.\+Dual\+Shock.\+Dual\+Shock\+Gamepad.\+Make\+Current (\begin{DoxyParamCaption}{}{}\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}, {\ttfamily [virtual]}}



Make this the current device of its type.

This method is called automatically by the input system when a device is added or when input is received on it. Many types of devices have {\ttfamily .current} getters that allow querying the last used device of a specific type directly (for example, see \doxylink{class_unity_engine_1_1_input_system_1_1_gamepad_a6f0bd4b71340da07b56f6ec56097da1d}{Gamepad.\+current}).

There is one special case, however, related to noise. A device that has noisy controls (i.\+e. controls for which \doxylink{class_unity_engine_1_1_input_system_1_1_input_control_afa5a8531d7c9a030e8e484560a59fe6f}{Input\+Control.\+noisy} is true) may receive input events that contain no meaningful user interaction but are simply just noise from the device. A good example of this is the PS4 gamepad which has a built-\/in gyro and may thus constantly feed events into the input system even if not being actually in use. If, for example, an Xbox gamepad and PS4 gamepad are both connected to a PC and the user is playing with the Xbox gamepad, the PS4 gamepad would still constantly make itself \doxylink{class_unity_engine_1_1_input_system_1_1_gamepad_a6f0bd4b71340da07b56f6ec56097da1d}{Gamepad.\+current} by simply flooding the system with events. Hence why by default, noise on {\ttfamily .current} getters will be filtered out and a device will only see {\ttfamily Make\+Current} getting called if there input was detected on non-\/noisy controls.
