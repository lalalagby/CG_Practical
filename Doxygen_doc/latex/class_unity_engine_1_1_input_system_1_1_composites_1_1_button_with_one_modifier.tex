\doxysubsection{Detailed Description}
A button with an additional modifier. The button only triggers when the modifier is pressed. 

This composite can be used to require another button to be held while pressing the button that triggers the action. This is most commonly used on keyboards to require one of the modifier keys (shift, ctrl, or alt) to be held in combination with another key, e.\+g. "{}\+CTRL+1"{}.

If set to {\ttfamily true}, \doxylink{class_unity_engine_1_1_input_system_1_1_composites_1_1_button_with_one_modifier_a3bcc676251e6c69189f231f7364a71dc}{modifier} can be pressed after \doxylink{class_unity_engine_1_1_input_system_1_1_composites_1_1_button_with_one_modifier_aaa8328374419a4f09a781edfdcdfb23e}{button} and the composite will still trigger. Default is false. 

By default, \doxylink{class_unity_engine_1_1_input_system_1_1_composites_1_1_button_with_one_modifier_a3bcc676251e6c69189f231f7364a71dc}{modifier} is required to be in pressed state before or at the same time that \doxylink{class_unity_engine_1_1_input_system_1_1_composites_1_1_button_with_one_modifier_aaa8328374419a4f09a781edfdcdfb23e}{button} goes into pressed state for the composite as a whole to trigger. This means that binding to, for example, {\ttfamily Shift+B}, the {\ttfamily shift} key has to be pressed before pressing the {\ttfamily B} key. This is the behavior usually expected with keyboard shortcuts.

This parameter can be used to bypass this behavior and allow any timing between \doxylink{class_unity_engine_1_1_input_system_1_1_composites_1_1_button_with_one_modifier_a3bcc676251e6c69189f231f7364a71dc}{modifier} and \doxylink{class_unity_engine_1_1_input_system_1_1_composites_1_1_button_with_one_modifier_aaa8328374419a4f09a781edfdcdfb23e}{button}. The only requirement is for them both to concurrently be in pressed state.
