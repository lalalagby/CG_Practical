 You can create your own Custom C\# node with a C\# script. Use your node in Script Graphs to run your own custom logic, and provide the script file to other users to let them use it in their own Visual Scripting projects.

You can create nodes that change the flow of logic in your graph, nodes that manipulate data, or both. You can also create \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-custom-events}{nodes for custom events}{0}.

To create a Custom C\# node, you must\+:


\begin{DoxyEnumerate}
\item Create a C\# script.
\item Add ports.
\item Add logic.
\item Add relations.
\end{DoxyEnumerate}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-custom-node_autotoc_md5255}{}\doxysection{\texorpdfstring{Create the initial C\# script}{Create the initial C\# script}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-custom-node_autotoc_md5255}
All custom nodes start with a C\# file.

After you create a C\# file, you must regenerate your Node Library through your \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-configuration}{project settings}{0}. This allows Visual Scripting to recognize your Custom C\# node and add it to \href{vs-interface-overview.md\#the-fuzzy-finder}{\texttt{ the fuzzy finder}} for use in your project\textquotesingle{}s graphs.

After you regenerate your Node Library the first time, you only need to regenerate your Node Library again if you change the location of a node in the fuzzy finder. You can change the location through the \href{vs-create-custom-node-attributes-reference.md\#unitcategory}{\texttt{ \mbox{[}Unit\+Category\mbox{]} attribute}}.

For more information on how to create your initial C\# script, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-custom-node-empty}{Create a new simple Custom C\# node}{0}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-custom-node_autotoc_md5256}{}\doxysection{\texorpdfstring{Add ports}{Add ports}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-custom-node_autotoc_md5256}
Add ports to a node to specify what triggers Visual Scripting to run logic in a node, and decide what data it sends or receives. For more information on ports and nodes, see \href{vs-nodes.md\#connections-and-ports}{\texttt{ Nodes}}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-custom-node_autotoc_md5257}{}\doxysubsection{\texorpdfstring{Port types}{Port types}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-custom-node_autotoc_md5257}
Visual Scripting has four different port types that you can add to a node\+:


\begin{DoxyItemize}
\item {\ttfamily Control\+Input}\+: Provides a connection to a previous node, which tells Visual Scripting when to start the logic in a node.
\item {\ttfamily Control\+Output}\+: Provides a connection to another node, which tells Visual Scripting when to run the logic for the next connected node in a Script Graph. ~\newline

\item {\ttfamily Value\+Input}\+: Provides a connection that allows you to pass data into a node for use in its logic.
\item {\ttfamily Value\+Output}\+: Provides a connection that allows you to pass data out of a node for use in other nodes.
\end{DoxyItemize}

You can add any number of ports to a node. You can also choose what data type the {\ttfamily Value\+Input} or {\ttfamily Value\+Output} ports send and receive\+:


\begin{DoxyItemize}
\item {\bfseries{Generic}}\+: The port can receive or output any data type. Corresponds to \doxylink{namespace_unity}{Unity}\textquotesingle{}s {\bfseries{Object}} type. ~\newline

\item {\bfseries{Type Value}}\+: The port can only receive or output a specific data type. For example, {\ttfamily string}, {\ttfamily float}, or {\ttfamily integer}.
\end{DoxyItemize}

For more information on types in Visual Scripting, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-types}{Object types}{0}. For more information on how to add ports to your node, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_069a57fd3a9eaef02c28bc1eb0e25b20e}{Add ports to your Custom C\# node}{0}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-custom-node_autotoc_md5258}{}\doxysection{\texorpdfstring{Add logic}{Add logic}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-custom-node_autotoc_md5258}
Add logic to a node to specify what it does when it runs in a Script Graph. If there isn\textquotesingle{}t any internal logic written for a node, the node can\textquotesingle{}t trigger another node, or modify any of the data it receives from other nodes in a Script Graph.

For more information on how to add logic to a node, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_003994998919097210a00a89943bb35d8}{Add logic to a Custom C\# node}{0}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-custom-node_autotoc_md5259}{}\doxysection{\texorpdfstring{Add relations}{Add relations}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-custom-node_autotoc_md5259}
Relations help define how a node and its Play mode animations appear in Visual Scripting. Without relations, Visual Scripting doesn\textquotesingle{}t know how to animate or display a node in the \href{vs-interface-overview.md\#the-graph-editor}{\texttt{ Graph Editor}}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-custom-node_autotoc_md5260}{}\doxysubsection{\texorpdfstring{Relation types}{Relation types}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-custom-node_autotoc_md5260}
You can add three types of relations to a node to help correctly display its internal flow of logic\+:


\begin{DoxyItemize}
\item {\bfseries{Assignment}}\+: Assignment relations are usually between a control input port and a data output port. Set an Assignment relation to tell Visual Scripting that a specific port needs to run before the node sends data to a data output port.
\item {\bfseries{Succession}}\+: Succession relations are usually between a control input port and a control output port. Set a Succession relation to tell Visual Scripting that a control input port exits at a control output port.
\item {\bfseries{Requirement}}\+: Requirement relations are usually between a control input port and a data port or ports. Set a Requirement relation to let Visual Scripting know that a specific data port or ports must have a value before the node can run any logic.
\end{DoxyItemize}

\begin{DoxyNote}{Note}
If you don\textquotesingle{}t set any Succession relations in a node, and {\bfseries{Dim}} is enabled in \href{vs-interface-overview.md\#the-graph-toolbar}{\texttt{ the Graph toolbar}}, Visual Scripting dims your Custom C\# node and any of its connected nodes in the Graph Editor during Play mode.
\end{DoxyNote}

After you\textquotesingle{}ve added relations, you can choose to \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0e3ad062e5030dcc2af8dd7bf95bec507}{add documentation for a node}{0}. Visual Scripting displays documentation in the \href{vs-interface-overview.md\#the-graph-inspector}{\texttt{ Graph Inspector}} when a user selects a node in a Script Graph. Add documentation to help other users understand the purpose of each part of a node.

You can also choose to \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_01cf766d7dc572e06b22fb591b88ded19}{customize a node with node attributes}{0}. You can add specific \href{vs-create-custom-node-attributes-reference.md\#node-class-attributes}{\texttt{ node class attributes}} to customize the entire node, or add \href{vs-create-custom-node-attributes-reference.md\#port-attributes}{\texttt{ port attributes}} to only customize specific ports. The attribute type determines the location where you must place the attribute in the node\textquotesingle{}s C\# file. Node class attributes must be placed above a node class definition, while port attributes must be placed above the variable definition for each port.
