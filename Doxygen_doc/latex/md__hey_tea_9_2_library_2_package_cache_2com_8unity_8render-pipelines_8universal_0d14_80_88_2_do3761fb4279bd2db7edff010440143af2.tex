 This section contains descriptions of URP-\/specific Shader\+Lab Pass tags.

\begin{quote}
{\bfseries{NOTE}}\+: URP does not support the following Light\+Mode tags\+: {\ttfamily Always}, {\ttfamily Forward\+Add}, {\ttfamily Prepass\+Base}, {\ttfamily Prepass\+Final}, {\ttfamily Vertex}, {\ttfamily Vertex\+LMRGBM}, {\ttfamily Vertex\+LM}. \end{quote}

The value of this tag lets the pipeline determine which Pass to use when executing different parts of the Render Pipeline.

If you do not set the {\ttfamily Light\+Mode} tag in a Pass, URP uses the {\ttfamily SRPDefault\+Unlit} tag value for that Pass.

The {\ttfamily Light\+Mode} tag can have the following values.

{\bfseries{Universal\+Forward}}   &The Pass renders object geometry and evaluates all light contributions. URP uses this tag value in the Forward Rendering Path.    \\\cline{1-2}
{\bfseries{Universal\+GBuffer}}   &The Pass renders object geometry without evaluating any light contribution. Use this tag value in Passes that \doxylink{namespace_unity}{Unity} must execute in the Deferred Rendering Path.    \\\cline{1-2}
{\bfseries{Universal\+Forward\+Only}}   &The Pass renders object geometry and evaluates all light contributions, similarly to when {\bfseries{Light\+Mode}} has the {\bfseries{Universal\+Forward}} value. The difference from {\bfseries{Universal\+Forward}} is that URP can use the Pass for both the Forward and the Deferred Rendering Paths.~\newline
Use this value if a certain Pass must render objects with the Forward Rendering Path when URP is using the Deferred Rendering Path. For example, use this tag if URP renders a Scene using the Deferred Rendering Path and the Scene contains objects with shader data that does not fit the GBuffer, such as Clear Coat normals.~\newline
If a shader must render in both the Forward and the Deferred Rendering Paths, declare two Passes with the {\ttfamily Universal\+Forward} and {\ttfamily Universal\+GBuffer} tag values.~\newline
If a shader must render using the Forward Rendering Path regardless of the Rendering Path that the URP Renderer uses, declare only a Pass with the {\ttfamily Light\+Mode} tag set to {\ttfamily Universal\+Forward\+Only}.~\newline
If you use the SSAO Renderer Feature, add a Pass with the {\ttfamily Light\+Mode} tag set to {\ttfamily Depth\+Normals\+Only}. For more information, see the {\ttfamily Depth\+Normals\+Only} value.    \\\cline{1-2}
{\bfseries{Depth\+Normals\+Only}}   &Use this value in combination with {\ttfamily Universal\+Forward\+Only} in the Deferred Rendering Path. This value lets \doxylink{namespace_unity}{Unity} render the shader in the Depth and normal prepass. In the Deferred Rendering Path, if the Pass with the {\ttfamily Depth\+Normals\+Only} tag value is missing, \doxylink{namespace_unity}{Unity} does not generate the ambient occlusion around the Mesh.    \\\cline{1-2}
{\bfseries{Universal2D}}   &The Pass renders objects and evaluates 2D light contributions. URP uses this tag value in the 2D Renderer.    \\\cline{1-2}
{\bfseries{Shadow\+Caster}}   &The Pass renders object depth from the perspective of lights into the Shadow map or a depth texture.    \\\cline{1-2}
{\bfseries{Depth\+Only}}   &The Pass renders only depth information from the perspective of a Camera into a depth texture.    \\\cline{1-2}
{\bfseries{Meta}}   &\doxylink{namespace_unity}{Unity} executes this Pass only when baking lightmaps in the \doxylink{namespace_unity}{Unity} Editor. \doxylink{namespace_unity}{Unity} strips this Pass from shaders when building a \doxylink{class_player}{Player}.    \\\cline{1-2}
{\bfseries{SRPDefault\+Unlit}}   &Use this {\ttfamily Light\+Mode} tag value to draw an extra Pass when rendering objects. Application example\+: draw an object outline. This tag value is valid for both the Forward and the Deferred Rendering Paths.~\newline
URP uses this tag value as the default value when a Pass does not have a {\ttfamily Light\+Mode} tag.   \\\cline{1-2}

The {\ttfamily Universal\+Material\+Type} tag can have the following values.

If this tag is not set in a Pass, \doxylink{namespace_unity}{Unity} uses the {\ttfamily Lit} value.

{\bfseries{Simple\+Lit}}   &This value indicates that the shader type is Simple\+Lit. During the G-\/buffer Pass, \doxylink{namespace_unity}{Unity} uses stencil to mark the pixels that use the Simple\+Lit shader type (specular model is Blinn-\/\+Phong).   \\\cline{1-2}
