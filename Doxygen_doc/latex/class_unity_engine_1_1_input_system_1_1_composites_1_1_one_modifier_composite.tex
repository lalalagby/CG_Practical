\doxysubsection{Detailed Description}
A binding with an additional modifier. The bound controls only trigger when the modifier is pressed. 

This composite can be used to require a button to be held in order to "{}activate"{} another binding. This is most commonly used on keyboards to require one of the modifier keys (shift, ctrl, or alt) to be held in combination with another control, e.\+g. "{}\+CTRL+1"{}.

bool Unity\+Engine.\+Input\+System.\+Composites.\+One\+Modifier\+Composite.\+override\+Modifiers\+Need\+To\+Be\+Pressed\+First}



If set to {\ttfamily true}, the built-\/in logic to determine if modifiers need to be pressed first is overridden. Default value is {\ttfamily false}. 

By default, if \doxylink{class_unity_engine_1_1_input_system_1_1_composites_1_1_one_modifier_composite_a8b879c56640440fe929e844e17068273}{binding} is bound to only \doxylink{class_unity_engine_1_1_input_system_1_1_controls_1_1_button_control}{Controls.\+Button\+Control}s, then the composite requires \doxylink{class_unity_engine_1_1_input_system_1_1_composites_1_1_one_modifier_composite_adcff2867c05082270c023d7a80a29c78}{modifier} to be pressed {\itshape before} pressing \doxylink{class_unity_engine_1_1_input_system_1_1_composites_1_1_one_modifier_composite_a8b879c56640440fe929e844e17068273}{binding}. This means that binding to, for example, {\ttfamily Ctrl+B}, the {\ttfamily ctrl} keys have to be pressed before pressing the {\ttfamily B} key. This is the behavior usually expected with keyboard shortcuts.
