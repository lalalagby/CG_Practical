 Visual Scripting has a set of default assemblies and generated nodes for \doxylink{namespace_unity}{Unity} features. Add more assemblies through the Visual Scripting Node Library in your Project Settings. Assemblies are special files that contain the code for the feature you want to use.

Visual Scripting can generate nodes from assemblies in packages and third-\/party assets. To use a new package or third-\/party asset in Visual Scripting, you must import it into \doxylink{namespace_unity}{Unity}. For more information on how to add packages to \doxylink{namespace_unity}{Unity}, see \href{https://docs.unity3d.com/2021.2/Documentation/Manual/upm-ui-actions.html}{\texttt{ Adding and removing}} in the \doxylink{namespace_unity}{Unity} User Manual. For more information on how to add third-\/party assets to \doxylink{namespace_unity}{Unity}, see \href{https://docs.unity3d.com/2021.2/Documentation/Manual/ImportingAssets.html}{\texttt{ Importing assets}} in the User Manual.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-add-remove-node-library_autotoc_md5149}{}\doxysection{\texorpdfstring{Add assemblies and nodes to the Node Library}{Add assemblies and nodes to the Node Library}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-add-remove-node-library_autotoc_md5149}
