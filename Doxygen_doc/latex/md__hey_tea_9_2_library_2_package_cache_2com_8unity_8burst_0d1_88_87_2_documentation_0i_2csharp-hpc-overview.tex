\doxylink{namespace_burst}{Burst} supports {\ttfamily throw} expressions for exceptions. Exceptions thrown in the {\bfseries{editor}} can be caught by managed code, and are reported in the console window. Exceptions thrown in {\bfseries{player builds}} will always cause the application to abort. Thus with \doxylink{namespace_burst}{Burst} you should only use exceptions for exceptional behavior. To ensure that code doesn\textquotesingle{}t end up relying on exceptions for things like general control flow, \doxylink{namespace_burst}{Burst} produces the following warning on code that tries to {\ttfamily throw} within a method not attributed with {\ttfamily \mbox{[}Conditional("{}\+ENABLE\+\_\+\+UNITY\+\_\+\+COLLECTIONS\+\_\+\+CHECKS"{})\mbox{]}}\+:

\begin{quote}
\doxylink{namespace_burst}{Burst} warning BC1370\+: An exception was thrown from a function without the correct \mbox{[}Conditional("{}\+ENABLE\+\_\+\+UNITY\+\_\+\+COLLECTIONS\+\_\+\+CHECKS"{})\mbox{]} guard. Exceptions only work in the editor and so should be protected by this guard \end{quote}

HPC\# doesn\textquotesingle{}t support the following C\# features\+:


\begin{DoxyItemize}
\item Catching exceptions {\ttfamily catch} in a {\ttfamily try}/{\ttfamily catch}.
\item Storing to static fields except via \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8burst_0d1_88_87_2_documentation_0i_2csharp-shared-static}{Shared Static}{0}
\item Any methods related to managed objects, for example, string methods. 
\end{DoxyItemize}
