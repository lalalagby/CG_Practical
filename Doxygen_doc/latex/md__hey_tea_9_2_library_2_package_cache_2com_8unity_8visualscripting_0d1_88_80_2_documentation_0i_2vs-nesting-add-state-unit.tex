 Add a State Unit to a Script Graph to trigger a change of state. A state is any set of behaviors that you want a Game\+Object to perform. For more information on state in Visual Scripting, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-graph-types}{Graphs}{0}.

You can add a new State Graph to a State Unit node, or use an existing State Graph from the project. For more information on the State Unit node, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-nesting-state-unit-node}{State Unit node}{0}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-nesting-add-state-unit_autotoc_md5397}{}\doxysection{\texorpdfstring{Add a new State Graph as a State Unit node}{Add a new State Graph as a State Unit node}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-nesting-add-state-unit_autotoc_md5397}
To add a new blank State Graph as a State Unit node to a Script Graph\+:


\begin{DoxyEnumerate}
\item 

\mbox{[}!include\doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_06fd8d5e00402366f8aaeca25bc9a4448}{with-\/graph-\/open-\/ff}{0}\mbox{]}


\item 

Go to {\bfseries{Nesting}}.


\item 

Select {\bfseries{State Unit}} to add a State Unit node to the graph.


\item 

Open the \href{vs-interface-overview.md\#the-graph-inspector}{\texttt{ Graph Inspector}}.


\item 

In the Graph Inspector, choose the source for the State Unit\+:


\begin{DoxyItemize}
\item 

{\bfseries{Embed}}\+: The State Graph only exists on the State Unit node. You can only change the State Graph from the node in its parent graph.


\item 

{\bfseries{Graph}}\+: The State Graph exists in a separate file. You can change the State Graph outside of its parent graph and reuse the graph in other areas of an application.


\end{DoxyItemize}
\item 

If you chose {\bfseries{Graph}}\+:


\begin{DoxyEnumerate}[label=\enumalphalphcnt*]

\item 

In the Graph Inspector, select {\bfseries{New}}.


\item 

Enter a name for the graph file.


\item 

Choose where you want to save the graph file in the project.


\item 

Select {\bfseries{Save}}.


\end{DoxyEnumerate}
\end{DoxyEnumerate}

To add an existing State Graph file as a State Unit node in a Script Graph\+:


\begin{DoxyEnumerate}
\item \mbox{[}!include\doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_06fd8d5e00402366f8aaeca25bc9a4448}{with-\/graph-\/open-\/ff}{0}\mbox{]}
\item Go to {\bfseries{Nesting}}.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Select {\bfseries{State Unit}} to add a State Unit node to the graph.
\item Open the \href{vs-interface-overview.md\#the-graph-inspector}{\texttt{ Graph Inspector}}.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item In the Graph Inspector, set the {\bfseries{Source}} to {\bfseries{Graph}}.
\item Do one of the following\+:
\begin{DoxyItemize}
\item In the {\bfseries{Graph}} field, select the object picker (circle icon) and choose a compatible State Graph from the project.
\item Click and drag a State Graph file from the Project window and release on the {\bfseries{Graph}} field.
\end{DoxyItemize}
\end{DoxyEnumerate}

\begin{DoxyRemark}{Remarks}
For a faster way to add a State Graph as a State Unit node\+:
\begin{DoxyItemize}
\item Click and drag a State Graph asset from the Project window into the Graph Editor to automatically create a State Unit node.
\item Right-\/click to open the fuzzy finder. Go to {\bfseries{Graphs}} and select a graph file.
\end{DoxyItemize}
\end{DoxyRemark}
