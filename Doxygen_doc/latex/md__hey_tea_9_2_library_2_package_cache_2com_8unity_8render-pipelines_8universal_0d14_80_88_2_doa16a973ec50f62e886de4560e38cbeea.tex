This section describes how to configure the following application example\+:


\begin{DoxyItemize}
\item The Scene contains two Point Lights (marked {\ttfamily A} and {\ttfamily B} in the illustration) and two Sphere Game\+Objects ({\ttfamily C} and {\ttfamily D} in the illustration).
\item Light {\ttfamily A} affects Sphere {\ttfamily D}, but not Sphere {\ttfamily C}. Light {\ttfamily B} affects Sphere {\ttfamily C}, but not Sphere {\ttfamily D}.
\end{DoxyItemize}

The following illustration shows the example\+:

~\newline
{\itshape Light {\ttfamily A} affects Sphere {\ttfamily D}, but not Sphere {\ttfamily C}. Light {\ttfamily B} affects Sphere {\ttfamily C}, but not Sphere {\ttfamily D}.}

To implement the example\+:


\begin{DoxyEnumerate}
\item Enable Rendering Layers in your project.
\item Create two Point Lights (call them {\ttfamily A}, and {\ttfamily B}) and two Spheres (call them {\ttfamily C}, and {\ttfamily D}). Position the objects so that both Spheres are within the emission range of Lights.
\item Go to {\bfseries{Project Settings \texorpdfstring{$>$}{>} \doxylink{namespace_graphics}{Graphics} \texorpdfstring{$>$}{>} URP Global Settings}}. Rename Rendering Layer 1 to {\ttfamily Red}, and Layer 2 to {\ttfamily Green}.


\item Select Light {\ttfamily A}, change its color to green. Select Light {\ttfamily B}, change its color to red. With this setup, both Lights affect both Spheres.


\item Make the following settings on Lights and Spheres\+:

Light {\ttfamily A}\+: in the property {\bfseries{Light \texorpdfstring{$>$}{>} General \texorpdfstring{$>$}{>} Rendering Layers}}, clear all options, and select {\ttfamily Green}.

Light {\ttfamily B}\+: in the property {\bfseries{Light \texorpdfstring{$>$}{>} General \texorpdfstring{$>$}{>} Rendering Layers}}, clear all options, and select {\ttfamily Red}.

Sphere {\ttfamily C}\+: in the property {\bfseries{Mesh Renderer \texorpdfstring{$>$}{>} Additional Settings \texorpdfstring{$>$}{>} Rendering Layer Mask}}, select all options, clear {\ttfamily Green}.

Sphere {\ttfamily D}\+: in the property {\bfseries{Mesh Renderer \texorpdfstring{$>$}{>} Additional Settings \texorpdfstring{$>$}{>} Rendering Layer Mask}}, select all options, clear {\ttfamily Red}.

Now Point Light {\ttfamily A} affects Sphere {\ttfamily D}, but not Sphere {\ttfamily C}. Point Light {\ttfamily B} affects Sphere {\ttfamily C}, but not Sphere {\ttfamily D}.


\end{DoxyEnumerate}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_doa16a973ec50f62e886de4560e38cbeea_autotoc_md2151}{}\doxysection{\texorpdfstring{\label{_shadow-layers}%

In the illustration above, Light {\ttfamily A} does not affect Sphere {\ttfamily C}, and the Sphere does not cast shadow from Light {\ttfamily A}.

The {\bfseries{Custom Shadow Layers}} property lets you configure the Scene so that Sphere {\ttfamily C} casts the shadow from Light {\ttfamily A}.


\begin{DoxyEnumerate}
\item Select Light {\ttfamily A}.
\item In {\bfseries{Light \texorpdfstring{$>$}{>} Shadows}}, select the {\bfseries{Custom Shadow Layers}} property. \doxylink{namespace_unity}{Unity} shows the {\bfseries{Layer}} property.
\item In the {\bfseries{Layer}} property, select the Rendering Layer that Sphere C belongs to.
\end{DoxyEnumerate}

Now Light {\ttfamily A} does not affect Sphere {\ttfamily C}, but Sphere {\ttfamily C} casts shadow from Light {\ttfamily A}.

This section describes how to configure the following application example\+:


\begin{DoxyItemize}
\item The Scene contains a Decal Projector.
\item The Decal Projector projects a decal on the wall and the ground, but not on the paint bucket.
\end{DoxyItemize}

The following illustration shows the example\+:

~\newline
{\itshape In image {\ttfamily 1}, the paint bucket has the {\ttfamily Receive decals} layer selected. In image {\ttfamily 2} it does not, so the Decal Projector does not project on the bucket.}

To implement the example\+:


\begin{DoxyEnumerate}
\item Enable Rendering Layers for Decals in your project.
\item \href{../renderer-feature-decal.md\#how-to-use-the-feature}{\texttt{ Create a Decal Projector}} in the Scene.
\item Go to {\bfseries{Project Settings \texorpdfstring{$>$}{>} \doxylink{namespace_graphics}{Graphics} \texorpdfstring{$>$}{>} URP Global Settings}}. Add a Rendering Layer called {\ttfamily Receive decals}.


\item Select the Decal Projector. In the Rendering Layers property, select {\ttfamily Receive decals}.


\item Select the paint bucket Game\+Object. In the {\bfseries{Rendering Layer Mask}} field, clear the {\ttfamily Receive decals} layer. Now the Decal Projector does not affect this Game\+Object.

This section contains information related to the impact of Rendering Layers on performance.


\begin{DoxyItemize}
\item Keep the Rendering Layer count as small as possible. Avoid creating Rendering Layers that you don\textquotesingle{}t use in the project.
\item When using Rendering Layers for decals, increasing the layer count increases the required memory bandwidth and decreases the performance.
\item When using Rendering Layers only for Lights in the Forward Rendering Path, the performance impact is insignificant.
\item Performance impact grows more significantly when the Rendering Layer count exceeds a multiple of 8. For example\+: increasing the layer count from 8 to 9 layers has a bigger relative impact than increasing the layer count from 9 to 10 layers. 
\end{DoxyItemize}
