bool Unity\+Engine.\+Input\+System.\+Layouts.\+Input\+Control\+Layout\+Attribute.\+is\+Noisy\hspace{0.3cm}{\ttfamily [get]}, {\ttfamily [set]}}



Allows marking a device as noisy regardless of control layout. 

\doxylink{namespace_unity_engine_1_1_input_system_1_1_controls}{Controls} can be individually marked as noisy using the \doxylink{class_unity_engine_1_1_input_system_1_1_layouts_1_1_input_control_attribute_aa7fe5c7bc349cd4a78b9b78b59579e62}{Input\+Control\+Attribute.\+noisy} attribute, but this property can be used to mark a device as noisy even when no control has been marked as such. This can be useful when a device state layout has only been partially implemented i.\+e. some data in the state memory has not been mapped to a control, and the unimplemented controls are noisy. Without doing this, the device will constantly be made current as the system has no way to know that the event data contains only noise. \Hypertarget{class_unity_engine_1_1_input_system_1_1_layouts_1_1_input_control_layout_attribute_a9a94fcc2ecb4ced620b2473239a79bda}\index{UnityEngine.InputSystem.Layouts.InputControlLayoutAttribute@{UnityEngine.InputSystem.Layouts.InputControlLayoutAttribute}!stateFormat@{stateFormat}}
