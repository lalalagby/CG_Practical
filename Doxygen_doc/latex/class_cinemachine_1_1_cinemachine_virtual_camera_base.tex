\doxysubsection{Detailed Description}
Base class for a Monobehaviour that represents a Virtual Camera within the \doxylink{namespace_unity}{Unity} scene. 

This is intended to be attached to an empty Transform Game\+Object. Inherited classes can be either standalone virtual cameras such as \doxylink{class_cinemachine_1_1_cinemachine_virtual_camera}{Cinemachine\+Virtual\+Camera}, or meta-\/cameras such as \doxylink{class_cinemachine_1_1_cinemachine_clear_shot}{Cinemachine\+Clear\+Shot} or \doxylink{class_cinemachine_1_1_cinemachine_free_look}{Cinemachine\+Free\+Look}.

A \doxylink{class_cinemachine_1_1_cinemachine_virtual_camera_base}{Cinemachine\+Virtual\+Camera\+Base} exposes a Priority property. When the behaviour is enabled in the game, the Virtual Camera is automatically placed in a queue maintained by the static \doxylink{class_cinemachine_1_1_cinemachine_core}{Cinemachine\+Core} singleton. The queue is sorted by priority. When a \doxylink{namespace_unity}{Unity} camera is equipped with a \doxylink{class_cinemachine_1_1_cinemachine_brain}{Cinemachine\+Brain} behaviour, the brain will choose the camera at the head of the queue. If you have multiple \doxylink{namespace_unity}{Unity} cameras with \doxylink{class_cinemachine_1_1_cinemachine_brain}{Cinemachine\+Brain} behaviours (say in a split-\/screen context), then you can filter the queue by setting the culling flags on the virtual cameras. The culling mask of the \doxylink{namespace_unity}{Unity} Camera will then act as a filter for the brain. Apart from this, there is nothing that prevents a virtual camera from controlling multiple \doxylink{namespace_unity}{Unity} cameras simultaneously.
