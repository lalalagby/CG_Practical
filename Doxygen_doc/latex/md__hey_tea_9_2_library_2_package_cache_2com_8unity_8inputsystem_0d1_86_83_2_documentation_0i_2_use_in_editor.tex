The coordinate system differs between {\ttfamily Editor\+Window} code and {\ttfamily Unity\+Engine.\+Screen}. {\ttfamily Editor\+Window} code has its origin in the upper-\/left corner, with Y down. {\ttfamily Unity\+Engine.\+Screen} has it in the bottom-\/left corner, with Y up.

The Input \doxylink{namespace_system}{System} compensates for that by automatically converting coordinates depending on whether you call it from your application or from Editor code. In other words, calling {\ttfamily Mouse.\+current.\+position.\+Read\+Value()} from inside {\ttfamily Editor\+Window} code returns mouse coordinates in Editor UI coordinates (Y down), and reading the position elsewhere returns it in application screen coordinates (Y up).

Internally, an editor-\/specific Processor called {\ttfamily Auto\+Window\+Space} handles this translation.
