The Forward+ Rendering Path lets you avoid the per object limit of the Forward Rendering Path.

The Forward+ Rendering Path has the following advantages compared with the Forward Rendering Path\+:


\begin{DoxyItemize}
\item There is no per-\/object limit for the number of Lights that affect Game\+Objects, the per-\/\+Camera limit still applies.~\newline
The per-\/\+Camera limits for different platforms are\+:
\begin{DoxyItemize}
\item Desktop and console platforms\+: 256 Lights
\item Mobile platforms\+: 32 Lights. Open\+GL ES 3.\+0 and earlier\+: 16 Lights.
\end{DoxyItemize}This implementation lets you avoid splitting big meshes when more than 8 lights affect them.
\item Blending of more than 2 reflection probes.
\item Support for multiple Lights when using \doxylink{namespace_unity}{Unity} Entity Component \doxylink{namespace_system}{System} (ECS).
\item More flexibility with procedural draws.
\end{DoxyItemize}

The LOD cross-\/fade lets you achieve a smoother transition blending between the current mesh LOD and the next LOD based on the object\textquotesingle{}s distance to the Camera.

As the Camera moves, \doxylink{namespace_unity}{Unity} shows different LODs to provide a good balance between quality and processing cost. Cross-\/fading lets you avoid harsh LOD snapping and popping.

For more information, see the \href{../universalrp-asset.md\#lod-cross-fade}{\texttt{ LOD Cross Fade}} property.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2682}{}\doxysubsection{\texorpdfstring{Temporal anti-\/aliasing (TAA)}{Temporal anti-\/aliasing (TAA)}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2682}
Temporal anti-\/aliasing (TAA) is a spatial multi-\/frame anti-\/aliasing technique that uses results from current and previous rendered frames to remove jaggies in the current frame and reduce temporal judder between frames.

TAA uses Motion Vectors to reduce or avoid shimmer and ghosting artifacts caused by moving objects that end up being in different pixel locations in different frames.

To enable TAA for a Camera\+:


\begin{DoxyItemize}
\item Select the Camera.
\item In the Inspector, in the {\bfseries{Rendering}} section, select {\bfseries{Temporal Anti-\/aliasing (TAA)}} in the {\bfseries{Anti-\/aliasing}} property.
\end{DoxyItemize}

This release implements multiple performance and quality improvements to the SSAO feature.

~\newline
{\itshape The Falloff Distance property lets you improve performance for Scenes with a lot of distant objects}

Performance improvements\+:


\begin{DoxyItemize}
\item New {\bfseries{Blur Quality}} property with three blur options\+: High, Medium, Low.
\item The {\bfseries{Downsample}} check box now not only affects the Ambient Occlusion pass but also the following blur passes.
\item The {\bfseries{Falloff Distance}} property lets you reduce computational work on objects far away from the Camera.
\item The last Blur pass and the After Opaque pass are now merged into one when the {\bfseries{After Opaque}} option is enabled.
\item The {\bfseries{\doxylink{namespace_samples}{Samples}}} property now has three options with pre-\/defined sample counts that provide a good balance of visual quality and performance.
\end{DoxyItemize}

Quality improvements\+:


\begin{DoxyItemize}
\item The {\bfseries{Method}} property lets you choose the algorithm that \doxylink{namespace_unity}{Unity} uses to calculate the ambient occlusion values. The {\bfseries{Blue Noise}} algorithm is added in this release.
\item A new depth test was added to avoid adding SSAO to objects far away from one another.
\end{DoxyItemize}

For more information, see the page \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5dc499cc0fd2cb2a0ae33d2260a7f243}{Screen Space Ambient Occlusion}{0}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2685}{}\doxysubsection{\texorpdfstring{64 bit high precision HDR render target format}{64 bit high precision HDR render target format}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2685}
URP can now render into 64-\/bit high precision HDR render target format. Compared to the default 32-\/bit HDR render target, the 64-\/bit option provides the following benefits\+:


\begin{DoxyItemize}
\item Better precision for all color channels which reduces banding.
\item Even precision for color channels eliminates the subtle blue-\/yellow banding caused by the lower blue channel precision of the 32-\/bit target.
\item Support for the alpha channel. The alpha channel enables the alpha output with some limitations.
\end{DoxyItemize}

The 64-\/bit format uses twice more memory compared to the 32-\/bit format. It can also have a significant performance impact on mobile platforms.

To select the 64-\/bit render target option\+: in URP Asset, navigate to {\bfseries{Quality}} \texorpdfstring{$>$}{>} {\bfseries{HDR}} \texorpdfstring{$>$}{>} {\bfseries{HDR Precision}}.

This setting controls only the internal HDR rendering precision, not HDR output.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2686}{}\doxysubsection{\texorpdfstring{New bloom quality settings}{New bloom quality settings}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2686}
URP 14 adds two new properties to the Bloom post-\/processing effect\+:


\begin{DoxyItemize}
\item {\bfseries{Downscale}}\+: set the bloom texture scale to either half size or quarter size.
\item {\bfseries{Max Iterations}}\+: set the maximum number of scale iterations (down and up) the bloom effect does. This property replaces the {\bfseries{Skip Iterations}} property, which skipped a number of last iterations, but did not limit the maximum number.
\end{DoxyItemize}

All the calls to {\ttfamily cmd.\+Blit} method are replaced with the \href{https://docs.unity3d.com/Packages/com.unity.render-pipelines.core@14.0/api/UnityEngine.Rendering.Blitter.html}{\texttt{ Blitter}} API. This ensures a correct and consistent way to perform full screen draws.

In the current URP version, using {\ttfamily cmd.\+Blit} might implicitly enable or disable XR shader keywords, which breaks XR SPI rendering.

Refer to the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_doeb4502541019934c65c2b374e29ea936}{Perform a full screen blit in URP}{0} page to read how to use the Blitter API.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2689}{}\doxysubsection{\texorpdfstring{More consistent lighting behavior on different platforms}{More consistent lighting behavior on different platforms}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2689}
Removed implicit mobile shader optimizations to keep the visual appearance of lighting consistent on all platforms. Changes are mostly related to light fade quality and shadow filtering. Light fade calculations on mobile platforms are now the same as on desktop platforms.

In this release, URP implements full quality spherical harmonics and always normalized normals for lighting.

{\ttfamily SHADER\+\_\+\+QUALITY\+\_\+\+LOW/\+MEDIUM/\+HIGH} and {\ttfamily SHADER\+\_\+\+HINT\+\_\+\+NICE\+\_\+\+QUALITY} shader defines were removed. If you used those defines in custom shaders, consider using {\ttfamily SHADER\+\_\+\+API\+\_\+\+MOBILE} or {\ttfamily SHADER\+\_\+\+API\+\_\+\+GLES} defines to replace {\ttfamily SHADER\+\_\+\+QUALITY\+\_\+\+LOW/\+MEDIUM/\+HIGH}.

URP uses the {\ttfamily SHADER\+\_\+\+API\+\_\+\+MOBILE} define only for platform-\/specific functions now, and not for implicit quality changes.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2690}{}\doxysubsection{\texorpdfstring{XRSystem API}{XRSystem API}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2690}
URP can now use the \href{https://docs.unity3d.com/Packages/com.unity.render-pipelines.core@14.0/api/UnityEngine.Experimental.Rendering.XRBuiltinShaderConstants.html}{\texttt{ XRSystem API}} override the Built-\/\+In Render Pipeline stereo matrices. This lets you inject modifications to the projection matrix and the view matrix in URP.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2691}{}\doxysubsection{\texorpdfstring{Shader stripping improvement\+: Light cookie stripping}{Shader stripping improvement\+: Light cookie stripping}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2691}
The URP Asset now has the option {\bfseries{Lighting}} \texorpdfstring{$>$}{>} {\bfseries{Light Cookies}} that lets you enable or disable Light Cookies. When Light Cookies are disabled in all URP Assets in {\bfseries{\doxylink{namespace_graphics}{Graphics}}} and {\bfseries{Quality}} settings, \doxylink{namespace_unity}{Unity} strips all shader variants with Light Cookies. This can reduce the shader variant count up to two times.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2692}{}\doxysubsection{\texorpdfstring{CPU performance improvements for light-\/heavy scenes}{CPU performance improvements for light-\/heavy scenes}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2692}
This release contains optimizations to avoid unnecessary copies of Light and Camera data. The changes mostly affect scenes containing tens or hundreds of Lights with Light cookies, with shadows enabled, and when using the Deferred Rendering Path. In scenes with hundreds of Lights that use all the mentioned features, the rendering performance can be up to 25\% faster in the Editor.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2693}{}\doxysubsection{\texorpdfstring{Soft Shadows Quality property on Lights}{Soft Shadows Quality property on Lights}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do56dbbc7b1f345a4e8fcbf1733ca30623_autotoc_md2693}
