Before\+Render&Input update that happens right before rendering. REVIEW\+: Axe this update type from the public API? 

The Before\+Render update affects only devices that have before-\/render updates enabled. This has to be done through a device\textquotesingle{}s layout (Input\+Control\+Layout.\+update\+Before\+Render and is visible through \doxylink{class_unity_engine_1_1_input_system_1_1_input_device_a8582d897119e1524b4f695447afbc6c1}{Input\+Device.\+update\+Before\+Render}.

Before\+Render updates are useful to minimize lag of transform data that is used in rendering but is coming from external tracking devices. An example are HMDs. If the head transform used for the render camera is not synchronized right before rendering, it can result in a noticeable lag between head and camera movement. \\
