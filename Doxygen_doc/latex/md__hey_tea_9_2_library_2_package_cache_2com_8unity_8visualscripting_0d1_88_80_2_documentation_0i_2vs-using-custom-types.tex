 Visual Scripting supports every class and struct type available in \doxylink{namespace_unity}{Unity}. By default, the most common are available in \href{vs-interface-overview.md\#the-fuzzy-finder}{\texttt{ the fuzzy finder}}. Add additional \doxylink{namespace_unity}{Unity} assemblies, such as custom types and classes, through your \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-configuration}{project settings}{0}.

You must write some additional code to use a custom type or class in a graph. You can\textquotesingle{}t assign a value to a variable with a custom type from \doxylink{namespace_unity}{Unity}\textquotesingle{}s \href{https://docs.unity3d.com/Manual/UsingTheInspector.html}{\texttt{ Inspector window}}, or initialize it from inside the \doxylink{namespace_unity}{Unity} Editor if this additional code isn\textquotesingle{}t available. You must assign a default value for a custom type through the Inspector window to use that type for a variable in Visual Scripting.

You have two options to enable variable assignment and initialization\+:


\begin{DoxyItemize}
\item If you have access to the source code, you can add the {\ttfamily \mbox{[}Inspectable\mbox{]}} attribute to the classes and fields that you want to display and modify in the Editor.
\item If you don\textquotesingle{}t have access to the source code, you must create a custom Property\+Drawer and generate the required property provider scripts.
\end{DoxyItemize}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-using-custom-types_autotoc_md5618}{}\doxysection{\texorpdfstring{Add the \mbox{[}Inspectable\mbox{]} attribute}{Add the \mbox{[}Inspectable\mbox{]} attribute}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-using-custom-types_autotoc_md5618}
Add the {\ttfamily \mbox{[}Inspectable\mbox{]}} attribute to the code for your custom class to display its available properties in the Inspector window and Visual Scripting\textquotesingle{}s \href{vs-interface-overview.md\#the-graph-inspector}{\texttt{ Graph Inspector}}. You can\textquotesingle{}t view your classes and fields in the Inspector window without the {\ttfamily \mbox{[}Inspectable\mbox{]}} attribute.

\doxylink{namespace_unity}{Unity} provides a basic UI for your types in the Inspector window, which might not give the aesthetic results you want. If you or your users want to configure a property for a custom type with a slider, for example, don\textquotesingle{}t use the {\ttfamily \mbox{[}Inspectable\mbox{]}} attribute method. For more information on how to add the {\ttfamily \mbox{[}Inspectable\mbox{]}} attribute to a custom class, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_02c7eb7fa04d5782bd90a1af21b9dcfff}{Add the Inspectable attribute to the source code for a custom type}{0}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-using-custom-types_autotoc_md5619}{}\doxysection{\texorpdfstring{Create a custom Property\+Drawer}{Create a custom Property\+Drawer}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-using-custom-types_autotoc_md5619}
Create a custom Property\+Drawer to choose how to display each property for a custom class in the Inspector window.

Without access to the source code, you must create a Property\+Drawer to interact with custom-\/typed variables in Visual Scripting. If you see an error in the \doxylink{namespace_unity}{Unity} Editor\textquotesingle{}s Inspector window when you try to use a type from a third-\/party package, you must create a custom Property\+Drawer.

\begin{DoxyNote}{Note}
If you are a package developer, or plan to provide your custom classes and types to other users and want those types to be available in Visual Scripting, create a custom Property\+Drawer to get the best results for your users.
\end{DoxyNote}
For more information on how to create a custom Property\+Drawer, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-custom-drawer}{Create a custom Property\+Drawer for a custom type}{0}.

After you create a custom Property\+Drawer for a custom type, you must generate the necessary property provider scripts. For more information, see the {\bfseries{Generate}} option in \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-configuration}{Configure project settings}{0}.
