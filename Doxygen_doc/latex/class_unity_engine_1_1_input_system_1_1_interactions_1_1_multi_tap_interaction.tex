\doxysubsection{Detailed Description}
Interaction that requires multiple taps (press and release within \doxylink{class_unity_engine_1_1_input_system_1_1_interactions_1_1_multi_tap_interaction_aeb2492700f47491c6fe1f89387461dcc}{tap\+Time}) spaced no more than \doxylink{class_unity_engine_1_1_input_system_1_1_interactions_1_1_multi_tap_interaction_a700dc1a3c79ee8b21e73270ca64980d3}{tap\+Delay} seconds apart. This equates to a chain of \doxylink{class_unity_engine_1_1_input_system_1_1_interactions_1_1_tap_interaction}{Tap\+Interaction} with a maximum delay between each tap. 

REVIEW\+: Why is this deriving from \doxylink{interface_unity_engine_1_1_input_system_1_1_i_input_interaction}{IInput\+Interaction$<$float$>$}? It goes by actuation just like Hold etc.

The interaction goes into \doxylink{namespace_unity_engine_1_1_input_system_aecc012480a29c4b720a87b8465f34182a8428552d86c0d262a542a528af490afa}{Input\+Action\+Phase.\+Started} on the first press and then will not trigger again until either the full tap sequence is performed (in which case the interaction triggers \doxylink{namespace_unity_engine_1_1_input_system_aecc012480a29c4b720a87b8465f34182a916b54ec8bd2d9be03994b6486310ed3}{Input\+Action\+Phase.\+Performed}) or the multi-\/tap is aborted by a timeout being hit (in which case the interaction will trigger \doxylink{namespace_unity_engine_1_1_input_system_aecc012480a29c4b720a87b8465f34182a0e22fe7d45f8e5632a4abf369b24e29c}{Input\+Action\+Phase.\+Canceled}).
