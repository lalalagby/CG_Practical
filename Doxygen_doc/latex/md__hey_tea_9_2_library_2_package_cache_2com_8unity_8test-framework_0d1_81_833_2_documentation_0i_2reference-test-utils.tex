{\ttfamily bool Are\+Floats\+Equal(float expected, float actual, float allowed\+Relative\+Error)}   &Relative epsilon comparison of two float values for equality. {\ttfamily allowed\+Relative\+Error} is the relative error to be used in relative epsilon comparison. The relative error is the absolute error divided by the magnitude of the exact value. Returns {\ttfamily true} if the actual value is equivalent to the expected value.    \\\cline{1-2}
{\ttfamily bool Are\+Floats\+Equal\+Absolute\+Error(float expected, float actual, float allowed\+Absolute\+Error)}   &Compares two floating point numbers for equality under the given absolute tolerance. {\ttfamily allowed\+Absolute\+Error} is the permitted error tolerance. Returns {\ttfamily true} if the actual value is equivalent to the expected value under the given tolerance.    \\\cline{1-2}
{\ttfamily Game\+Object Create\+Primitive( type)}   &Creates a \href{https://docs.unity3d.com/ScriptReference/GameObject.html}{\texttt{ Game\+Object}} with a primitive \href{https://docs.unity3d.com/ScriptReference/MeshRenderer.html}{\texttt{ Mesh\+Renderer}}. This is an analogue to the \href{https://docs.unity3d.com/ScriptReference/GameObject.CreatePrimitive.html}{\texttt{ Game\+Object.\+Create\+Primitive}}, but creates a primitive {\ttfamily Mesh\+Renderer} with a fast \href{https://docs.unity3d.com/ScriptReference/Shader.html}{\texttt{ Shader}} instead of the default built-\/in {\ttfamily Shader}, optimized for testing performance. {\ttfamily type} is the \href{https://docs.unity3d.com/ScriptReference/PrimitiveType.html}{\texttt{ primitive type}} of the required {\ttfamily Game\+Object}. Returns a {\ttfamily Game\+Object} with primitive {\ttfamily Mesh\+Renderer} and \href{https://docs.unity3d.com/ScriptReference/Collider.html}{\texttt{ Collider}}.   \\\cline{1-2}
