 In the Universal Render Pipeline (URP), multiple \href{camera-types-and-render-type.md\#base-camera}{\texttt{ Base Cameras}} or \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_documentation_0i_2camera-stacking}{Camera Stacks}{0} can render to the same render target. This allows you to create effects such as split screen rendering.

If more than one Base Camera or Camera Stack renders to the same area of a render target, \doxylink{namespace_unity}{Unity} draws each pixel in the overlapping area multiple times. \doxylink{namespace_unity}{Unity} draws the Base Camera or Camera Stack with the highest priority last, on top of the previously drawn pixels. For more information on overdraw, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do624362eba0d84f67f05d1918f83d229f}{Advanced information}{0}.

You use the Base Camera\textquotesingle{}s {\bfseries{Output Target}} property to define the render target, and the {\bfseries{Viewport Rect}} property to define the area of the render target to render to. For more information on viewport coordinates, see the \href{https://docs.unity3d.com/Manual/class-Camera.html}{\texttt{ Unity Manual}} and \href{https://docs.unity3d.com/ScriptReference/Camera-rect.html}{\texttt{ API documentation}}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do86ff61978b71fc44fa3b67096696d1c1_autotoc_md2428}{}\doxysection{\texorpdfstring{Setting up split screen rendering}{Setting up split screen rendering}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do86ff61978b71fc44fa3b67096696d1c1_autotoc_md2428}



\begin{DoxyEnumerate}
\item Create a Camera in your Scene. Its {\bfseries{Render Mode}} defaults to {\bfseries{Base}}, making it a Base Camera.
\item Select the Camera. In the Inspector, scroll to the Output section. Ensure that {\bfseries{Output Target}} is set to {\bfseries{Camera}}, and change the values for {\itshape Viewport rect} to the following\+:
\begin{DoxyItemize}
\item X\+: 0
\item Y\+: 0
\item W\+: 0.\+5
\item H\+: 1
\end{DoxyItemize}
\item Create another Camera in your Scene. Its {\bfseries{Render Mode}} defaults to {\bfseries{Base}}, making it a Base Camera.
\item Select the Camera. In the Inspector, scroll to the Output section. Ensure that {\bfseries{Output Target}} is set to {\bfseries{Camera}}, and change the values for {\itshape Viewport rect} to the following\+:
\begin{DoxyItemize}
\item X\+: 0.\+5
\item Y\+: 0
\item W\+: 0.\+5
\item H\+: 1
\end{DoxyItemize}
\end{DoxyEnumerate}

\doxylink{namespace_unity}{Unity} renders the first Camera to the left-\/hand side of the screen, and the second Camera to the right-\/hand side of the screen.

You can change the Viewport rect for a Camera in a script by setting its {\ttfamily rect} property, like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{myUniversalAdditionalCameraData.rect\ =\ new\ Rect(0.5f,\ 0f,\ 0.5f,\ 0f);}

\end{DoxyCode}
