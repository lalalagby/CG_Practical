\doxysubsection{Detailed Description}
Base class for on-\/screen controls. 

The set of on-\/screen controls together forms a device. A control layout is automatically generated from the set and a device using the layout is added to the system when the on-\/screen controls are enabled.

The layout that the generated layout is based on is determined by the control paths chosen for each on-\/screen control. If, for example, an on-\/screen control chooses the \textquotesingle{}a\textquotesingle{} key from the "{}\+Keyboard"{} layout as its path, a device layout is generated that is based on the "{}\+Keyboard"{} layout and the on-\/screen control becomes the \textquotesingle{}a\textquotesingle{} key in that layout.

If a Game\+Object has multiple on-\/screen controls that reference different types of device layouts (e.\+g. one control references \textquotesingle{}button\+West\textquotesingle{} on a gamepad and another references \textquotesingle{}left\+Button\textquotesingle{} on a mouse), then a device is created for each type referenced by the setup.

string Unity\+Engine.\+Input\+System.\+On\+Screen.\+On\+Screen\+Control.\+control\+Path\hspace{0.3cm}{\ttfamily [get]}, {\ttfamily [set]}}



The control path (see Input\+Control\+Path) for the control that the on-\/screen control will feed input into. 

A device will be created from the device layout referenced by the control path (see Input\+Control\+Path.\+Try\+Get\+Device\+Layout). The path is then used to look up \doxylink{class_unity_engine_1_1_input_system_1_1_on_screen_1_1_on_screen_control_a430da0e4fe868e03ff198a985c46cb07}{control} on the device. The resulting control will be fed values from the on-\/screen control.

Multiple on-\/screen controls sharing the same device layout will together create a single virtual device. If, for example, one component uses {\ttfamily "{}<\+Gamepad>/button\+South"{}} and another uses {\ttfamily "{}<\+Gamepad>/left\+Stick"{}} as the control path, a single \doxylink{class_unity_engine_1_1_input_system_1_1_gamepad}{Gamepad} will be created and the first component will feed data to \doxylink{class_unity_engine_1_1_input_system_1_1_gamepad_a55b57fe1d0a61c4fe2e671284ec2ab3b}{Gamepad.\+button\+South} and the second component will feed data to \doxylink{class_unity_engine_1_1_input_system_1_1_gamepad_a9f608a9e12af3301fb2d704e072ea024}{Gamepad.\+left\+Stick}.
