\item Has State Graph

The Set State Graph node has one required input parameter, called {\bfseries{Container Type}}, which is set using the dropdown in the node\textquotesingle{}s header. The {\bfseries{Container Type}} specifies what component type the node should expect as an input for its {\bfseries{Target}}\+:


\begin{DoxyItemize}
\item If you choose {\bfseries{Game\+Object}}, the node expects to receive a Game\+Object, and assigns the graph to the first State Machine attached to that Game\+Object.
\item If you choose {\bfseries{State Machine}}, the node expects to receive a State Machine, and you can specify the exact State Machine where you want to set your State Graph.
\end{DoxyItemize}

The node has three input ports, located on the left side. The first port, {\bfseries{Enter}}, connects to the node that should start the execution of the Set State Graph node.

The other two ports collect the Set State Graph node\textquotesingle{}s required input data\+:


\begin{DoxyItemize}
\item The {\bfseries{Target}}, or the Game\+Object or State Machine where you want to set a State Graph.
\item The {\bfseries{Graph}}, or the State Graph to assign to the Game\+Object or State Machine.
\end{DoxyItemize}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-states-reference_autotoc_md5586}{}\doxysubsection{\texorpdfstring{Outputs}{Outputs}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-states-reference_autotoc_md5586}
The Set State Graph node has two output ports, located on the right side.

The first port, {\bfseries{Exit}}, establishes the connection to the node that should execute after the Set State Graph node has finished.

The Has State Graph node returns {\ttfamily true} if it finds the specified State Graph. Otherwise, it returns {\ttfamily false}.

You can use a control node connected to the Has State Graph\textquotesingle{}s output port to change what your script does next, based on the result from Has State Graph. The Get State Graph node is a data node. It can\textquotesingle{}t control any logic in your script, and is only used to return data.

The node has a single input port, located on the left side, which collects the node\textquotesingle{}s required input data\+:


\begin{DoxyItemize}
\item The Game\+Object where the node should retrieve the State Graph. You can choose a specific Game\+Object, or leave the default selection as {\bfseries{This}} to use the Game\+Object where your script is currently running.
\end{DoxyItemize}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-states-reference_autotoc_md5592}{}\doxysubsection{\texorpdfstring{Outputs}{Outputs}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-states-reference_autotoc_md5592}
The Get State Graph node has a single output port, located on the right side. The output port returns the Game\+Object\textquotesingle{}s first set State Graph, or {\ttfamily null}, if there is no set State Graph.

\begin{DoxyNote}{Note}
The Get State Graph node returns only the first State Graph set on a Game\+Object. To return all State Graphs set on a Game\+Object, use the Get State Graphs node.
\end{DoxyNote}

The Get State Graphs node has a single output port, located on the right side. The output port returns a list of all set State Graphs for the Game\+Object, or an empty list, if there are no set State Graphs.
