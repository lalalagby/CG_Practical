\doxysubsection{Detailed Description}
A mechanism for collecting a series of input actions (see \doxylink{class_unity_engine_1_1_input_system_1_1_input_action}{Input\+Action}) and treating them as a group. 

Each action map is a named collection of bindings and actions. Both are stored as a flat list. The bindings are available through the \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_map_ae98277c3697a798a28f8b16e187217ff}{bindings} property and the actions are available through the \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_map_ab6ee4af8fc528a22edda96efd0e77a62}{actions} property.

The actions in a map are owned by the map. No action can appear in two maps at the same time. To find the action map an action belongs to, use the \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_ae6817835d32520aa1433e7119c6e30e8}{Input\+Action.\+action\+Map} property. Note that actions can also stand on their own and thus do not necessarily need to belong to a map (in which case the \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_ae6817835d32520aa1433e7119c6e30e8}{Input\+Action.\+action\+Map} property is {\ttfamily null}).

Within a map, all actions have to have names and each action name must be unique. The \doxylink{struct_unity_engine_1_1_input_system_1_1_input_binding_a83005305665b72fa6dadd5a105613495}{Input\+Binding.\+action} property of bindings in a map are resolved within the \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_map_ab6ee4af8fc528a22edda96efd0e77a62}{actions} in the map. Looking up actions by name can be done through \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_map_af63d7ef83f7b7841fe6456da0e4e89f2}{Find\+Action(string,bool)}.

The \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_map_a6340c011493841e2c380fa8f70174378}{name} of the map itself can be empty, except if the map is part of an \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_asset}{Input\+Action\+Asset} in which case it is required to have a name which also must be unique within the asset.

Action maps are most useful for grouping actions that contextually belong together. For example, one common usage is to separate the actions that can be performed in the \doxylink{namespace_unity_engine_1_1_u_i}{UI} or in the main menu from those that can be performed during gameplay. However, even within gameplay, multiple action maps can be employed. For example, one could have different action maps for driving and for walking plus one more map for the actions shared between the two modes.

Action maps are usually created in the \href{../manual/ActionAssets.html}{\texttt{ action editor}} as part of \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_asset}{Input\+Action\+Asset}s. However, they can also be created standing on their own directly in code or from JSON (see \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_map_a7a724c1e16864a4f675fcf89732fcd6c}{From\+Json}).

Actions in action maps, like actions existing by themselves outside of action maps, do not actively process input except if enabled. Actions can either be enabled individually (see \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_abc6ff7728cbf947406b3c426cf82437f}{Input\+Action.\+Enable} and \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_a301a6f1f456e71eee25363f62861ff93}{Input\+Action.\+Disable}) or in bulk by enabling and disabling the entire map (see \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_map_a1070fcb5a4631ca6c0212fb28a19644f}{Enable} and \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_map_a215d9741eaf7205b3ad4ac865efae047}{Disable}).

Binding mask to apply to all actions in the asset. 

Optional mask that determines which bindings in the action map to enable.

Binding masks can be applied at three different levels\+: for an entire asset through \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_asset_a7cf65e9b19155bd044d9d06250e61225}{Input\+Action\+Asset.\+binding\+Mask}, for a specific map through this property, and for single actions through \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_a41caa3a44bc876781c5b1bcb04622cd2}{Input\+Action.\+binding\+Mask}. By default, none of the masks will be set (that is, they will be {\ttfamily null}).

When an action is enabled, all the binding masks that apply to it are taken into account. Specifically, this means that any given binding on the action will be enabled only if it matches the mask applied to the asset, the mask applied to the map that contains the action, and the mask applied to the action itself. All the masks are individually optional.

Masks are matched against bindings using \doxylink{struct_unity_engine_1_1_input_system_1_1_input_binding_aa69fd73f446257c84723c95ad8fbcbe8}{Input\+Binding.\+Matches}.

Note that if you modify the masks applicable to an action while it is enabled, the action\textquotesingle{}s \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_a8e3f5d3329be13ca8474314137045af4}{Input\+Action.\+controls} will get updated immediately to respect the mask. To avoid repeated binding resolution, it is most efficient to apply binding masks before enabling actions.

Binding masks are non-\/destructive. All the bindings on the action are left in place. Setting a mask will not affect the value of the \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_a44fdaf3ff9ffc22609a9df942372d9bb}{Input\+Action.\+bindings} and \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_map_ae98277c3697a798a28f8b16e187217ff}{bindings} properties.
