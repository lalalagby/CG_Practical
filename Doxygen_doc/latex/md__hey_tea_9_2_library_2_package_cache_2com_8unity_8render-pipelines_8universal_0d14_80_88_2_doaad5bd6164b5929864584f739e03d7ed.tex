 \doxylink{namespace_unity}{Unity} has several preset levels of \href{https://docs.unity3d.com/Manual/class-QualitySettings.html}{\texttt{ Quality settings}} and you might add more to your project. To accommodate different hardware specifications, you can switch between these levels and the associated URP Asset from C\# scripts. The following examples show how to use API to change Quality setting levels and the active URP Asset, and how to change specific settings in the URP Asset at runtime.

{\bfseries{Note}}\+: You should only change Quality settings and URP Asset settings at runtime at points where performance is not essential, such as during loading screens or on static menus. This is because these changes cause a temporary but significant performance impact.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_doaad5bd6164b5929864584f739e03d7ed_autotoc_md2374}{}\doxysection{\texorpdfstring{Change URP Asset at runtime}{Change URP Asset at runtime}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_doaad5bd6164b5929864584f739e03d7ed_autotoc_md2374}
Each quality level uses a URP Asset to control many of the specific graphics settings. You can assigning different URP Assets to each quality level and switch between them at runtime.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_doaad5bd6164b5929864584f739e03d7ed_autotoc_md2375}{}\doxysubsection{\texorpdfstring{Configure Project Quality settings}{Configure Project Quality settings}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_doaad5bd6164b5929864584f739e03d7ed_autotoc_md2375}
To use Quality settings to switch between URP Assets, ensure that the quality levels of your project are configured to use different URP Assets. The URP 3D Sample scene has this configuration by default.


\begin{DoxyEnumerate}
\item Create a URP Asset for each quality level. To do this, right-\/click in the Project window and select {\bfseries{Create}} \texorpdfstring{$>$}{>} {\bfseries{Rendering}} \texorpdfstring{$>$}{>} {\bfseries{URP Asset (with Universal Renderer)}}.
\end{DoxyEnumerate}

\begin{quote}
{\bfseries{Note}}\+: These instructions are also valid for URP Assets that use the 2D Renderer. \end{quote}



\begin{DoxyEnumerate}
\item Configure and name the new URP Assets as necessary.
\item Open the Quality section in the Project Settings ({\bfseries{Edit}} \texorpdfstring{$>$}{>} {\bfseries{Project Settings}} \texorpdfstring{$>$}{>} {\bfseries{Quality}}).
\item Assign each URP Asset to a quality level. To do this, select a quality level from the Levels list, then go to {\bfseries{Rendering}} \texorpdfstring{$>$}{>} {\bfseries{Render Pipeline Asset}} and choose the URP Asset you created for this quality level. Do this for each quality level.
\end{DoxyEnumerate}

The quality levels of your project are now ready to be used to change between URP Assets at runtime.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_doaad5bd6164b5929864584f739e03d7ed_autotoc_md2376}{}\doxysubsection{\texorpdfstring{Change Quality Level}{Change Quality Level}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_doaad5bd6164b5929864584f739e03d7ed_autotoc_md2376}
You can change the quality level \doxylink{namespace_unity}{Unity} uses at runtime through the \href{https://docs.unity3d.com/ScriptReference/QualitySettings.html}{\texttt{ Quality\+Settings API}}. With the quality levels setup as shown previously, this enables you to switch between URP Assets as well as Quality settings presets.

In the following simple example, the C\# script uses the system\textquotesingle{}s total \doxylink{namespace_graphics}{Graphics} Memory to determine the appropriate quality level without any input from the user when they open the built project.


\begin{DoxyEnumerate}
\item Create a new C\# script with the name Quality\+Controls.
\item Open the Quality\+Controls script and add the {\ttfamily Switch\+Quality\+Level} method to the {\ttfamily Quality\+Controls} class.

Now when this scene loads, \doxylink{namespace_unity}{Unity} runs the {\ttfamily Switch\+Quality\+Level} method in the Quality\+Controls script which detects the system\textquotesingle{}s total graphics memory and sets the quality level. The quality level sets the URP Asset as the active Render Pipeline Asset.

You can create more complex systems and sequences of checks to determine which quality level to use, but the fundamental process remains the same. When the project starts, run a script which uses \href{https://docs.unity3d.com/ScriptReference/QualitySettings.SetQualityLevel.html}{\texttt{ {\ttfamily Quality\+Settings.\+Set\+Quality\+Level}}} to select a quality level and through that select the URP Asset for the project to use at runtime.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_doaad5bd6164b5929864584f739e03d7ed_autotoc_md2377}{}\doxysection{\texorpdfstring{Change URP Asset settings}{Change URP Asset settings}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_doaad5bd6164b5929864584f739e03d7ed_autotoc_md2377}
You can change some properties of the URP Asset at runtime with C\# scripts. This can help fine tune performance on devices with hardware that doesn\textquotesingle{}t perfectly match any of the quality levels in your project.

\begin{quote}
{\bfseries{Note}}\+: To change a property of the URP Asset with a C\# script, the property must have a {\ttfamily set} method. For more information on these properties see Accessible Properties. \end{quote}
