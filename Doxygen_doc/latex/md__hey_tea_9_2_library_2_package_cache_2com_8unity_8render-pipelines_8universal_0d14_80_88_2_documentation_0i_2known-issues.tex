When importing the URP package samples, \doxylink{namespace_unity}{Unity} does not set the necessary URP asset in {\bfseries{Quality}} \texorpdfstring{$>$}{>} {\bfseries{Render Pipeline Asset}}, and certain sample rendering effects do not work.

To fix this issue\+:

In {\bfseries{Project Settings}} \texorpdfstring{$>$}{>} {\bfseries{Quality}} \texorpdfstring{$>$}{>} {\bfseries{Render Pipeline Asset}}, select {\ttfamily Samples\+Pipeline\+Asset}.

\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_documentation_0i_2known-issues_autotoc_md2185}{}\doxysection{\texorpdfstring{Renaming a URP Renderer asset to a name matching one of the Renderer Feature names causes erroneous behavior}{Renaming a URP Renderer asset to a name matching one of the Renderer Feature names causes erroneous behavior}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_documentation_0i_2known-issues_autotoc_md2185}
If a URP Renderer asset has any Renderer Features assigned, renaming the Renderer asset to a name matching one of the Renderer Feature names causes erroneous behavior\+: the URP Renderer and the Renderer Feature switch places.

The following scenario shows how the error occurs\+:


\begin{DoxyItemize}
\item Let\textquotesingle{}s assume that the URP Renderer in your project is called {\ttfamily Universal\+Renderer}.
\item The Renderer has a Renderer Feature called {\ttfamily New\+Render\+Objects} assigned.


\item Renaming {\ttfamily Universal\+Renderer} to {\ttfamily New\+Render\+Objects} causes erroneous behavior\+:~\newline
The Renderer switches places with the Renderer Feature and does not behave correctly.
\end{DoxyItemize}

To avoid the issue, do not give the URP Renderer asset the same name as the Renderer Feature asset.

To see updates on this issue, refer to the \href{https://issuetracker.unity3d.com/issues/parent-and-child-nested-scriptable-object-assets-switch-places-when-parent-scriptable-object-asset-is-renamed}{\texttt{ Unity Issue Tracker}}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_documentation_0i_2known-issues_autotoc_md2186}{}\doxysection{\texorpdfstring{Warning about \+\_\+\+Additional\+Lights property when upgrading the URP package}{Warning about \+\_\+\+Additional\+Lights property when upgrading the URP package}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_documentation_0i_2known-issues_autotoc_md2186}
In certain cases, you might see the following warning when upgrading the URP package to a newer version\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{Property\ (\_AdditionalLights<...>)\ exceeds\ previous\ array\ size\ (256\ vs\ 16).\ Cap\ to\ previous\ size.}

\end{DoxyCode}


This warning does not cause issues with the project, the warning disappears if you restart the Editor.
