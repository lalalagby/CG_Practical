 This page describes common compilation warnings, and how to fix them.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8burst_0d1_88_87_2_documentation_0i_2compilation-warnings_autotoc_md309}{}\doxysection{\texorpdfstring{Ignore\+Warning attribute}{Ignore\+Warning attribute}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8burst_0d1_88_87_2_documentation_0i_2compilation-warnings_autotoc_md309}
The \href{xref:Unity.Burst.CompilerServices.IgnoreWarningAttribute}{\texttt{ {\ttfamily Unity.\+Burst.\+Compiler\+Services.\+Ignore\+Warning\+Attribute}}} attribute lets you suppress warnings for a specific function that is being compiled from \doxylink{namespace_burst}{Burst}. However, the warnings that the \doxylink{namespace_burst}{Burst} compiler generates are very important to pay attention to, so this attribute should be used sparingly and only when necessary. The sections below describe the specific situations in which you might want to suppress warnings.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8burst_0d1_88_87_2_documentation_0i_2compilation-warnings_autotoc_md310}{}\doxysection{\texorpdfstring{BC1370}{BC1370}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8burst_0d1_88_87_2_documentation_0i_2compilation-warnings_autotoc_md310}
Warning BC1370 produces the message\+:

\begin{quote}
An exception was thrown from a function without the correct {\ttfamily \mbox{[}Conditional("{}\+ENABLE\+\_\+\+UNITY\+\_\+\+COLLECTIONS\+\_\+\+CHECKS"{})\mbox{]}} guard... \end{quote}


This warning happens if \doxylink{namespace_unity}{Unity} encounters a throw in code that {\ttfamily \mbox{[}Conditional("{}\+ENABLE\+\_\+\+UNITY\+\_\+\+COLLECTIONS\+\_\+\+CHECKS"{})\mbox{]}} doesn\textquotesingle{}t guard. In the Editor, thrown exceptions will be caught and logged to the Console, but in a \doxylink{class_player}{Player} build, a {\ttfamily throw} becomes an abort, which crashes your application. \doxylink{namespace_burst}{Burst} warns you about these exceptions, and advises you to place them in functions guarded with {\ttfamily \mbox{[}Conditional("{}\+ENABLE\+\_\+\+UNITY\+\_\+\+COLLECTIONS\+\_\+\+CHECKS"{})\mbox{]}}, because functions guarded with that attribute will not be included in \doxylink{class_player}{Player} builds. However, if you want to purposely throw an exception to crash your application, use the {\ttfamily Ignore\+Warning\+Attribute} to suppress the warnings that \doxylink{namespace_burst}{Burst} provides on the {\ttfamily throw}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{preprocessor}{\ \#}}
\DoxyCodeLine{[IgnoreWarning(1370)]}
\DoxyCodeLine{\textcolor{keywordtype}{int}\ DoSomethingMaybe(\textcolor{keywordtype}{int}\ x)}
\DoxyCodeLine{\{}
\DoxyCodeLine{\ \ \ \ \textcolor{keywordflow}{if}\ (x\ <\ 0)\ \textcolor{keywordflow}{throw}\ \textcolor{keyword}{new}\ Exception(\textcolor{stringliteral}{"{}Dang\ -\/\ sorry\ I\ crashed\ your\ game!"{}});}
\DoxyCodeLine{}
\DoxyCodeLine{\ \ \ \ \textcolor{keywordflow}{return}\ x\ *\ x;}
\DoxyCodeLine{\}}

\end{DoxyCode}


\begin{DoxyNote}{Note}
This warning is only produced for exceptions that persist into \doxylink{class_player}{Player} builds. Editor-\/only or debug-\/only exception throws that aren\textquotesingle{}t compiled into \doxylink{class_player}{Player} builds will not trigger this warning.
\end{DoxyNote}

Warning BC1371 produces the message\+:

\begin{quote}
A call to the method \textquotesingle{}xxx\textquotesingle{} has been discarded, due to its use as an argument to a discarded method... \end{quote}


To understand this warning, consider the following example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{preprocessor}{\ \#}}
\DoxyCodeLine{[BurstDiscard]}
\DoxyCodeLine{\textcolor{keyword}{static}\ \textcolor{keywordtype}{void}\ DoSomeManagedStuff(\textcolor{keywordtype}{int}\ x)}
\DoxyCodeLine{\{}
\DoxyCodeLine{\}}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{comment}{//\ A\ function\ that\ computes\ some\ result\ which\ we\ need\ to\ pass\ to\ managed\ code}}
\DoxyCodeLine{\textcolor{keywordtype}{int}\ BurstCompiledCode(\textcolor{keywordtype}{int}\ x,\textcolor{keywordtype}{int}\ y)}
\DoxyCodeLine{\{}
\DoxyCodeLine{\ \ \ \ \textcolor{keywordflow}{return}\ y+2*x;}
\DoxyCodeLine{\}}
\DoxyCodeLine{}
\DoxyCodeLine{[BurstCompile]}
\DoxyCodeLine{\textcolor{keywordtype}{void}\ BurstMethod()}
\DoxyCodeLine{\{}
\DoxyCodeLine{\ \ \ \ var\ myValue\ =\ BurstCompiledCode(1,3);}
\DoxyCodeLine{\ \ \ \ DoSomeManagedStuff(myValue);}
\DoxyCodeLine{\}}

\end{DoxyCode}


When \doxylink{namespace_unity}{Unity} compiles your C\# code in release mode, it optimizes and removes the local variable {\ttfamily my\+Value}. This means that \doxylink{namespace_burst}{Burst} receives something like the following code \+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{preprocessor}{\ \#}}
\DoxyCodeLine{[BurstCompile]}
\DoxyCodeLine{\textcolor{keywordtype}{void}\ BurstedMethod()}
\DoxyCodeLine{\{}
\DoxyCodeLine{\ \ \ \ DoSomeManagedStuff(BurstCompiledCode(1,3));}
\DoxyCodeLine{\}}

\end{DoxyCode}


This makes \doxylink{namespace_burst}{Burst} generate the warning, because \doxylink{namespace_burst}{Burst} discards {\ttfamily Do\+Some\+Managed\+Stuff}, along with the {\ttfamily Burst\+Compiled\+Code} argument. This means that the {\ttfamily Burst\+Compiled\+Code} function is no longer executed, which generates the warning.

If this isn\textquotesingle{}t what you intended then ensure the variable has multiple uses. For example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{preprocessor}{\ \#}}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{keywordtype}{void}\ IsUsed(ref\ \textcolor{keywordtype}{int}\ x)}
\DoxyCodeLine{\{}
\DoxyCodeLine{\ \ \ \ \textcolor{comment}{//\ Dummy\ function\ to\ prevent\ removal}}
\DoxyCodeLine{\}}
\DoxyCodeLine{}
\DoxyCodeLine{[BurstCompile]}
\DoxyCodeLine{\textcolor{keywordtype}{void}\ BurstedMethod()}
\DoxyCodeLine{\{}
\DoxyCodeLine{\ \ \ \ var\ myValue\ =\ BurstCompiledCode(1,3);}
\DoxyCodeLine{\ \ \ \ DoSomeManagedStuff(myValue);}
\DoxyCodeLine{\ \ \ \ IsUsed(ref\ myValue);}
\DoxyCodeLine{\}}

\end{DoxyCode}


Alternatively, if you\textquotesingle{}re happy that the code is being discarded correctly, ignore the warning on the {\ttfamily Bursted\+Method} like so\+:

\DoxyCodeLine{\textcolor{keywordtype}{void}\ BurstMethod()}
\DoxyCodeLine{\{}
\DoxyCodeLine{\ \ \ \ var\ myValue\ =\ BurstCompiledCode(1,3);}
\DoxyCodeLine{\ \ \ \ DoSomeManagedStuff(myValue);}
\DoxyCodeLine{\}}

\end{DoxyCode}
