\doxysubsection{Detailed Description}
\doxylink{class_unity_engine_1_1_input_system_1_1_low_level_1_1_input_event_trace}{Input\+Event\+Trace} lets you record input events for later processing. It also has features for writing traces to disk, for loading them from disk, and for playing back previously recorded traces. 

\doxylink{class_unity_engine_1_1_input_system_1_1_low_level_1_1_input_event_trace}{Input\+Event\+Trace} lets you record input events into a buffer for either a specific device, or for all events received by the input system. This is useful for testing purposes or for replaying recorded input.

Note that event traces {\itshape must} be disposed of (by calling \doxylink{class_unity_engine_1_1_input_system_1_1_low_level_1_1_input_event_trace_a38416148dcc4329ebfc2c537542d86c7}{Dispose}) after use or they will leak memory on the unmanaged (C++) memory heap.

Event traces are serializable such that they can survive domain reloads in the editor.

Event storage in memory may be circular if the event buffer is fixed in size or has reached maximum size and new events start overwriting old events. This method will automatically start with the first event when the given {\itshape current}  event is null. Any subsequent call with then loop over the remaining events until no more events are available.

Note that it is VERY IMPORTANT that the buffer is not modified while iterating over events this way. If this is not ensured, invalid memory accesses may result.

If true, input update boundaries will be recorded as events. By default, this is off. 

Whether frame boundaries should be recorded in the trace.

When recording with this off, all events are written one after the other for as long as the recording is active. This means that when a recording runs over multiple frames, it is no longer possible for the trace to tell which events happened in distinct frames.

By turning this feature on, frame marker events (i.\+e. \doxylink{struct_unity_engine_1_1_input_system_1_1_low_level_1_1_input_event}{Input\+Event} instances with \doxylink{struct_unity_engine_1_1_input_system_1_1_low_level_1_1_input_event_a40a35f018657b0603e16a6210b8d5a19}{Input\+Event.\+type} set to \doxylink{class_unity_engine_1_1_input_system_1_1_low_level_1_1_input_event_trace_a3d6520b2e98ba03ccac0fd388e59ecee}{Frame\+Marker\+Event}) will be written to the trace every time an input update occurs.
