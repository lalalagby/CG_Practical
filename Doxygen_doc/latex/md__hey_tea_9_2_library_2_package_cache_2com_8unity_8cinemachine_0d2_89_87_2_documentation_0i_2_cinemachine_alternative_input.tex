 Some \doxylink{namespace_cinemachine}{Cinemachine} components (e.\+g. Free\+Look, POV, Orbital\+Transposer) require user input to position or orient the camera. By default, \doxylink{namespace_cinemachine}{Cinemachine} gets user input by querying the standard \doxylink{namespace_unity}{Unity} {\ttfamily Input.\+Get\+Axis(name)}. ~\newline


When alternative input systems are used in a project, this default behaviour must be overridden so that input is obtained from the appropriate source.

\doxylink{namespace_cinemachine}{Cinemachine} has defined an interface\+: {\ttfamily \doxylink{interface_cinemachine_1_1_axis_state_1_1_i_input_axis_provider}{Cinemachine.\+Axis\+State.\+IInput\+Axis\+Provider}}. If a {\ttfamily Mono\+Behaviour} implementing this interface is added to a Virtual Camera or Free\+Look, then it will be queried for input instead of the standard input system.

\doxylink{namespace_cinemachine}{Cinemachine} ships with an example of such a behaviour that uses the new {\bfseries{Unity\+Engine.\+Input}} package\+: {\bfseries{Cinemachine\+Input\+Provider}}. It has dual purpose\+:
\begin{DoxyEnumerate}
\item Provide a simple out-\/of-\/the-\/box adapter for the new Unity\+Engine.\+Input package, to cover common usecases
\item To serve as a source code example of how to integrate \doxylink{namespace_cinemachine}{Cinemachine} with custom or 3rd-\/party input systems
\end{DoxyEnumerate}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8cinemachine_0d2_89_87_2_documentation_0i_2_cinemachine_alternative_input_autotoc_md498}{}\doxysection{\texorpdfstring{Cinemachine\+Input\+Provider}{Cinemachine\+Input\+Provider}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8cinemachine_0d2_89_87_2_documentation_0i_2_cinemachine_alternative_input_autotoc_md498}


This is a behaviour that is intended to be added to a virtual camera or Free\+Look. It is an input source override, which causes the virtual camera to obtain input using the new {\bfseries{Unity\+Engine.\+Input}} package instead of the standard \doxylink{namespace_unity}{Unity} Input system.

This behaviour is available when the {\bfseries{Unity\+Engine.\+Input}} package is installed in the project.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8cinemachine_0d2_89_87_2_documentation_0i_2_cinemachine_alternative_input_autotoc_md499}{}\doxysubsection{\texorpdfstring{Properties\+:}{Properties\+:}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8cinemachine_0d2_89_87_2_documentation_0i_2_cinemachine_alternative_input_autotoc_md499}
