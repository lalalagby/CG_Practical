 If you don\textquotesingle{}t use a version control system, such as \href{https://docs.unity3d.com/Manual/UnityCollaborate.html}{\texttt{ Unity Collaborate}}, \href{https://git-scm.com/}{\texttt{ Git}}, or \href{https://subversion.apache.org/}{\texttt{ Subversion}}, it\textquotesingle{}s a good practice to create backups of your Visual Scripting assets and settings. Create a backup at any time from your \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-configuration}{Project Settings}{0}.

Back up your data before you update Visual Scripting to a new version. For more information on the update process, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-update}{Update Visual Scripting}{0}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-restore-backups_autotoc_md5291}{}\doxysection{\texorpdfstring{Create a new backup}{Create a new backup}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-restore-backups_autotoc_md5291}
To create a new backup of your Visual Scripting assets and settings\+:


\begin{DoxyEnumerate}
\item \mbox{[}!include\doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0de63b7cc6f4d7ec04359954e9b889ab1}{open-\/project-\/settings}{0}\mbox{]}

\begin{DoxyEnumerate}

Visual Scripting creates a .zip file, with a name in the format {\ttfamily Assets\+\_\+\+YYYY\+\_\+\+MM\+\_\+\+DD\+\_\+\+HH\+\_\+\+MM\+\_\+\+SS}, in a {\ttfamily Backups} folder inside the \doxylink{namespace_unity}{Unity} Project.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-restore-backups_autotoc_md5292}{}\doxysection{\texorpdfstring{Restore an existing backup}{Restore an existing backup}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-create-restore-backups_autotoc_md5292}
To restore an existing backup of your Visual Scripting assets and settings\+:


\begin{DoxyEnumerate}
\item \mbox{[}!include\doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0de63b7cc6f4d7ec04359954e9b889ab1}{open-\/project-\/settings}{0}\mbox{]}
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Select {\bfseries{Visual Scripting}}.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Select {\bfseries{Restore Backup}}.
\end{DoxyEnumerate}

Visual Scripting opens your {\ttfamily Backups} folder in your system\textquotesingle{}s file explorer. You can extract a .zip back-\/up file and import graphs and settings back into \doxylink{namespace_unity}{Unity}.

For more information on how to import assets into \doxylink{namespace_unity}{Unity}, see \href{https://docs.unity3d.com/Manual/ImportingAssets.html}{\texttt{ Importing assets}} in the \doxylink{namespace_unity}{Unity} User Manual.
