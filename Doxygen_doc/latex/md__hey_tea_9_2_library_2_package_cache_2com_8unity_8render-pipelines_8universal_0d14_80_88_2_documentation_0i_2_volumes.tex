 The Universal Render Pipeline (URP) uses the Volume framework. Volumes can override or extend Scene properties depending on the Camera position relative to each Volume.

URP uses the Volume framework for \href{integration-with-post-processing.md\#post-proc-how-to}{\texttt{ post-\/processing}} effects.

URP implements dedicated Game\+Objects for Volumes\+: {\bfseries{Global Volume}}, {\bfseries{Box Volume}}, {\bfseries{Sphere Volume}}, {\bfseries{Convex Mesh Volume}}.



The Volume component contains the {\bfseries{Mode}} property that defines whether the Volume is Global or Local.



With Mode set to {\bfseries{Global}}, Volumes affect the Camera everywhere in the Scene. With Mode set to {\bfseries{Local}}, Volumes affect the Camera if the Camera is within the bounds of the Collider. For more information, see How to use Local Volumes.

You can add a {\bfseries{Volume}} component to any Game\+Object. A Scene can contain multiple Game\+Objects with Volume components. You can add multiple Volume components to a Game\+Object.

The Volume component references a \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_documentation_0i_2_volume_profile}{Volume Profile}{0}, which contains the Scene properties. A Volume Profile contains default values for every property and hides them by default. \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5d8e84fb42b2bb0720761c155081d9b4}{Volume Overrides}{0} let you change or extend the default properties in a \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_documentation_0i_2_volume_profile}{Volume Profile}{0}.

At runtime, URP goes through all of the enabled Volume components attached to active Game\+Objects in the Scene, and determines each Volume\textquotesingle{}s contribution to the final Scene settings. URP uses the Camera position and the Volume component properties to calculate the contribution. URP interpolates values from all Volumes with a non-\/zero contribution to calculate the final property values.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_documentation_0i_2_volumes_autotoc_md2672}{}\doxysection{\texorpdfstring{Volume component properties}{Volume component properties}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_documentation_0i_2_volumes_autotoc_md2672}

This section describes how to use a Local Volume to implement a location-\/based post-\/processing effect.

In this example, URP applies a post-\/processing effect when the Camera is within a certain Box Collider.


\begin{DoxyEnumerate}
\item In the Scene, create a new Box Volume ({\bfseries{Game\+Object \texorpdfstring{$>$}{>} Volume \texorpdfstring{$>$}{>} Box Volume}}).
\item Select the Box Volume. In Inspector, In the {\bfseries{Volume}} component, In the {\bfseries{Profile}} field, click {\bfseries{New}}.



\doxylink{namespace_unity}{Unity} creates the new Volume Profile and adds the {\bfseries{Add Override}} button to the Volume component.


\item If you have other Volumes in the Scene, change the value of the Priority property to ensure that the Overrides from this Volume have higher priority than those of other Volumes.


\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Click \href{VolumeOverrides.md\#volume-add-override}{\texttt{ Add Override}}. In the Volume Overrides dialog box, select a post-\/processing effect.
\item In the Collider component, adjust the Size and the Center properties so that the Collider occupies the volume where you want the local post-\/processing effect to be.



Ensure that the {\bfseries{Is Trigger}} check box is selected.
\end{DoxyEnumerate}

Now, when the Camera is within the bounds of the Volume\textquotesingle{}s Box Collider, URP uses the Volume Overrides from the Box Volume.
