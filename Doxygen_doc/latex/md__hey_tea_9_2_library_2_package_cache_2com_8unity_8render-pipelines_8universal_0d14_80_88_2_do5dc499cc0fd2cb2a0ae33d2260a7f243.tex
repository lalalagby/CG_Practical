 The Ambient Occlusion effect darkens creases, holes, intersections and surfaces that are close to each other in real-\/time. In the real world, such areas tend to block out or occlude ambient light, so they appear darker.

URP implements the Screen Space Ambient Occlusion (SSAO) effect as a \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do4e3c1604c0720cb973c486c434c47f28}{Renderer Feature}{0}. It works with every shader that the Universal Render Pipeline (URP) provides as well as any custom opaque Shader Graphs you create.

\begin{quote}
{\bfseries{Note}}\+: The SSAO effect is a Renderer Feature and works independently from the post-\/processing effects in URP. This effect does not depend on or interact with Volumes. \end{quote}

This section describes the properties of the SSAO Renderer Feature.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5dc499cc0fd2cb2a0ae33d2260a7f243_autotoc_md2323}{}\doxysubsection{\texorpdfstring{Method}{Method}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5dc499cc0fd2cb2a0ae33d2260a7f243_autotoc_md2323}
This property defines the type of noise the SSAO effect uses.

Available Options\+:


\begin{DoxyItemize}
\item {\bfseries{Interleaved Gradient Noise}}\+: Uses interleaved gradient noise to generate static SSAO.
\item {\bfseries{Blue Noise}}\+: Uses a selection of blue noise textures to generate dynamic SSAO. This creates an animated effect as the texture changes with every frame, as a result the SSAO effect is more subtle when the camera is in motion.
\end{DoxyItemize}

{\bfseries{Performance impact}}\+: Insignificant.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5dc499cc0fd2cb2a0ae33d2260a7f243_autotoc_md2324}{}\doxysubsection{\texorpdfstring{Intensity}{Intensity}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5dc499cc0fd2cb2a0ae33d2260a7f243_autotoc_md2324}
This property defines the intensity of the darkening effect.

{\bfseries{Performance impact}}\+: Insignificant.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5dc499cc0fd2cb2a0ae33d2260a7f243_autotoc_md2325}{}\doxysubsection{\texorpdfstring{Radius}{Radius}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5dc499cc0fd2cb2a0ae33d2260a7f243_autotoc_md2325}
When \doxylink{namespace_unity}{Unity} calculates the Ambient Occlusion value, the SSAO effect takes samples of the normal texture within this radius from the current pixel.

{\bfseries{Performance impact}}\+: High.

A lower {\bfseries{Radius}} value improves performance, because the SSAO Renderer Feature samples pixels closer to the source pixel. This makes caching more efficient.

Calculating the Ambient Occlusion Pass on objects closer to the Camera takes longer than on objects further from the Camera. This is because the {\bfseries{Radius}} property scales with the object.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5dc499cc0fd2cb2a0ae33d2260a7f243_autotoc_md2326}{}\doxysubsection{\texorpdfstring{Falloff Distance}{Falloff Distance}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5dc499cc0fd2cb2a0ae33d2260a7f243_autotoc_md2326}
SSAO does not apply to objects farther than this distance from the Camera.

A lower value increases performance in scenes that contain many distant objects. The performance improvement is smaller in smaller scenes with fewer objects.

Select the source of the normal vector values. The SSAO Renderer Feature uses normal vectors for calculating how exposed each point on a surface is to ambient lighting.

Available options\+:


\begin{DoxyItemize}
\item {\bfseries{Depth Normals}}\+: SSAO uses the normal texture generated by the {\ttfamily Depth\+Normals} Pass. This option lets \doxylink{namespace_unity}{Unity} make use of a more accurate normal texture.
\item {\bfseries{Depth}}\+: SSAO does not use the {\ttfamily Depth\+Normals} Pass to generate the normal texture. SSAO reconstructs the normal vectors using the depth texture instead. Use this option only if you want to avoid using the {\ttfamily Depth\+Normals} Pass block in your custom shaders. Selecting this option enables the {\bfseries{Normal Quality}} property.
\end{DoxyItemize}

{\bfseries{Performance impact}}\+: Depends on the application.

When you switch between the options {\bfseries{Depth Normals}} and {\bfseries{Depth}}, there might be a variation in performance, which depends on the target platform and the application. In a wide range of applications the difference in performance is small. In most cases, {\bfseries{Depth Normals}} produce a better visual look.

The reduction in resolution of the Ambient Occlusion Pass by a factor of two reduces the pixel count to process by a factor of four. This reduces the load on the GPU significantly, but makes the effect less detailed.

When you enable {\bfseries{After Opaque}}, \doxylink{namespace_unity}{Unity} calculates and applies the SSAO effect after the opaque render pass. This can increase performance when used with {\bfseries{Depth}} as the {\bfseries{Source}} for normal vector values as \doxylink{namespace_unity}{Unity} does not perform the skips depth prepass to calculate SSAO and instead uses the existing depth values.

{\bfseries{After Opaque}} can also increase performance on mobile devices that use tile-\/based rendering.

{\bfseries{Performance impact}}\+: Medium.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5dc499cc0fd2cb2a0ae33d2260a7f243_autotoc_md2333}{}\doxysubsection{\texorpdfstring{Blur Quality}{Blur Quality}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5dc499cc0fd2cb2a0ae33d2260a7f243_autotoc_md2333}
This property defines the quality of blur that \doxylink{namespace_unity}{Unity} applies to the SSAO effect. Higher quality blur creates a smoother, higher fidelity effect but requires more processing power.

Available options\+:


\begin{DoxyItemize}
\item {\bfseries{High}} (Bilateral)\+: Bilateral blur, takes three passes to process.
\item {\bfseries{Medium}} (Gaussian)\+: Gaussian blur, takes two passes to process.
\item {\bfseries{Low}} (Kawase)\+: Kawase blur, takes a single pass to process.
\end{DoxyItemize}

{\bfseries{Performance impact}}\+: Very high.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5dc499cc0fd2cb2a0ae33d2260a7f243_autotoc_md2334}{}\doxysubsection{\texorpdfstring{Samples}{Samples}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5dc499cc0fd2cb2a0ae33d2260a7f243_autotoc_md2334}
For each pixel, the SSAO Render Feature takes the selected number of samples within the specified radius to calculate the Ambient Occlusion value. A higher value makes the effect smoother and more detailed, but also reduces performance.

Available options\+:


\begin{DoxyItemize}
\item {\bfseries{High}}\+: 12 \doxylink{namespace_samples}{Samples}
\item {\bfseries{Medium}}\+: 8 \doxylink{namespace_samples}{Samples}
\item {\bfseries{Low}}\+: 4 \doxylink{namespace_samples}{Samples}
\end{DoxyItemize}

{\bfseries{Performance impact}}\+: High.

An increase in the {\bfseries{Sample Count}} value from 4 to 8 doubles the computational load on the GPU.

The SSAO Renderer Feature uses normal vectors for calculating how exposed each point on a surface is to ambient lighting.

URP 10.\+0 implements the {\ttfamily Depth\+Normals} Pass block that generates the the normal texture {\ttfamily \+\_\+\+Camera\+Normals\+Texture} for the current frame. By default, the SSAO Renderer Feature uses this texture to calculate Ambient Occlusion values.

If you implement your custom SRP and if you do not want to implement the {\ttfamily Depth\+Normals} Pass block in your shaders, you can use the SSAO Renderer Feature and set its {\bfseries{Source}} property to {\bfseries{Depth}}. In this case, \doxylink{namespace_unity}{Unity} does not use the {\ttfamily Depth\+Normals} Pass to generate the normal vectors, it reconstructs the normal vectors using the depth texture instead.

Selecting the option {\bfseries{Depth}} in the {\bfseries{Source}} property enables the {\bfseries{Normal Quality}} property. The options in this property (Low, Medium, and High) determine the number of samples of the depth texture that \doxylink{namespace_unity}{Unity} takes when reconstructing the normal vector from the depth texture. The number of samples per quality level\+: Low\+: 1, Medium\+: 5, High\+: 9.
