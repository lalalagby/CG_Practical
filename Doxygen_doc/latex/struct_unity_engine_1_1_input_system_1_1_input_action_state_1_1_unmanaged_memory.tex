\doxysubsection{Detailed Description}
Unmanaged memory kept for action maps. 

Most of the dynamic execution state for actions we keep in a single block of unmanaged memory. Essentially, only the C\# heap objects (like \doxylink{interface_unity_engine_1_1_input_system_1_1_i_input_interaction}{IInput\+Interaction} and such) we keep in managed arrays. Aside from being able to condense the data into a single block of memory and not having to have it spread out on the GC heap, we gain the advantage of being able to freely allocate and re-\/allocate these blocks without creating garbage on the GC heap.

The data here is set up by Input\+Binding\+Resolver.
