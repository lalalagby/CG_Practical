\doxysubsection{Detailed Description}
An Xbox-\/style gamepad with two sticks, a D-\/\+Pad, four face buttons, two triggers, two shoulder buttons, and two menu buttons that usually sit in the midsection of the gamepad. 

The \doxylink{class_unity_engine_1_1_input_system_1_1_gamepad}{Gamepad} layout provides a standardized layouts for gamepads. Generally, if a specific device is represented as a \doxylink{class_unity_engine_1_1_input_system_1_1_gamepad}{Gamepad}, the controls, such as the face buttons, are guaranteed to be mapped correctly and consistently. If, based on the set of supported devices available to the input system, this cannot be guaranteed, a given device is usually represented as a generic \doxylink{class_unity_engine_1_1_input_system_1_1_joystick}{Joystick} or as just a plain \doxylink{class_unity_engine_1_1_input_system_1_1_h_i_d_1_1_h_i_d}{HID.\+HID} instead.

\hline
{\em high\+Frequency} & Speed of the high-\/frequency (right) motor. Normalized \mbox{[}0..1\mbox{]} value with 1 indicating maximum speed and 0 indicating the motor is turned off. Will automatically be clamped into range.\\
\hline

Also, how quickly the motors react and how often the speed can be updated will depend on the hardware and drivers.

\mbox{\hyperlink{class_unity_engine_1_1_input_system_1_1_gamepad}{Gamepad}} Unity\+Engine.\+Input\+System.\+Gamepad.\+current\hspace{0.3cm}{\ttfamily [static]}, {\ttfamily [get]}}



The gamepad last used/connected by the player or {\ttfamily null} if there is no gamepad connected to the system. 

When added, a device is automatically made current (see \doxylink{class_unity_engine_1_1_input_system_1_1_input_device_a0a98203c32eb180ab8b7c862dc27635c}{Input\+Device.\+Make\+Current}), so when connecting a gamepad, it will also become current. After that, it will only become current again when input change on non-\/noisy controls (see \doxylink{class_unity_engine_1_1_input_system_1_1_input_control_afa5a8531d7c9a030e8e484560a59fe6f}{Input\+Control.\+noisy}) is received.

For local multiplayer scenarios (or whenever there are multiple gamepads that need to be usable in a concurrent fashion), it is not recommended to rely on this property. Instead, it is recommended to use \doxylink{class_unity_engine_1_1_input_system_1_1_player_input}{Player\+Input} or \doxylink{struct_unity_engine_1_1_input_system_1_1_users_1_1_input_user}{Users.\+Input\+User}.
