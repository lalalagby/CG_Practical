\doxylink{namespace_unity}{Unity} raises a {\ttfamily begin\+Camera\+Rendering} event before it renders each active Camera in every frame. If a Camera is inactive (for example, if the {\bfseries{Camera}} component checkbox is cleared on a Camera Game\+Object), \doxylink{namespace_unity}{Unity} does not raise a {\ttfamily begin\+Camera\+Rendering} event for this Camera.

When you subscribe a method to this event, you can execute custom logic before \doxylink{namespace_unity}{Unity} renders the Camera. Examples of custom logic include rendering extra Cameras to Render Textures, and using those Textures for effects like planar reflections or surveillance camera views.

Other events in the \href{https://docs.unity3d.com/ScriptReference/Rendering.RenderPipelineManager.html}{\texttt{ Render\+Pipeline\+Manager}} class provide more ways to customize URP. You can also use the principles described in this article with those events.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5bd064643a7d4c95e1a7c7490852589b_autotoc_md2667}{}\doxysection{\texorpdfstring{begin\+Camera\+Rendering event example}{begin\+Camera\+Rendering event example}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do5bd064643a7d4c95e1a7c7490852589b_autotoc_md2667}
