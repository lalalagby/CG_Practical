\doxysubsection{Detailed Description}
A mapping of controls to an action. 

Each binding represents a value received from controls (see \doxylink{class_unity_engine_1_1_input_system_1_1_input_control}{Input\+Control}). There are two main types of bindings\+: "{}normal"{} bindings and "{}composite"{} bindings.

Normal bindings directly bind to control(s) by means of \doxylink{struct_unity_engine_1_1_input_system_1_1_input_binding_a0776a05b5acf2b545496b95b4f556e9d}{path} which is a "{}control path"{} (see Input\+Control\+Path for details about how to form paths). At runtime, the path of such a binding may match none, one, or multiple controls. Each control matched by the path will feed input into the binding.

Composite bindings do not bind to controls themselves. Instead, they receive their input from their "{}part"{} bindings and then return a value representing a "{}composition"{} of those inputs. What composition specifically is performed depends on the type of the composite. \doxylink{class_unity_engine_1_1_input_system_1_1_composites_1_1_axis_composite}{Composites.\+Axis\+Composite}, for example, will return a floating-\/point axis value computed from the state of two buttons.

The action that is triggered by a binding is determined by its \doxylink{struct_unity_engine_1_1_input_system_1_1_input_binding_a83005305665b72fa6dadd5a105613495}{action} property. The resolution to an \doxylink{class_unity_engine_1_1_input_system_1_1_input_action}{Input\+Action} depends on where the binding is used. For example, bindings that are part of \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_map_ae98277c3697a798a28f8b16e187217ff}{Input\+Action\+Map.\+bindings} will resolve action names to actions in the same \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_map}{Input\+Action\+Map}.

A binding can also be used as a form of search mask or filter. In this use, \doxylink{struct_unity_engine_1_1_input_system_1_1_input_binding_a0776a05b5acf2b545496b95b4f556e9d}{path}, \doxylink{struct_unity_engine_1_1_input_system_1_1_input_binding_a83005305665b72fa6dadd5a105613495}{action}, and \doxylink{struct_unity_engine_1_1_input_system_1_1_input_binding_a718d4735f0458bbc689951a64ba25a89}{groups} become search criteria that are matched against other bindings. See \doxylink{struct_unity_engine_1_1_input_system_1_1_input_binding_aa69fd73f446257c84723c95ad8fbcbe8}{Matches(\+Input\+Binding)} for details. This use is employed in places such as Input\+Action\+Rebinding\+Extensions as well as in binding masks on actions (\doxylink{class_unity_engine_1_1_input_system_1_1_input_action_a41caa3a44bc876781c5b1bcb04622cd2}{Input\+Action.\+binding\+Mask}), action maps (\doxylink{class_unity_engine_1_1_input_system_1_1_input_action_map_af68ee392460d425649d96935d8f7345d}{Input\+Action\+Map.\+binding\+Mask}), and assets (\doxylink{class_unity_engine_1_1_input_system_1_1_input_action_asset_a7cf65e9b19155bd044d9d06250e61225}{Input\+Action\+Asset.\+binding\+Mask}).

In this method, the current binding acts as a "{}mask"{}. When used this way, only three properties of the binding are taken into account\+: \doxylink{struct_unity_engine_1_1_input_system_1_1_input_binding_a0776a05b5acf2b545496b95b4f556e9d}{path}, \doxylink{struct_unity_engine_1_1_input_system_1_1_input_binding_a718d4735f0458bbc689951a64ba25a89}{groups}, and \doxylink{struct_unity_engine_1_1_input_system_1_1_input_binding_a83005305665b72fa6dadd5a105613495}{action}.

For each of these properties, the method checks whether they are set on the current binding and, if so, matches them against the respective property in {\itshape binding} .

The way this matching works is that the value of the property in the current binding is allowed to be a semicolon-\/separated list where each element specifies one possible value that will produce a match.

Note that all comparisons are case-\/insensitive.
