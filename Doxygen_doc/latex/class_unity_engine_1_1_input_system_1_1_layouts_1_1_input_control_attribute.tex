\doxysubsection{Detailed Description}
Mark a field or property as representing/identifying an input control in some form. 

This attribute is used in different places for different purposes.

When creating input control layouts (\doxylink{class_unity_engine_1_1_input_system_1_1_layouts_1_1_input_control_layout}{Input\+Control\+Layout}) in C\#, applying the attribute to fields in a state struct (see IInput\+State\+Type\+Info or Gamepad\+State for an example) or to properties in an input device (\doxylink{class_unity_engine_1_1_input_system_1_1_input_device}{Input\+Device}), will cause an \doxylink{class_unity_engine_1_1_input_system_1_1_input_control}{Input\+Control} to be created from the field or property at runtime. The attribute can be applied multiple times to create multiple input controls (e.\+g. when having an int field that represents a bitfield where each bit is a separate button).

Another use is for marking {\ttfamily string} type fields that represent input control paths. Applying the attribute to them will cause them to automatically use a custom inspector similar to the one found in the action editor. For this use, only the \doxylink{class_unity_engine_1_1_input_system_1_1_layouts_1_1_input_control_attribute_af4f2d9f4d0708ae41565d12d8aa06fb4}{layout} property is taken into account.

Layout to use for the control. 

Layout to use for the control.

If this is not set, the system tries to infer the layout type from the value type of the field or property. If the value type is itself registered as a layout, that layout will be used (e.\+g. when you have a property of type \doxylink{class_unity_engine_1_1_input_system_1_1_controls_1_1_button_control}{Controls.\+Button\+Control}, the layout will be inferred to be "{}\+Button"{}). Otherwise, if a layout with the same name as the type is registered, that layout will be used (e.\+g. when you have a field of type Vector3, the layout will be inferred to be "{}\+Vector3"{}).

string Unity\+Engine.\+Input\+System.\+Layouts.\+Input\+Control\+Attribute.\+processors\hspace{0.3cm}{\ttfamily [get]}, {\ttfamily [set]}}



Optional list of processors to add to the control. 

\doxylink{namespace_unity_engine_1_1_input_system_1_1_processors}{Processors} to apply to the control.

Each element in the list is a name of a processor (as registered with Input\+System.\+Register\+Processor$<$\+T$>$) followed by an optional list of parameters.

For example, {\ttfamily "{}normalize(min=0,max=256)"{}} is one element that puts a {\ttfamily Normalize\+Processor} on the control and sets its {\ttfamily min} field to 0 and its its {\ttfamily max} field to 256.

Multiple processors can be put on a control by separating them with a comma. For example, {\ttfamily "{}normalize(max=256),scale(factor=2)"{}} puts both a {\ttfamily Normalize\+Processor} and a {\ttfamily Scale\+Processor} on the control. \doxylink{namespace_unity_engine_1_1_input_system_1_1_processors}{Processors} are applied in the order they are listed.
