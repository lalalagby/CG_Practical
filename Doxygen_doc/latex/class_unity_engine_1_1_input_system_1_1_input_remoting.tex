\doxysubsection{Detailed Description}
Makes the activity and data of an Input\+Manager observable in message form. 

Can act as both the sender and receiver of these message so the flow is fully bidirectional, i.\+e. the Input\+Manager on either end can mirror its layouts, devices, and events over to the other end. This permits streaming input not just from the player to the editor but also feeding input from the editor back into the player.

Remoting sits entirely on top of the input system as an optional piece of functionality. In development players and the editor, we enable it automatically but in non-\/development players it has to be explicitly requested by the user.

To see devices and input from players in the editor, open the Input Debugger through "{}\+Windows $>$$>$ Input Debugger"{}.
