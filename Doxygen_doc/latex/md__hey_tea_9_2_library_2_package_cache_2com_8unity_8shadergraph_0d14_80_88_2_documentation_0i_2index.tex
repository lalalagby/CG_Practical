Shader Graph enables you to build shaders visually. Instead of writing code, you create and connect nodes in a graph framework. Shader Graph gives instant feedback that reflects your changes, and itâ€™s simple enough for users who are new to shader creation.

For an introduction to Shader Graph, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2_getting-_started}{Getting Started}{0}.

Shader Graph is available through the Package Manager window in supported versions of the \doxylink{namespace_unity}{Unity} Editor. If you install a Scriptable Render Pipeline (SRP) such as the \href{https://docs.unity3d.com/Packages/com.unity.render-pipelines.universal@latest}{\texttt{ Universal Render Pipeline (URP)}} or the \href{https://docs.unity3d.com/Packages/com.unity.render-pipelines.high-definition@latest}{\texttt{ High Definition Render Pipeline (HDRP)}}, \doxylink{namespace_unity}{Unity} automatically installs Shader Graph in your project.

Shader Graph package versions on \doxylink{namespace_unity}{Unity} Engine 2018.\+x are {\itshape Preview} versions, which do not receive bug fixes and feature maintenance. To work with an actively supported version of Shader Graph, use \doxylink{namespace_unity}{Unity} Engine 2019.\+1 or higher.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2index_autotoc_md3349}{}\doxysubsection{\texorpdfstring{SRP packages are part of the core}{SRP packages are part of the core}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8shadergraph_0d14_80_88_2_documentation_0i_2index_autotoc_md3349}
With the release of \doxylink{namespace_unity}{Unity} 2021.\+1, graphics packages are relocating to the core of \doxylink{namespace_unity}{Unity}. This move simplifies the experience of working with new \doxylink{namespace_unity}{Unity} graphics features, as well as ensuring that your projects are always running on the latest verified graphics code.

For each release of \doxylink{namespace_unity}{Unity} (alpha / beta / patch release) graphics packages are embedded within the main \doxylink{namespace_unity}{Unity} installer. When you install the latest release of \doxylink{namespace_unity}{Unity}, you also get the latest \href{https://docs.unity3d.com/Packages/com.unity.render-pipelines.universal@latest}{\texttt{ Universal Render Pipeline (URP)}}, \href{https://docs.unity3d.com/Packages/com.unity.render-pipelines.high-definition@latest}{\texttt{ High Definition Render Pipeline (HDRP)}}, Shader Graph, \href{https://docs.unity3d.com/Packages/com.unity.visualeffectgraph@latest}{\texttt{ Visual Effect (VFX) Graph}} packages, among others.

Tying graphics packages to the main \doxylink{namespace_unity}{Unity} release allows better testing to ensure that the graphics packages you use have been tested extensively with the version of \doxylink{namespace_unity}{Unity} you have downloaded.

You can also use a local copy or a custom version of the graphics packages with an override in the manifest file.
