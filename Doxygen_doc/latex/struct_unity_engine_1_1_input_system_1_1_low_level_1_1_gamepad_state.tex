\doxysubsection{Detailed Description}
Default state layout for gamepads. 

Be aware that unlike some other devices such as \doxylink{class_unity_engine_1_1_input_system_1_1_mouse}{Mouse} or \doxylink{class_unity_engine_1_1_input_system_1_1_touchscreen}{Touchscreen}, gamepad devices tend to have wildly varying state formats, i.\+e. forms in which they internally store their input data. In practice, even on the same platform gamepads will often store their data in different formats. This means that \doxylink{struct_unity_engine_1_1_input_system_1_1_low_level_1_1_gamepad_state}{Gamepad\+State} will often {\itshape not} be the format in which a particular gamepad (such as \doxylink{class_unity_engine_1_1_input_system_1_1_x_input_1_1_x_input_controller}{XInput.\+XInput\+Controller}, for example) stores its data.

If your gamepad data is arriving in a different format, you should extend the "{}\+Gamepad"{} layout and customize its \doxylink{namespace_unity_engine_1_1_input_system_1_1_controls}{Controls}.

A real-\/world example of this is the Xbox Controller on mac\+OS, which is supported through \doxylink{namespace_unity_engine_1_1_input_system_1_1_h_i_d}{HID}. Its layout looks like this\+:
