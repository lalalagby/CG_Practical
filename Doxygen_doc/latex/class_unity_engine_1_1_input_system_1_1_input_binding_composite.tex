\doxysubsection{Detailed Description}
A binding that synthesizes a value from from several component bindings. 

A binding composite arranges several bindings such that they form a "{}virtual control"{}.

This is the base class for composite bindings. See Input\+Binding\+Composite$<$\+TValue$>$ for more details about composites and for how to define custom composites.

Composite bindings are a special type of \doxylink{struct_unity_engine_1_1_input_system_1_1_input_binding}{Input\+Binding}. Whereas normally an input binding simply references a set of controls and returns whatever input values are generated by those controls, a composite binding sources input from several controls and derives a new value from that.

A good example for that is a classic WASD keyboard binding\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var\ moveAction\ =\ \textcolor{keyword}{new}\ InputAction(name:\ \textcolor{stringliteral}{"{}move"{}});}
\DoxyCodeLine{moveAction.AddCompositeBinding(\textcolor{stringliteral}{"{}Vector2"{}})}
\DoxyCodeLine{\ \ \ \ .With(\textcolor{stringliteral}{"{}Up"{}},\ \textcolor{stringliteral}{"{}<Keyboard>/w"{}})}
\DoxyCodeLine{\ \ \ \ .With(\textcolor{stringliteral}{"{}Down"{}},\ \textcolor{stringliteral}{"{}<Keyboard>/s"{}})}
\DoxyCodeLine{\ \ \ \ .With(\textcolor{stringliteral}{"{}Left"{}},\ \textcolor{stringliteral}{"{}<Keyboard>/a"{}})}
\DoxyCodeLine{\ \ \ \ .With(\textcolor{stringliteral}{"{}Right"{}},\ \textcolor{stringliteral}{"{}<Keyboard>/d"{}})}

\end{DoxyCode}
 

Here, each direction is represented by a separate binding. "{}\+Up"{} is bound to "{}\+W"{}, "{}\+Down"{} is bound to "{}\+S"{}, and so on. Each direction individually returns a 0 or 1 depending on whether it is pressed or not.

However, as a composite, the binding to the "{}move"{} action returns a combined {\ttfamily Vector2} that is computed from the state of each of the directional controls. This is what composites do. They take inputs from their "{}parts"{} to derive an input for the binding as a whole.

Note that the properties and methods defined in \doxylink{class_unity_engine_1_1_input_system_1_1_input_binding_composite}{Input\+Binding\+Composite} and this class will generally be called internally by the input system and are not generally meant to be called directly from user land.

The set of composites available in the system is extensible. While some composites are such as \doxylink{class_unity_engine_1_1_input_system_1_1_composites_1_1_vector2_composite}{Composites.\+Vector2\+Composite} and \doxylink{class_unity_engine_1_1_input_system_1_1_composites_1_1_button_with_one_modifier}{Composites.\+Button\+With\+One\+Modifier} are available out of the box, new composites can be implemented by anyone and simply be registered with Input\+System.\+Register\+Binding\+Composite$<$\+T$>$.

See the "{}\+Custom Composite"{} sample (can be installed from package manager \doxylink{namespace_unity_engine_1_1_u_i}{UI}) for a detailed example of how to create a custom composite.
