\doxysubsection{Detailed Description}
A test fixture for writing tests that use the input system. Can be derived from or simply instantiated from another test fixture. 

The fixture will put the input system into a known state where it has only the built-\/in set of basic layouts and no devices. The state of the system before starting a test is recorded and restored when the test finishes.

The test fixture will also sever the tie of the input system to the \doxylink{namespace_unity}{Unity} runtime. This means that while the test fixture is active, the input system will not receive input and device discovery or removal notifications from platform code. This ensures that while the test is running, input that may be generated on the machine running the test will not infer with it.
