\doxysubsection{Detailed Description}
Read-\/only access to an array or to a slice of an array. 


\begin{DoxyTemplParams}{Template Parameters}
{\em TValue} & Type of values stored in the array.\\
\hline
\end{DoxyTemplParams}


The purpose of this struct is to allow exposing internal arrays directly such that no boxing and no going through interfaces is required but at the same time not allowing the internal arrays to be modified.

It differs from {\ttfamily Read\+Only\+Span\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}} in that it can be stored on the heap and differs from {\ttfamily Read\+Only\+Collection\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}} in that it supports slices directly without needing an intermediate object representing the slice.

Note that in most cases, the \doxylink{struct_unity_engine_1_1_input_system_1_1_utilities_1_1_read_only_array}{Read\+Only\+Array} instance should be treated as a {\itshape temporary}. The actual array referred to by a \doxylink{struct_unity_engine_1_1_input_system_1_1_utilities_1_1_read_only_array}{Read\+Only\+Array} instance is usually owned and probably mutated by another piece of code. When that code makes changes to the array, the \doxylink{struct_unity_engine_1_1_input_system_1_1_utilities_1_1_read_only_array}{Read\+Only\+Array} instance will not get updated.
