 The Input \doxylink{namespace_system}{System} currently has limited support for joysticks as generic \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_h_i_d}{HIDs}{0} only. The system attempts to identify Controls based on the information provided in the HID descriptor of the Device, but it might not always be accurate. These Devices often work best when you allow the user to \href{../api/UnityEngine.InputSystem.InputActionRebindingExtensions.html}{\texttt{ manually remap the Controls}}.

To better support specific joysticks Devices, you can also \href{HID.md\#creating-a-custom-device-layout}{\texttt{ provide your own custom mappings for those Devices}}. \doxylink{namespace_unity}{Unity} might extend the Input \doxylink{namespace_system}{System} to include some mappings for common devices in the future. See the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_h_i_d}{manual page on HID}{0} for more information.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_joystick_autotoc_md1419}{}\doxysection{\texorpdfstring{Controls}{Controls}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_joystick_autotoc_md1419}
The Input \doxylink{namespace_system}{System} supports Generic HID Input Devices which are recognized as joysticks via the \href{../api/UnityEngine.InputSystem.Joystick.html}{\texttt{ {\ttfamily Joystick}}} class. Joystick Devices can have any number of Controls as reported by the Device\textquotesingle{}s HID descriptor, but the Input \doxylink{namespace_system}{System} always tries to at least match these common Controls\+:
