\doxylink{namespace_unity}{Unity} now issues an error when instances of {\ttfamily Scriptable\+Renderer\+Feature} attempt to access render targets before they are allocated by the {\ttfamily Scriptable\+Renderer} class.

The {\ttfamily Scriptable\+Renderer\+Feature} class has a new virtual function {\ttfamily Setup\+Render\+Passes} which is called when render targets are allocated and ready to be used.

If your code uses the {\ttfamily Scriptable\+Renderer.\+camera\+Color\+Target} or the {\ttfamily Scriptable\+Renderer.\+camera\+Depth\+Target} property inside of the {\ttfamily Add\+Render\+Passes} method override, you should move that implementation to the {\ttfamily Scriptable\+Renderer\+Feature.\+Setup\+Render\+Passes} method.

The calls to the {\ttfamily Scriptable\+Renderer.\+Enqueue\+Pass} method should still happen in the {\ttfamily Add\+Render\+Passes} method.

The Universal Renderer is now using \href{https://docs.unity3d.com/Packages/com.unity.render-pipelines.core@14.0/manual/rthandle-system.html}{\texttt{ the RTHandle system}} for its internal targets and in its internal passes.

All usages of the {\ttfamily Render\+Target\+Handle} struct are set as obsolete and the struct will be removed in the future.

The public interfaces {\ttfamily Scriptable\+Renderer.\+camera\+Color\+Target} and {\ttfamily Scriptable\+Renderer.\+camera\+Depth\+Target} are marked as obsolete. Replace them with {\ttfamily Scriptable\+Renderer.\+camera\+Color\+Target\+Handle} and {\ttfamily Scriptable\+Renderer.\+camera\+Depth\+Target\+Handle} respectively.

{\ttfamily RTHandle} targets do not use the {\ttfamily Command\+Buffer.\+Get\+Temporary\+RT} method and persist for more frames than the {\ttfamily Render\+Target\+Identifier} structs. You cannot allocate {\ttfamily RTHandle} targets with the properties {\ttfamily Graphics\+Format} and {\ttfamily Depth\+Buffer\+Bits} set to any value except for 0. The {\ttfamily camera\+Depth\+Target} properties must be separate from the {\ttfamily camera\+Color\+Target} properties.

The following helper functions let you create and use temporary render target with the {\ttfamily RTHandle} system in a similar way as with the {\ttfamily Get\+Temporary\+RT} method previously\+:


\begin{DoxyItemize}
\item {\ttfamily Rendering\+Utils.\+Re\+Allocate\+If\+Needed}
\item {\ttfamily Shadow\+Utils.\+Shadow\+RTRe\+Allocate\+If\+Needed}
\end{DoxyItemize}

If the render target does not change within the lifetime of the application, use the {\ttfamily RTHandles.\+Alloc} method to allocate an {\ttfamily RTHandle} target. This method is efficient since the code does not have to check if a render target should be allocated on each frame.

If the render target is a full screen texture, which means that its resolution matches or is a fraction of the resolution of the screen, use a scaling factor such as {\ttfamily Vector2\+D.\+one} to support dynamic scaling.
