{\ttfamily Unity\+Set\+Up} and {\ttfamily Unity\+Tear\+Down} can be used with either the {\ttfamily Test} or {\ttfamily Unity\+Test} test attributes. In both cases the relative execution order of \doxylink{namespace_unity}{Unity} and non-\/\+Unity {\ttfamily Set\+Up} and {\ttfamily Tear\+Down} attributes is the same. The only difference is that a {\ttfamily Unity\+Test} allows for yielding instructions during the test that can result in a domain reload, in which case the non-\/\+Unity {\ttfamily Set\+Up} and {\ttfamily Tear\+Down} methods are re-\/run before proceeding to the second part of the test.

The term {\bfseries{base}} in the execution order denotes a base class from which a test class inherits. {\ttfamily Unity\+Set\+Up} and {\ttfamily Unity\+Tear\+Down} follow the same pattern as NUnit {\ttfamily Set\+Up} and {\ttfamily Tear\+Down} attributes in determining execution order between base classes and their derivatives. {\ttfamily Set\+Up} methods are called on base classes first, and then on derived classes. {\ttfamily Tear\+Down} methods are called on derived classes first, and then on the base class. See the \href{https://docs.nunit.org/articles/nunit/technical-notes/usage/SetUp-and-TearDown.html}{\texttt{ NUnit Documentation}} for more details.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0716d1700c82ca4324abf1039f6677e70_autotoc_md4376}{}\doxysubsection{\texorpdfstring{Base and Derived class example}{Base and Derived class example}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0716d1700c82ca4324abf1039f6677e70_autotoc_md4376}
