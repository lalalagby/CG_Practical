 URP draws objects in the {\bfseries{Draw\+Opaque\+Objects}} and {\bfseries{Draw\+Transparent\+Objects}} passes. You might need to draw objects at a different point in the frame rendering, or interpret and write rendering data (like depth and stencil) in alternate ways. The Render Objects Renderer Feature lets you do such customizations by letting you draw objects on a certain layer, at a certain time, with specific overrides.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do1ed6d080483a8688083d1986785cc6ec_autotoc_md2421}{}\doxysection{\texorpdfstring{How to use the Render Objects Renderer Feature}{How to use the Render Objects Renderer Feature}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_do1ed6d080483a8688083d1986785cc6ec_autotoc_md2421}

{\bfseries{Name}}   &Use this field to edit the name of the feature.    \\\cline{1-2}
{\bfseries{Event}}   &The event in the URP queue when \doxylink{namespace_unity}{Unity} executes this Renderer Feature.    \\\cline{1-2}
{\bfseries{Filters}}   &Settings that let you configure which objects this Renderer Feature renders.    \\\cline{1-2}
Queue   &Select whether the feature renders opaque or transparent objects.    \\\cline{1-2}
Layer Mask   &The Renderer Feature renders objects from layers you select in this property.    \\\cline{1-2}
{\bfseries{Pass Names}}   &If a Pass in a shader has the {\ttfamily Light\+Mode} Pass Tag, this Renderer Feature processes only the shaders where the value of the {\ttfamily Light\+Mode} Pass Tag equals one of the values in the Pass Names property.    \\\cline{1-2}
{\bfseries{Overrides}}   &Settings in this section let you configure overrides for certain properties when rendering with this Renderer Feature.    \\\cline{1-2}
Override Mode   &Specify the material override mode.    \\\cline{1-2}
Material   &(Override Mode is set to Material) When rendering an object, \doxylink{namespace_unity}{Unity} replaces the Material assigned to it with this Material. This will override all material properties with this material    \\\cline{1-2}
Shader   &(Override Mode is set to Shader) When rendering an object, \doxylink{namespace_unity}{Unity} replaces the material assigned to it with this shader. This maintains all material properties and allows the override shader to access these properties. This is currently not SRPBatcher compatible and less performant.    \\\cline{1-2}
Depth   &Selecting this option lets you specify how this Renderer Feature affects or uses the Depth buffer. This option contains the following items\+:~\newline
Write Depth\+: this option defines whether the Renderer Feature updates the Depth buffer when rendering objects.~\newline
Depth Test\+: the condition which determines when this Renderer Feature renders pixels of a given object.    \\\cline{1-2}
Stencil   &With this check box selected, the Renderer processes the Stencil buffer values.~\newline
For more information on how \doxylink{namespace_unity}{Unity} works with the Stencil buffer, see \href{https://docs.unity3d.com/Manual/SL-Stencil.html}{\texttt{ Shader\+Lab\+: Stencil}}.    \\\cline{1-2}
Camera   &Selecting this option lets you override the following Camera properties\+:~\newline
Field of View\+: when rendering objects, the Renderer Feature uses this Field of View instead of the value specified on the Camera.~\newline
Position Offset\+: when rendering objects, the Renderer Feature moves them by this offset.~\newline
Restore\+: with this option selected, the Renderer Feature restores the original Camera matrices after executing the render passes in this Renderer Feature.   \\\cline{1-2}
