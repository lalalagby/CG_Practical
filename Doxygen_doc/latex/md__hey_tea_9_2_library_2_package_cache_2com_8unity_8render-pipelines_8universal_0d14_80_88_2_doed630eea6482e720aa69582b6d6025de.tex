The Rendering Debugger window is available in the following modes\+:


\begin{DoxyItemize}
\item The Editor.
\item The Play mode.
\item At runtime in the standalone \doxylink{namespace_unity}{Unity} \doxylink{class_player}{Player}, on any device. The window is only available in {\bfseries{Development builds}}.
\end{DoxyItemize}

When using the {\bfseries{Rendering Debugger}} window in the Development build, clear the {\bfseries{Strip Debug Variants}} check box in {\bfseries{Project Settings \texorpdfstring{$>$}{>} \doxylink{namespace_graphics}{Graphics} \texorpdfstring{$>$}{>} URP Global Settings}}.

Use one of the following options to open the {\bfseries{Rendering Debugger}} window.

{\bfseries{In the Editor}}\+:


\begin{DoxyItemize}
\item Select {\bfseries{Window \texorpdfstring{$>$}{>} Analysis \texorpdfstring{$>$}{>} Rendering Debugger}}.
\item Press {\bfseries{Ctrl+\+Backspace}} ({\bfseries{Ctrl+\+Delete}} on mac\+OS).
\end{DoxyItemize}

{\bfseries{In the Play mode or at runtime in a Development build}}\+:


\begin{DoxyItemize}
\item On a desktop or laptop computer, press {\bfseries{Left\+Ctrl+\+Backspace}} ({\bfseries{Left\+Ctrl+\+Delete}} on mac\+OS).
\item On a console controller, press L3 and R3 (Left Stick and Right Stick).
\item On a mobile device, use a three-\/finger double tap.
\end{DoxyItemize}

You can disable the runtime UI using the \href{https://docs.unity3d.com/Packages/com.unity.render-pipelines.core@14.0/api/UnityEngine.Rendering.DebugManager.html\#UnityEngine_Rendering_DebugManager_enableRuntimeUI}{\texttt{ enable\+Runtime\+UI}} property.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_doed630eea6482e720aa69582b6d6025de_autotoc_md2129}{}\doxysection{\texorpdfstring{\label{_ui-sections}%

A bottleneck is a condition that occurs when one process performs significantly slower than other components, and other components depend on it.

The {\bfseries{Bottlenecks}} section describes the distribution of the last 60 frames across the CPU and GPU. You can only see the Bottleneck information when you build your player on a device.

{\bfseries{Note}}\+: Vsync limits the {\bfseries{Frame Rate}} based on the refresh rate of your deviceâ€™s screen. This means when you enable Vsync, the {\bfseries{Present Limited}} category is 100\% in most cases. To turn Vsync off, go to {\bfseries{Edit}} \texorpdfstring{$>$}{>} {\bfseries{Project settings}} \texorpdfstring{$>$}{>} {\bfseries{Quality}} \texorpdfstring{$>$}{>} {\bfseries{Current Active Quality Level}} and set the {\bfseries{Vsync Count}} set to {\bfseries{Don\textquotesingle{}t Sync}}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_doed630eea6482e720aa69582b6d6025de_autotoc_md2133}{}\doxysubsubsection{\texorpdfstring{Bottleneck categories}{Bottleneck categories}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_doed630eea6482e720aa69582b6d6025de_autotoc_md2133}

If Vsync limited 20 of the 60 most recent frames, the Bottleneck section might appear as follows\+:


\begin{DoxyItemize}
\item {\bfseries{CPU}} 0.\+0\%\+: This indicates that HDRP did not render any of the last 60 frames on the CPU.
\item {\bfseries{GPU}} 66.\+6\%\+: This indicates that the GPU limited 66.\+6\% of the 60 most recent frames rendered by HDRP.
\item {\bfseries{Present Limited}} 33.\+3\%\+: This indicates that presentation constraints (Vsync or the \href{https://docs.unity3d.com/ScriptReference/Application-targetFrameRate.html}{\texttt{ target framerate}}) limited 33.\+3\% of the last 60 frames.
\item {\bfseries{Balanced}} 0.\+0\%\+: This indicates that in the last 60 frames, there were 0 frames where the CPU processing time and GPU processing time were the same.
\end{DoxyItemize}

In this example, the bottleneck is the GPU.

{\bfseries{Lighting Debug Mode}}   &Specifies which lighting and shadow information to overlay on-\/screen to debug. The options are\+:
\begin{DoxyItemize}
\item {\bfseries{None}}\+: Renders the scene normally without a debug overlay.
\item {\bfseries{Shadow Cascades}}\+: Overlays shadow cascade information so you can see which shadow cascade each pixel uses. Use this to debug shadow cascade distances. For information on which color represents which shadow cascade, see the \href{../universalrp-asset.md\#shadows}{\texttt{ Shadows section of the URP Asset}}.
\item {\bfseries{Lighting Without Normal Maps}}\+: Renders the scene to visualize lighting. This mode uses neutral materials and disables normal maps. This and the {\bfseries{Lighting With Normal Maps}} mode are useful for debugging lighting issues caused by normal maps.
\item {\bfseries{Lighting With Normal Maps}}\+: Renders the scene to visualize lighting. This mode uses neutral materials and allows normal maps.
\item {\bfseries{Reflections}}\+: Renders the scene to visualize reflections. This mode applies perfectly smooth, reflective materials to every Mesh Renderer.
\item {\bfseries{Reflections With Smoothness}}\+: Renders the scene to visualize reflections. This mode applies reflective materials without an overridden smoothness to every Game\+Object.
\end{DoxyItemize}\\\cline{1-2}
{\bfseries{Lighting Features}}   &Specifies flags for which lighting features contribute to the final lighting result. Use this to view and debug specific lighting features in your scene. The options are\+:
\begin{DoxyItemize}
\item {\bfseries{Nothing}}\+: Shortcut to disable all flags.
\item {\bfseries{Everything}}\+: Shortcut to enable all flags.
\item {\bfseries{Global Illumination}}\+: Indicates whether to render \href{https://docs.unity3d.com/Manual/realtime-gi-using-enlighten.html}{\texttt{ global illumination}}.
\item {\bfseries{Main Light}}\+: Indicates whether the main directional \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8universal_0d14_80_88_2_documentation_0i_2light-component}{Light}{0} contributes to lighting.
\item {\bfseries{Additional Lights}}\+: Indicates whether lights other than the main directional light contribute to lighting.

This section describes how to navigate the {\bfseries{Rendering Debugger}} interface at runtime.

To change the current active item\+:


\begin{DoxyItemize}
\item {\bfseries{Keyboard}}\+: use the arrow keys.
\item {\bfseries{Touch screen}}\+: tap the arrows next to properties.
\item {\bfseries{Xbox controller}}\+: use the Directional pad (D-\/\+Pad).
\item {\bfseries{Play\+Station controller}}\+: use the Directional buttons.
\end{DoxyItemize}

To change the current tab\+:


\begin{DoxyItemize}
\item {\bfseries{Keyboard}}\+: use the Page up and Page down keys (Fn + Up and Fn + Down keys for Mac\+OS).
\item {\bfseries{Touch screen}}\+: tap the arrows next to tab title.
\item {\bfseries{Xbox controller}}\+: use the Left Bumper and Right Bumper.
\item {\bfseries{Play\+Station controller}}\+: use the L1 button and R1 button.
\end{DoxyItemize}

To display the current active item independently of the debug window\+:


\begin{DoxyItemize}
\item {\bfseries{Keyboard}}\+: press the right Shift key.
\item {\bfseries{Xbox controller}}\+: press the X button.
\item {\bfseries{Play\+Station controller}}\+: press the Square button. 
\end{DoxyItemize}
