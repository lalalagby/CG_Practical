A \href{../api/UnityEngine.InputSystem.Gamepad.html}{\texttt{ {\ttfamily Gamepad}}} is narrowly defined as a Device with two thumbsticks, a D-\/pad, and four face buttons. Additionally, gamepads usually have two shoulder and two trigger buttons. Most gamepads also have two buttons in the middle.

A gamepad can have additional Controls, such as a gyro, which the Device can expose. However, all gamepads are guaranteed to have at least the minimum set of Controls described above.

Gamepad support guarantees the correct location and functioning of Controls across platforms and hardware. For example, a PS4 Dual\+Shock controller layout should look identical regardless of which platform it is supported on. A gamepad\textquotesingle{}s south face button should always be the lowermost face button.

\texorpdfstring{$>$}{>}NOTE\+: Generic \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_h_i_d}{HID}{0} gamepads will {\bfseries{not}} be surfaced as \href{../api/UnityEngine.InputSystem.Gamepad.html}{\texttt{ {\ttfamily Gamepad}}} devices but rather be created as generic \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_joystick}{joysticks}{0}. This is because the Input \doxylink{namespace_system}{System} cannot guarantee correct mapping of buttons and axes on the controller (the information is simply not available at the HID level). Only HID gamepads that are explicitly supported by the Input \doxylink{namespace_system}{System} (like the PS4 controller) will come out as gamepads. Note that you can set up the same kind of support for specific HID gamepads yourself (see \href{./HID.md\#creating-a-custom-device-layout}{\texttt{ "{}\+Overriding the HID Fallback"{}}}).

On Windows (XInput controllers only), Universal Windows Platform (UWP), and Switch, \doxylink{namespace_unity}{Unity} polls gamepads explicitly rather than deliver updates as events.

You can control polling frequency manually. The default polling frequency is 60 Hz. Use \href{../api/UnityEngine.InputSystem.InputSystem.html\#UnityEngine_InputSystem_InputSystem_pollingFrequency}{\texttt{ {\ttfamily Input\+System.\+polling\+Frequency}}} to get or set the frequency.
