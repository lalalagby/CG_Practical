Visual Scripting supports live editing. In live editing you can create and edit graphs while in play mode and see the Visual Scripting updates in real time. This provides a fast way to iterate and test ideas without the need to recompile project changes.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-live_autotoc_md5393}{}\doxysection{\texorpdfstring{Working in live edit}{Working in live edit}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-live_autotoc_md5393}
Live editing is not limited to tweaking values â€” you can add and remove nodes, connections (edges), etc while live. Anything that can be done in a normal edit, can be done in a live edit.

In accordance with the \doxylink{namespace_unity}{Unity} convention\+:


\begin{DoxyItemize}
\item Changes made to embeds are reverted when you exit play mode -\/ the changes live inside components.
\item Changes made to graphs are saved when you exit play mode -\/ the changes live inside assets
\end{DoxyItemize}

\begin{DoxyRemark}{Remarks}
To preserve the changes made to a component graph, copy the modified nodes before exiting play mode. You\textquotesingle{}ll then be able to paste back while in edit mode.
\end{DoxyRemark}
When in live mode, Visual Scripting is displayed the flow as droplets on connections.
