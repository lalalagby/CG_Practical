Pressure-\/level of the touch against the touchscreen. 

Pressure of touch.

The core range for this value is \mbox{[}0..1\mbox{]} with 1 indicating maximum pressure. Note, however, that the actual value may go beyond 1 in practice. This is because the system will usually define "{}maximum pressure"{} to be less than the physical maximum limit the hardware is capable of reporting so that to achieve maximum pressure, one does not need to press as hard as possible.

Numeric ID of the touch.

While a touch is ongoing, it must have a non-\/zero ID different from all other ongoing touches. Starting with \doxylink{namespace_unity_engine_1_1_input_system_a586a6521911c62d23469e51a336d29dba7b81520329371177ede3ccbc307bc85d}{Touch\+Phase.\+Began} and ending with \doxylink{namespace_unity_engine_1_1_input_system_a586a6521911c62d23469e51a336d29dba0157fbb99be9622bdaa6e544270c0622}{Touch\+Phase.\+Ended} or \doxylink{namespace_unity_engine_1_1_input_system_a586a6521911c62d23469e51a336d29dba0e22fe7d45f8e5632a4abf369b24e29c}{Touch\+Phase.\+Canceled}, a touch is identified by its ID, i.\+e. a \doxylink{struct_unity_engine_1_1_input_system_1_1_low_level_1_1_touch_state}{Touch\+State} with the same ID belongs to the same touch.

After a touch has ended or been canceled, an ID can be reused.
