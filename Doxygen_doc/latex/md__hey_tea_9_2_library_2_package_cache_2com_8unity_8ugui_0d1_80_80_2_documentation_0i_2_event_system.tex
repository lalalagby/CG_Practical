 The Event \doxylink{namespace_system}{System} is a way of sending events to objects in the application based on input, be it keyboard, mouse, touch, or custom input. The Event \doxylink{namespace_system}{System} consists of a few components that work together to send events.

When you add an Event \doxylink{namespace_system}{System} component to a Game\+Object you will notice that it does not have much functionality exposed, this is because the Event \doxylink{namespace_system}{System} itself is designed as a manager and facilitator of communication between Event \doxylink{namespace_system}{System} modules.

The primary roles of the Event \doxylink{namespace_system}{System} are as follows\+:


\begin{DoxyItemize}
\item Manage which Game\+Object is considered selected
\item Manage which Input Module is in use
\item Manage Raycasting (if required)
\item Updating all Input Modules as required
\end{DoxyItemize}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2_event_system_autotoc_md4804}{}\doxysection{\texorpdfstring{Input Modules}{Input Modules}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2_event_system_autotoc_md4804}
An Input Module is where the main logic of how you want the Event \doxylink{namespace_system}{System} to behave lives, they are used for\+:


\begin{DoxyItemize}
\item Handling Input
\item Managing event state
\item Sending events to scene objects.
\end{DoxyItemize}

Only one Input Module can be active in the Event \doxylink{namespace_system}{System} at a time, and they must be components on the same Game\+Object as the Event \doxylink{namespace_system}{System} component.

If you want to write a custom Input Module, send events supported by existing UI components in \doxylink{namespace_unity}{Unity}. To extend and write your own events, see the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2_messaging_system}{Messaging System}{0} documentation.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2_event_system_autotoc_md4805}{}\doxysection{\texorpdfstring{Raycasters}{Raycasters}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2_event_system_autotoc_md4805}
Raycasters are used for figuring out what the pointer is over. It is common for Input Modules to use the Raycasters configured in the Scene to calculate what the pointing device is over.

There are 3 provided Raycasters that exist by default\+:


\begin{DoxyItemize}
\item \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2script-_graphic_raycaster}{Graphic Raycaster}{0} -\/ Used for UI elements
\item \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2script-_physics2_d_raycaster}{Physics 2D Raycaster}{0} -\/ Used for 2D physics elements
\item \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8ugui_0d1_80_80_2_documentation_0i_2script-_physics_raycaster}{Physics Raycaster}{0} -\/ Used for 3D physics elements
\end{DoxyItemize}

If you have a 2d / 3d Raycaster configured in your Scene, it is easy to make non-\/\+UI elements receive messages from the Input Module. Simply attach a script that implements one of the event interfaces. For examples of this, see the \href{xref:UnityEngine.EventSystems.IPointerEnterHandler}{\texttt{ IPointer\+Enter\+Handler}} and \href{xref:UnityEngine.EventSystems.IPointerClickHandler}{\texttt{ IPointer\+Click\+Handler}} Scripting Reference pages.
