The Subgraph node is in the {\bfseries{Nesting}} category in the fuzzy finder.

You can go to the {\bfseries{Graphs}} category and select any Script Graph to create a Subgraph node. For more information on how to create a Subgraph, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-nesting-add-subgraph}{Add a Subgraph to a Script Graph}{0}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-nesting-subgraph-node_autotoc_md5426}{}\doxysection{\texorpdfstring{Available ports}{Available ports}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-nesting-subgraph-node_autotoc_md5426}
By default, a Subgraph node has no ports.

Use the \href{vs-interface-overview.md\#the-graph-inspector}{\texttt{ Graph Inspector}} to specify the following on a Script Graph\+:


\begin{DoxyItemize}
\item Trigger Inputs.
\item Trigger Outputs.
\item Data Inputs.
\item Data Outputs.
\end{DoxyItemize}

These determine the type and number of ports available on its Subgraph node. For more information on how to define ports on a Script Graph, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_06f95905303333ea92719076820d2246b}{Add a Trigger or Data port to a Script Graph}{0}.

\begin{DoxyRemark}{Remarks}
A Subgraph node can use a new blank Script Graph or an existing Script Graph from a project. For more information, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-nesting-add-subgraph}{Add a Subgraph to a Script Graph}{0}.
\end{DoxyRemark}
In the following example, the Subgraph node {\bfseries{Character Move}} references a graph that makes a Game\+Object move based on a user\textquotesingle{}s input. It has the following\+:


\begin{DoxyItemize}
\item One Trigger Input port.
\item One Trigger Output port.
\item Three Data Input ports.
\item One Data Output port.
\end{DoxyItemize}

After every Update Event in the application, the {\bfseries{Character Move}} Subgraph node triggers and takes the X, Y, and Z coordinates of the current Game\+Object\textquotesingle{}s Transform component. The Subgraph node then outputs a new Vector 3 value, which the parent graph assigns to the current Game\+Object with a Transform Set Position node. ~\newline




The Subgraph node reduces the number of nodes in the parent graph.
