 The Calculate Level of Detail Texture 2D node takes an input {\bfseries{Texture 2D}} and outputs the mip level of a Texture sample. This node is useful in situations where you need to know the mip level of a Texture, such as when you might want to modify the mip level before sampling in your shader.



The Calculate Level of Detail Texture 2D node also has a clamped and unclamped mode\+:


\begin{DoxyItemize}
\item {\bfseries{Clamped}}\+: The node clamps the returned mip level to the actual mips available on the Texture. The node uses the \href{https://docs.microsoft.com/en-us/windows/win32/direct3dhlsl/dx-graphics-hlsl-to-calculate-lod}{\texttt{ Calculate\+Level\+Of\+Detail}} HLSL intrinsic function. Use this mode when you want to know which mip to sample your Texture from and restrict the result to an existing mip.
\item {\bfseries{Unclamped}}\+: The node returns the ideal mip level, based on an idealized Texture with all its mips present. The node uses the \href{https://docs.microsoft.com/en-us/windows/win32/direct3dhlsl/dx-graphics-hlsl-to-calculate-lod-unclamped}{\texttt{ Calculate\+Level\+Of\+Detail\+Unclamped}} HLSL intrinsic function. Use this mode when you need a more generic value for your mip level.
\end{DoxyItemize}

For example, a Texture might only have 3 mips\+: a 64×64 mip, a 32×32 mip, and a 16×16 mip. When you use the Calculate Level Of Detail Texture 2D node in its {\bfseries{Clamped}} mode, the node restricts the {\bfseries{LOD}} output to one of the 3 mips on the Texture, even if the ideal mip level might be a smaller resolution, such as an 8×8 version. In its {\bfseries{Unclamped}} mode, the node outputs the ideal 8×8 mip level, even though it doesn\textquotesingle{}t exist on the Texture.

\begin{DoxyNote}{Note}
On platforms where these HLSL functions don\textquotesingle{}t exist, Shader Graph determines an appropriate approximation to use, instead.
\end{DoxyNote}
