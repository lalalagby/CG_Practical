\item Has Script Graph

The Set Script Graph node has one required input parameter, called {\bfseries{Container Type}}, which is set using the dropdown in the node\textquotesingle{}s header. The {\bfseries{Container Type}} specifies what component type the node should expect as an input for its {\bfseries{Target}}\+:


\begin{DoxyItemize}
\item If you choose {\bfseries{Game\+Object}}, the node expects to receive a Game\+Object, and assigns the graph to the first Script Machine attached to that Game\+Object.
\item If you choose {\bfseries{Script Machine}}, the node expects to receive a Script Machine, and you can specify the exact Script Machine where you want to set your Script Graph.
\end{DoxyItemize}

Depending on which {\bfseries{Container Type}} you select, the icon displayed next to the {\bfseries{Target}} input port on the node changes\+:

The node has three input ports, located on the left side. The first port, {\bfseries{Enter}}, connects to the node that should start the execution of the Set Script Graph node.

The other two ports collect the Set Script Graph node\textquotesingle{}s required input data\+:


\begin{DoxyItemize}
\item The {\bfseries{Target}}, or the Game\+Object or Script Machine where you want to set a Script Graph.
\item The {\bfseries{Graph}}, or the Script Graph to assign to the Game\+Object or Script Machine.
\end{DoxyItemize}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-scripts-reference_autotoc_md5563}{}\doxysubsection{\texorpdfstring{Outputs}{Outputs}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-scripts-reference_autotoc_md5563}
The Set Script Graph node has two output ports, located on the right side.

The first port, {\bfseries{Exit}}, establishes the connection to the node that should execute after the Set Script Graph node has finished.

The Has Script Graph node returns {\ttfamily true} if it finds the specified Script Graph. Otherwise, it returns {\ttfamily false}.

You can use a control node connected to the Has Script Graph\textquotesingle{}s output port to change what your script does next, based on the result from Has Script Graph. The Get Script Graph node is a data node. It can\textquotesingle{}t control any logic in your script, and is only used to return data.

The node has a single input port, located on the left side, which collects the node\textquotesingle{}s required input data\+:


\begin{DoxyItemize}
\item The Game\+Object where the node should retrieve the Script Graph. You can choose a specific Game\+Object, or leave the default selection as {\bfseries{This}} to use the Game\+Object where your script is currently running.
\end{DoxyItemize}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-scripts-reference_autotoc_md5569}{}\doxysubsection{\texorpdfstring{Outputs}{Outputs}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-scripts-reference_autotoc_md5569}
The Get Script Graph node has a single output port, located on the right side. The output port returns the Game\+Object\textquotesingle{}s first set Script Graph, or {\ttfamily null}, if there is no set Script Graph.

\begin{DoxyNote}{Note}
The Get Script Graph node returns only the first Script Graph set on a Game\+Object. To return all Script Graphs set on a Game\+Object, use the Get Script Graphs node.
\end{DoxyNote}
