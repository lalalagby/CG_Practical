Formula is a powerful node that evaluates logical and mathematical expressions directly via a textual Formula and a list of Arguments.

\begin{DoxyImportant}{Important}
Due to the binary tree traversal overhead (despite caching attempts), performance in using the formula node is significantly slower than using the operator nodes individually. It is preferable to avoid using this node at every frame.
\end{DoxyImportant}
Although a formula can return either a boolean for logic, or a number for math, formulas can return any type of value.



The first text field in the header is the formula itself.

The second text field is the number of arguments. It\textquotesingle{}s set to 2 by default, giving us A and B as inputs. Formulas can have up to 10 arguments, which are always ordered alphabetically. If more are required, they are called B, C, D, E, and so forth.

For example, this formula returns a boolean indicating\+:


\begin{DoxyItemize}
\item whether, at a minimum, 10 seconds have elapsed since the start of the game and
\item the current object\textquotesingle{}s name is \doxylink{class_player}{Player}.
\end{DoxyItemize}

Variable names can be directly used in the formula. For example, a graph variable named health can return a boolean just by typing the formula health \texorpdfstring{$>$}{>} 50. The argument names are evaluated in the following order of priority\+:


\begin{DoxyEnumerate}
\item Alphabetical argument names (a -\/ z)
\item Graph variable names
\item Object variable names
\item Scene variable names
\item Application variable names
\item Saved variable names
\end{DoxyEnumerate}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-formula_autotoc_md5361}{}\doxysubsection{\texorpdfstring{Properties and Methods}{Properties and Methods}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-formula_autotoc_md5361}
Retrieve the value of a property on an argument or variable by using the \mbox{[}arg.\+prop\mbox{]} notation. For example, if position is a Vector 3 object variable, check if it is equal to zero with\+: \mbox{[}position.\+x\mbox{]} = 0. Get the return value of parameterless methods with the \mbox{[}arg.\+Method()\mbox{]} notation.

Note\+: Accessing properties and methods is not guaranteed to be compatible with AOT platforms, because the AOT pre-\/build cannot generate stubs for members that are only accessed by name.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-formula_autotoc_md5362}{}\doxysubsection{\texorpdfstring{Literals}{Literals}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0i_2vs-formula_autotoc_md5362}
