Unity\+Engine.\+Input\+System.\+IInput\+Action\+Collection.\+devices\hspace{0.3cm}{\ttfamily [get]}, {\ttfamily [set]}}



Devices to use with the actions in this collection. 

REVIEW\+: should this allow restricting to a set of controls instead of confining it to just devices?

If this is set, actions in the collection will exclusively bind to devices in the given list. For example, if two gamepads are present in the system yet only one gamepad is listed here, then a "{}<\+Gamepad>/left\+Stick"{} binding will only bind to the gamepad in the list and not to the one that is only available globally.

Modifying this property after bindings in the collection have already been resolved, will lead to \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_a8e3f5d3329be13ca8474314137045af4}{Input\+Action.\+controls} getting refreshed. If any of the actions in the collection are currently in progress (see \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_a86bd545c9dfe1fb0200e296c44389e57}{Input\+Action.\+phase}), the actions will remain unaffected and in progress except if the controls currently driving them (see \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_a472cf9d7e67051b780bca5df73c7bb84}{Input\+Action.\+active\+Control}) are no longer part of any of the selected devices. In that case, the action is \doxylink{class_unity_engine_1_1_input_system_1_1_input_action_a905374d6d7af6dc57a54acfa22018976}{Input\+Action.\+canceled}.
