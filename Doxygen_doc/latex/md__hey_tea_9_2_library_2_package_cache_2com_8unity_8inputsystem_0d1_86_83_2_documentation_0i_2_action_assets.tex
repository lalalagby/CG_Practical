An Input Action Asset is an Asset which contains \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_actions}{Input Actions}{0} and their associated \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_action_bindings}{Bindings}{0} and \href{ActionBindings.md\#control-schemes}{\texttt{ Control Schemes}}. These Assets have the {\ttfamily .inputactions} file extension and are stored in a plain JSON format.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_action_assets_autotoc_md1225}{}\doxysection{\texorpdfstring{Creating Input Action Assets}{Creating Input Action Assets}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_action_assets_autotoc_md1225}
To create an Asset that contains \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_actions}{Input Actions}{0} in \doxylink{namespace_unity}{Unity}, right-\/click in the {\bfseries{Project}} window or go to {\bfseries{Assets \texorpdfstring{$>$}{>} Create \texorpdfstring{$>$}{>} Input Actions}} from \doxylink{namespace_unity}{Unity}\textquotesingle{}s main menu.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_action_assets_autotoc_md1226}{}\doxysection{\texorpdfstring{Editing Input Action Assets}{Editing Input Action Assets}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_action_assets_autotoc_md1226}
To bring up the Action editor, double-\/click an {\ttfamily .inputactions} Asset in the Project Browser, or select the {\bfseries{Edit Asset}} button in the Inspector for that Asset. You can have more than one editor window open at the same time, but not for the same Asset.



The Action editor appears as a separate window, which you can also dock into \doxylink{namespace_unity}{Unity}\textquotesingle{}s main UI.

\texorpdfstring{$>$}{>}{\bfseries{Note}}\+: For details about how Action Maps, Actions, and Bindings work, see documentation on \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_actions}{Actions}{0}.

By default, \doxylink{namespace_unity}{Unity} doesn\textquotesingle{}t save edits you make in the Action Asset window when you save the Project. To save your changes, select {\bfseries{Save Asset}} in the window\textquotesingle{}s toolbar. To discard your changes, close the window and choose {\bfseries{Don\textquotesingle{}t Save}} when prompted. Alternatively, you can toggle auto-\/saving on by enabling the {\bfseries{Auto-\/\+Save}} checkbox in the toolbar. This saves any changes to that Asset.

\texorpdfstring{$>$}{>}{\bfseries{Note}}\+: This setting affects all {\ttfamily .inputactions} Assets, and persists across \doxylink{namespace_unity}{Unity} Editor sessions.

The Action editor window is divided into three panes\+:


\begin{DoxyEnumerate}
\item The left pane lists the Action Maps in the Asset. Each Action Map is a collection of Actions that you can enable or disable in bulk.
\item The middle pane contains the Actions in the currently selected Action Map, and the bindings associated with each Action.
\item The right pane contains the properties of the currently selected Action or Binding.
\end{DoxyEnumerate}

If you select a Binding, you can edit its properties in the right-\/hand pane of the window\+:

\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_action_assets_autotoc_md1230}{}\doxysubsubsection{\texorpdfstring{Picking Controls}{Picking Controls}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_action_assets_autotoc_md1230}
The most important property of any Binding is the \href{Controls.md\#control-paths}{\texttt{ control path}} it\textquotesingle{}s bound to. To edit it, open the {\bfseries{Path}} drop-\/down list. This displays a Control picker window.



In the Control picker window, you can explore a tree of Input Devices and Controls that the Input \doxylink{namespace_system}{System} recognizes, and bind to these Controls. \doxylink{namespace_unity}{Unity} filters this list by the Action\textquotesingle{}s \href{../api/UnityEngine.InputSystem.InputAction.html\#UnityEngine_InputSystem_InputAction_expectedControlType}{\texttt{ {\ttfamily Control Type}}} property. For example, if the Control type is {\ttfamily Vector2}, you can only select a Control that generates two-\/dimensional values, like a stick.

The Device and Control tree is organized hierarchically from generic to specific. For example, the {\bfseries{Gamepad}} Control path {\ttfamily \texorpdfstring{$<$}{<}Gamepad\texorpdfstring{$>$}{>}/button\+South} matches the lower action button on any gamepad. Alternatively, if you navigate to {\bfseries{Gamepad}} \texorpdfstring{$>$}{>} {\bfseries{More Specific Gamepads}} and select {\bfseries{PS4 Controller}}, and then choose the Control path {\ttfamily \texorpdfstring{$<$}{<}Dual\+Shock\+Gamepad\texorpdfstring{$>$}{>}/button\+South}, this only matches the "{}\+Cross"{} button on Play\+Station gamepads, and doesn\textquotesingle{}t match any other gamepads.

Instead of browsing the tree to find the Control you want, it\textquotesingle{}s easier to let the Input \doxylink{namespace_system}{System} listen for input. To do that, select the {\bfseries{Listen}} button. At first, the list of Controls is empty. Once you start pressing buttons or actuating Controls on the Devices you want to bind to, the Control picker window starts listing any Bindings that match the controls you pressed. Select any of these Bindings to view them.

Finally, you can choose to manually edit the Binding path, instead of using the Control picker. To do that, select the {\bfseries{T}} button next to the Control path popup. This changes the popup to a text field, where you can enter any Binding string. This also allows you to use wildcard ({\ttfamily \texorpdfstring{$\ast$}{*}}) characters in your Bindings. For example, you can use a Binding path such as {\ttfamily \texorpdfstring{$<$}{<}Touchscreen\texorpdfstring{$>$}{>}/touch\texorpdfstring{$\ast$}{*}/press} to bind to any finger being pressed on the touchscreen, instead of manually binding to {\ttfamily \texorpdfstring{$<$}{<}Touchscreen\texorpdfstring{$>$}{>}/touch0/press}, {\ttfamily \texorpdfstring{$<$}{<}Touchscreen\texorpdfstring{$>$}{>}/touch1/press} and so on.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_action_assets_autotoc_md1231}{}\doxysubsubsection{\texorpdfstring{Editing Composite Bindings}{Editing Composite Bindings}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_action_assets_autotoc_md1231}
Composite Bindings are Bindings consisting of multiple parts, which form a Control together. For instance, a \href{ActionBindings.md\#2d-vector}{\texttt{ 2D Vector Composite}} uses four buttons (left, right, up, down) to simulate a 2D stick input. See the \href{ActionBindings.md\#composite-bindings}{\texttt{ Composite Bindings}} documentation to learn more.



To create a Composite Binding, in the Input Action Asset editor window, select the Add (+) icon on the Action you want to add it to, and select the Composite Binding type from the popup menu.

\texorpdfstring{$>$}{>}{\bfseries{Important}}\+: The set of Composites displayed in the menu is filtered based on the value type of the Action. This means that, for example, if the Action is set to type "{}\+Button"{}, then only Composites able of returning values of type {\ttfamily float} will be shown.



This creates multiple Binding entries for the Action\+: one for the Composite as a whole, and then, one level below that, one for each Composite part. The Composite itself doesn\textquotesingle{}t have a Binding path property, but its individual parts do, and you can edit these parts like any other Binding. Once you bind all the Composite\textquotesingle{}s parts, the Composite can work together as if you bound a single control to the Action.

To change the type of a Composite retroactively, select the Composite, then select the new type from the {\bfseries{Composite Type}} drop-\/down in the {\bfseries{Properties}} pane.



To can change the part of the Composite to which a particular Binding is assigned, use the {\bfseries{Composite Part}} drop-\/down in the Binding\textquotesingle{}s properties.



Multiple Bindings can be assigned to the same part. You can also duplicate individual part Bindings\+: right-\/click the Binding, then select {\bfseries{Duplicate}} to create new part Bindings for the Composite. This can be used, for example, to create a single Composite for both "{}\+WASD"{} style controls and arrow keys.

\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_action_assets_autotoc_md1232}{}\doxysubsection{\texorpdfstring{Editing Control Schemes}{Editing Control Schemes}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_action_assets_autotoc_md1232}
Input Action Assets can have multiple \href{ActionBindings.md\#control-schemes}{\texttt{ Control Schemes}}, which let you enable or disable different sets of Bindings for your Actions for different types of Devices.



To see the Control Schemes in the Input Action Asset editor window, open the Control Scheme drop-\/down list in the top left of the window. This menu lets you add or remove Control Schemes to your Asset. If the Asset contains any Control Schemes, you can select a Control Scheme, and then the window only shows bindings that belong to that Scheme. If you select a binding, you can now pick the Control Schemes for which this binding should be active in the {\bfseries{Properties}} view to the left of the window. When you add a new Control Scheme, or select an existing Control Scheme, and then select {\bfseries{Edit Control Scheme…}}, you can edit the name of the Control Scheme and which devices the Scheme should be active for.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_action_assets_autotoc_md1233}{}\doxysection{\texorpdfstring{Using Input Action Assets}{Using Input Action Assets}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_action_assets_autotoc_md1233}

One of the most convenient ways to work with {\ttfamily .inputactions} Assets in scripts is to automatically generate a C\# wrapper class for them. This removes the need to manually look up Actions and Action Maps using their names, and also provides an easier way to set up callbacks.

To enable this option, tick the {\bfseries{Generate C\# Class}} checkbox in the importer properties in the Inspector of the {\ttfamily .inputactions} Asset, then select {\bfseries{Apply}}.



You can optionally choose a path name, class name, and namespace for the generated script, or keep the default values.

This generates a C\# script that simplifies working with the Asset.

The \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_player_input}{Player Input}{0} component provides a convenient way to handle input for one or multiple players. It requires you to set up all your Actions in an Input Action Asset, which you can then assign to the \doxylink{class_player}{Player} Input component. The \doxylink{class_player}{Player} Input component can then automatically handle activating Action Maps and selecting Control Schemes for you.

There are several ways to modify an Input Action Asset at runtime. Any modifications that you make during Play mode to an Input Action Asset do not persist in the Input Action Asset after you exit Play mode. This means you can test your application in a realistic manner in the Editor without having to worry about inadvertently modifying the asset. For examples on how to modify an Input Action Asset, see the documentation on \href{Actions.md\#creating-actions-in-code}{\texttt{ Creating Actions in code}} and \href{ActionBindings.md\#changing-bindings}{\texttt{ Changing Bindings}}.
