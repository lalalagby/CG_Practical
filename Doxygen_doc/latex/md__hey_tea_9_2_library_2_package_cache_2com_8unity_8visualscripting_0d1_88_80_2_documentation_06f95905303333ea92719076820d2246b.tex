 A Script Graph used as a Subgraph can receive data and logic from its parent graph. Add and define ports on a graph to choose what data graphs can send and receive.

For more information about Subgraphs, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_0b052f136142546305a9d053d2653f106}{Subgraphs and State Units}{0}.

To add a Trigger Input, Trigger Output, Data Input, or Data Output port to a Script Graph\+:


\begin{DoxyEnumerate}
\item 

\href{vs-open-graph-edit.md}{\texttt{ Open the Script Graph}} you want to edit in the Graph window.


\item 

With no nodes or groups selected in the graph, open the \href{vs-interface-overview.md\#the-graph-inspector}{\texttt{ Graph Inspector}}.


\item 

Select {\bfseries{Add}} (+) under the port type you want to add\+:


\begin{DoxyItemize}
\item 

{\bfseries{Trigger Inputs}}


\item 

{\bfseries{Trigger Outputs}}


\item 

{\bfseries{Data Inputs}}


\item 

{\bfseries{Data Outputs}}


\end{DoxyItemize}
\item 

In the {\bfseries{Key}} field, enter a unique key name for the port. The Key value can\textquotesingle{}t match the Key of any existing ports on the current Script Graph.

\doxysubsubsubsubsection*{NOTE}



If two {\bfseries{Key}} values are the same on the same graph, Visual Scripting ignores the second port definition and displays a warning in the Graph Inspector. If you change the {\bfseries{Key}} value for a port after you\textquotesingle{}ve made a connection to that port in a graph, the connections break and you must reconnect them.

 
\item 

In the {\bfseries{Label}} field, enter a label to display for the port. The label displays on the Subgraph node and its Input or Output node.

\doxysubsubsubsubsection*{NOTE}



If you don\textquotesingle{}t set a {\bfseries{Label}}, Visual Scripting uses the value from the {\bfseries{Key}} field.

 
\item 

In the {\bfseries{Summary}} field, enter a brief summary of the port to display in the Graph Inspector when you select the Subgraph node, Input node, or Output node.


\item 

Toggle {\bfseries{Hide Label}} to do the following\+:


\begin{DoxyItemize}
\item 

Enable {\bfseries{Hide Label}} to hide the port label on any Subgraph node, Input node, or Output node.


\item 

Disable {\bfseries{Hide Label}} to display the data from the {\bfseries{Label}} field.


\end{DoxyItemize}
\item 

(Data Inputs and Data Outputs Only) Set a data type for the port\+:


\begin{DoxyEnumerate}[label=\enumalphalphcnt*]

\item 

Select the {\bfseries{Type}} list to open the Type menu.


\item 

Select a data type from the list to set the data type the port accepts.


\end{DoxyEnumerate}
\item 

(Data Inputs Only) Enable {\bfseries{Has Default Value}} to display the {\bfseries{Default Value}} field. Disable {\bfseries{Has Default Value}} to hide the {\bfseries{Default Value}} field.


\begin{DoxyEnumerate}[label=\enumalphalphcnt*]

\item 

In the {\bfseries{Default Value}} field, enter the default value the port uses if it doesn\textquotesingle{}t receive a data input while the Script Graph runs.


\end{DoxyEnumerate}
\end{DoxyEnumerate}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_06f95905303333ea92719076820d2246b_autotoc_md5406}{}\doxysection{\texorpdfstring{Add ports with Input and Output nodes}{Add ports with Input and Output nodes}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8visualscripting_0d1_88_80_2_documentation_06f95905303333ea92719076820d2246b_autotoc_md5406}

\begin{DoxyEnumerate}
\item 

\href{vs-open-graph-edit.md}{\texttt{ Open the Script Graph}} you want to edit in the Graph window.


\item

\item 

Go to {\bfseries{Nesting}}.


\item 

Do one of the following\+:


\begin{DoxyItemize}
\item

\item 

To add a Trigger Output or Data Output port to the graph, select {\bfseries{Output}}.


\end{DoxyItemize}
\item 

Select the new Input or Output node in the graph.


\item 

Open the \href{vs-interface-overview.md\#the-graph-inspector}{\texttt{ Graph Inspector}}.


\item

\doxysubsubsubsubsection*{NOTE}



If you don\textquotesingle{}t set a {\bfseries{Label}}, Visual Scripting uses the value from the {\bfseries{Key}} field.

 
\item 

In the {\bfseries{Summary}} field, enter a brief summary of the port to display in the Graph Inspector when you select the Subgraph node, Input node, or Output node.


\item 

Toggle {\bfseries{Hide Label}} to do the following\+:


\begin{DoxyItemize}
\item 

Enable {\bfseries{Hide Label}} to hide the port label on any Subgraph node, Input node, or Output node.


\item 

Disable {\bfseries{Hide Label}} to display the data from the {\bfseries{Label}} field.


\end{DoxyItemize}
\item
