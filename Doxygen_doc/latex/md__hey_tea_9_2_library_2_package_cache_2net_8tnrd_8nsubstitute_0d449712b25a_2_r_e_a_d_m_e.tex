\doxylink{namespace_n_substitute}{NSubstitute} is designed as a friendly substitute for .NET mocking libraries.

It is an attempt to satisfy our craving for a mocking library with a succinct syntax that helps us keep the focus on the intention of our tests, rather than on the configuration of our test doubles. We\textquotesingle{}ve tried to make the most frequently required operations obvious and easy to use, keeping less usual scenarios discoverable and accessible, and all the while maintaining as much natural language as possible.

Perfect for those new to testing, and for others who would just like to to get their tests written with less noise and fewer lambdas.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2net_8tnrd_8nsubstitute_0d449712b25a_2_r_e_a_d_m_e_autotoc_md5721}{}\doxysubsubsection{\texorpdfstring{Installation}{Installation}}\label{md__hey_tea_9_2_library_2_package_cache_2net_8tnrd_8nsubstitute_0d449712b25a_2_r_e_a_d_m_e_autotoc_md5721}

Let\textquotesingle{}s say we have a basic calculator interface\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{public}\ \textcolor{keyword}{interface\ }ICalculator}
\DoxyCodeLine{\{}
\DoxyCodeLine{\ \ \ \ \textcolor{keywordtype}{int}\ Add(\textcolor{keywordtype}{int}\ a,\ \textcolor{keywordtype}{int}\ b);}
\DoxyCodeLine{\ \ \ \ \textcolor{keywordtype}{string}\ Mode\ \{\ \textcolor{keyword}{get};\ \textcolor{keyword}{set};\ \}}
\DoxyCodeLine{\ \ \ \ \textcolor{keyword}{event}\ Action\ PoweringUp;}
\DoxyCodeLine{\}}

\end{DoxyCode}


We can ask \doxylink{namespace_n_substitute}{NSubstitute} to create a substitute instance for this type. We could ask for a stub, mock, fake, spy, test double etc., but why bother when we just want to substitute an instance we have some control over?


\begin{DoxyCode}{0}
\DoxyCodeLine{\_calculator\ =\ Substitute.For<ICalculator>();}

\end{DoxyCode}


⚠️ {\bfseries{Note}}\+: \doxylink{namespace_n_substitute}{NSubstitute} will only work properly with interfaces or with {\ttfamily virtual} members of classes. Be careful substituting for classes with non-\/virtual members. See \href{https://nsubstitute.github.io/help/creating-a-substitute/\#substituting_infrequently_and_carefully_for_classes}{\texttt{ Creating a substitute}} for more information.

Now we can tell our substitute to return a value for a call\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\_calculator.Add(1,\ 2).Returns(3);}
\DoxyCodeLine{Assert.That(\_calculator.Add(1,\ 2),\ Is.EqualTo(3));}

\end{DoxyCode}


We can check that our substitute received a call, and did not receive others\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\_calculator.Add(1,\ 2);}
\DoxyCodeLine{\_calculator.Received().Add(1,\ 2);}
\DoxyCodeLine{\_calculator.DidNotReceive().Add(5,\ 7);}

\end{DoxyCode}


If our Received() assertion fails, \doxylink{namespace_n_substitute}{NSubstitute} tries to give us some help as to what the problem might be\+:

\begin{DoxyVerb}NSubstitute.Exceptions.ReceivedCallsException : Expected to receive a call matching:
    Add(1, 2)
Actually received no matching calls.
Received 2 non-matching calls (non-matching arguments indicated with '*' characters):
    Add(1, *5*)
    Add(*4*, *7*)
\end{DoxyVerb}
 We can also work with properties using the Returns syntax we use for methods, or just stick with plain old property setters (for read/write properties)\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\_calculator.Mode.Returns(\textcolor{stringliteral}{"{}DEC"{}});}
\DoxyCodeLine{Assert.That(\_calculator.Mode,\ Is.EqualTo(\textcolor{stringliteral}{"{}DEC"{}}));}
\DoxyCodeLine{}
\DoxyCodeLine{\_calculator.Mode\ =\ \textcolor{stringliteral}{"{}HEX"{}};}
\DoxyCodeLine{Assert.That(\_calculator.Mode,\ Is.EqualTo(\textcolor{stringliteral}{"{}HEX"{}}));}

\end{DoxyCode}


\doxylink{namespace_n_substitute}{NSubstitute} supports argument matching for setting return values and asserting a call was received\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\_calculator.Add(10,\ -\/5);}
\DoxyCodeLine{\_calculator.Received().Add(10,\ Arg.Any<\textcolor{keywordtype}{int}>());}
\DoxyCodeLine{\_calculator.Received().Add(10,\ Arg.Is<\textcolor{keywordtype}{int}>(x\ =>\ x\ <\ 0));}

\end{DoxyCode}


We can use argument matching as well as passing a function to Returns() to get some more behaviour out of our substitute (possibly too much, but that\textquotesingle{}s your call)\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\_calculator}
\DoxyCodeLine{\ \ \ .Add(Arg.Any<\textcolor{keywordtype}{int}>(),\ Arg.Any<\textcolor{keywordtype}{int}>())}
\DoxyCodeLine{\ \ \ .Returns(x\ =>\ (\textcolor{keywordtype}{int})x[0]\ +\ (\textcolor{keywordtype}{int})x[1]);}
\DoxyCodeLine{Assert.That(\_calculator.Add(5,\ 10),\ Is.EqualTo(15));}

\end{DoxyCode}


Returns() can also be called with multiple arguments to set up a sequence of return values.


\begin{DoxyCode}{0}
\DoxyCodeLine{\_calculator.Mode.Returns(\textcolor{stringliteral}{"{}HEX"{}},\ \textcolor{stringliteral}{"{}DEC"{}},\ \textcolor{stringliteral}{"{}BIN"{}});}
\DoxyCodeLine{Assert.That(\_calculator.Mode,\ Is.EqualTo(\textcolor{stringliteral}{"{}HEX"{}}));}
\DoxyCodeLine{Assert.That(\_calculator.Mode,\ Is.EqualTo(\textcolor{stringliteral}{"{}DEC"{}}));}
\DoxyCodeLine{Assert.That(\_calculator.Mode,\ Is.EqualTo(\textcolor{stringliteral}{"{}BIN"{}}));}

\end{DoxyCode}


Finally, we can raise events on our substitutes (unfortunately C\# dramatically restricts the extent to which this syntax can be cleaned up)\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{bool}\ eventWasRaised\ =\ \textcolor{keyword}{false};}
\DoxyCodeLine{\_calculator.PoweringUp\ +=\ ()\ =>\ eventWasRaised\ =\ \textcolor{keyword}{true};}
\DoxyCodeLine{}
\DoxyCodeLine{\_calculator.PoweringUp\ +=\ Raise.Event<Action>();}
\DoxyCodeLine{Assert.That(eventWasRaised);}

\end{DoxyCode}

\doxylink{namespace_n_substitute}{NSubstitute} and its tests can be compiled and run using Visual Studio and Visual Studio for Mac. Note that some tests are marked {\ttfamily \mbox{[}Pending\mbox{]}} and are not meant to pass at present, so it is a good idea to exclude tests in the Pending category from test runs.

There are also build scripts in the {\ttfamily ./build} directory for command line builds, and CI configurations in the project root.

To do \href{https://github.com/nsubstitute/NSubstitute/wiki/Release-procedure}{\texttt{ full builds}} you\textquotesingle{}ll also need Ruby, as the jekyll gem is used to generate the website.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2net_8tnrd_8nsubstitute_0d449712b25a_2_r_e_a_d_m_e_autotoc_md5725}{}\doxysubsubsection{\texorpdfstring{Other libraries you may be interested in}{Other libraries you may be interested in}}\label{md__hey_tea_9_2_library_2_package_cache_2net_8tnrd_8nsubstitute_0d449712b25a_2_r_e_a_d_m_e_autotoc_md5725}
