You can receive callbacks when the active test run, or individual tests, starts and finishes. You can register callbacks by invoking {\ttfamily Register\+Callbacks} on the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2reference-test-runner-api}{Test\+Runner\+Api}{0} with an instance of a class that implements \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2reference-icallbacks}{ICallbacks}{0}. There are four {\ttfamily ICallbacks} methods for the start and finish of both the whole run and each level of the test tree.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2extension-get-test-results_autotoc_md4204}{}\doxysection{\texorpdfstring{Example}{Example}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2extension-get-test-results_autotoc_md4204}

\begin{quote}
{\bfseries{Note}}\+: The registered callbacks are not persisted on domain reloads. So it is necessary to re-\/register the callback after a domain reloads, usually with \href{https://docs.unity3d.com/Manual/RunningEditorCodeOnLaunch.html}{\texttt{ Initialize\+On\+Load}}. \end{quote}


It is possible to provide a {\ttfamily priority} as an integer as the second argument when registering a callback. This influences the invocation order of different callbacks. The default value is zero. It is also possible to provide {\ttfamily Register\+Callbacks} with a class instance that implements \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2reference-ierror-callbacks}{IError\+Callbacks}{0} that is an extended version of {\ttfamily ICallbacks}. {\ttfamily IError\+Callbacks} also has a callback method for {\ttfamily On\+Error} that invokes if the run fails to start, for example, due to compilation errors or if an \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2reference-setup-and-cleanup}{IPrebuild\+Setup}{0} throws an exception.
