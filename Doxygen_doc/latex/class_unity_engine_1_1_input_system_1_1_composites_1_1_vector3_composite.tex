\doxysubsection{Detailed Description}
A 3D vector formed from six floating-\/point inputs. 

Depending on the setting of \doxylink{class_unity_engine_1_1_input_system_1_1_composites_1_1_vector3_composite_af8c5562a121c3cfb8cc82dbb92361a21}{mode}, the vector is either in the \mbox{[}-\/1..1\mbox{]} range on each axis (normalized or not depending on \doxylink{class_unity_engine_1_1_input_system_1_1_composites_1_1_vector3_composite_af8c5562a121c3cfb8cc82dbb92361a21}{mode}) or is in the full value range of the input controls.

Analog&Part controls are treated as analog meaning that the floating-\/point values read from controls will come through as is (minus the fact that the down and left direction values are negated). \\
\hline

\raisebox{\heightof{T}}[0pt][0pt]{\index{DigitalNormalized@{DigitalNormalized}!UnityEngine.InputSystem.Composites.Vector3Composite@{UnityEngine.InputSystem.Composites.Vector3Composite}}\index{UnityEngine.InputSystem.Composites.Vector3Composite@{UnityEngine.InputSystem.Composites.Vector3Composite}!DigitalNormalized@{DigitalNormalized}}}\Hypertarget{class_unity_engine_1_1_input_system_1_1_composites_1_1_vector3_composite_a9c38a2d93c81caa02bde96aa56a9d750a56882c0995e52fa61c7d8959bfdc3b50}\label{class_unity_engine_1_1_input_system_1_1_composites_1_1_vector3_composite_a9c38a2d93c81caa02bde96aa56a9d750} 
Digital\+Normalized&Part controls are treated as buttons (on/off) and the resulting vector is normalized. This means that if, for example, both left and up are pressed, instead of returning a vector (-\/1,1,0), a vector of roughly (-\/0.\+7,0.\+7,0) (that is, corresponding to {\ttfamily new Vector3(-\/1,1,0).normalized}) is returned instead. \\
\hline

Digital&Part controls are treated as buttons (on/off) and the resulting vector is not normalized. This means that if both left and up are pressed, for example, the resulting vector is (-\/1,1,0) and has a length greater than 1. \\
