 When writing tests, it is possible to avoid duplication of code by using the \href{https://docs.nunit.org/articles/nunit/technical-notes/usage/SetUp-and-TearDown.html}{\texttt{ Set\+Up and Tear\+Down}} methods built into \href{http://www.nunit.org/}{\texttt{ NUnit}}. The \doxylink{namespace_unity}{Unity} Test Framework has extended these methods with extra functionality, which can yield commands and skip frames, in the same way as \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_062284b282c4f87cfc2410902afc579cf}{Unity\+Test}{0}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0207711f1e4d22e4fcd7abf617ef2b51d_autotoc_md4231}{}\doxysection{\texorpdfstring{Action execution order}{Action execution order}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0207711f1e4d22e4fcd7abf617ef2b51d_autotoc_md4231}
The actions related to a test run in the following order\+:


\begin{DoxyItemize}
\item Attributes implementing \href{https://docs.nunit.org/articles/nunit/extending-nunit/IApplyToContext-Interface.html}{\texttt{ IApply\+To\+Context}}
\item Any attribute implementing \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0b47efa583a0a44bbd60d9f747ff816c4}{Outer\+Unity\+Test\+Action}{0} has its {\ttfamily Before\+Test} invoked
\item \doxylink{namespace_tests}{Tests} with \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0716d1700c82ca4324abf1039f6677e70}{Unity\+Set\+Up}{0} methods in their test class
\item Attributes implementing \href{https://docs.nunit.org/articles/nunit/extending-nunit/ICommandWrapper-Interface.html}{\texttt{ IWrap\+Set\+Up\+Tear\+Down}}
\item Any method with the \mbox{[}Set\+Up\mbox{]}) attribute
\item \href{https://nunit.org/docs/2.6/actionAttributes.html}{\texttt{ Action attributes}} have their {\ttfamily Before\+Test} method invoked
\item Attributes implementing \href{https://docs.nunit.org/articles/nunit/extending-nunit/ICommandWrapper-Interface.html}{\texttt{ IWrap\+Test\+Method}} ~\newline

\item {\bfseries{The test itself runs}}
\item \href{https://nunit.org/docs/2.6/actionAttributes.html}{\texttt{ Action attributes}} have their {\ttfamily After\+Test} method invoked
\item Any method with the \href{https://docs.nunit.org/articles/nunit/technical-notes/usage/SetUp-and-TearDown.html}{\texttt{ Tear\+Down}} attribute
\item \doxylink{namespace_tests}{Tests} with \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0716d1700c82ca4324abf1039f6677e70}{Unity\+Tear\+Down}{0} methods in their test class
\item Any \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0b47efa583a0a44bbd60d9f747ff816c4}{Outer\+Unity\+Test\+Action}{0} has its {\ttfamily After\+Test} invoked
\end{DoxyItemize}

In {\bfseries{Edit Mode}} tests it is possible to yield instructions that can result in a domain reload, such as entering or exiting {\bfseries{Play Mode}} (see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0a7960e96e42640710807a331b7891678}{Custom yield instructions}{0}). When a domain reload happens, all non-\/\+Unity actions (such as {\ttfamily One\+Time\+Setup} and {\ttfamily Setup}) are rerun before the code, which initiated the domain reload, continues. \doxylink{namespace_unity}{Unity} actions (such as {\ttfamily Unity\+Setup}) are not rerun. If the \doxylink{namespace_unity}{Unity} action is the code that initiated the domain reload, then the rest of the code in the {\ttfamily Unity\+Setup} method runs after the domain reload.
