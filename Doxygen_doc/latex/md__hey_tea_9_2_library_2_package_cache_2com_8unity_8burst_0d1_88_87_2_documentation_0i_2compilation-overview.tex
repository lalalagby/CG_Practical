 \doxylink{namespace_burst}{Burst} compiles your code in different ways depending on its context.


\begin{DoxyItemize}
\item In Play mode, \doxylink{namespace_burst}{Burst} compiles your code \href{https://en.wikipedia.org/wiki/Just-in-time_compilation}{\texttt{ just-\/in-\/time (JIT)}}.
\item When you build and run your application to a player, \doxylink{namespace_burst}{Burst} compiles \href{https://en.wikipedia.org/wiki/Ahead-of-time_compilation}{\texttt{ ahead-\/of-\/time (AOT)}}.
\end{DoxyItemize}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8burst_0d1_88_87_2_documentation_0i_2compilation-overview_autotoc_md304}{}\doxysection{\texorpdfstring{Just-\/in-\/time compilation}{Just-\/in-\/time compilation}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8burst_0d1_88_87_2_documentation_0i_2compilation-overview_autotoc_md304}
While your application is in Play mode in the Editor, \doxylink{namespace_burst}{Burst} compiles your code asynchronously at the point that \doxylink{namespace_unity}{Unity} uses it. This means that your code runs under the default \href{https://docs.unity3d.com/Manual/Mono.html}{\texttt{ Mono compiler}} until \doxylink{namespace_burst}{Burst} completes its work in the background.

To force \doxylink{namespace_unity}{Unity} to compile your code synchronously while in the Editor, see \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8burst_0d1_88_87_2_documentation_0i_2compilation-synchronous}{Synchronous compilation}{0}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8burst_0d1_88_87_2_documentation_0i_2compilation-overview_autotoc_md305}{}\doxysection{\texorpdfstring{Ahead-\/of-\/time compilation}{Ahead-\/of-\/time compilation}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8burst_0d1_88_87_2_documentation_0i_2compilation-overview_autotoc_md305}
When you build your project, \doxylink{namespace_burst}{Burst} compiles all the supported code ahead-\/of-\/time (AOT) into a native library which \doxylink{namespace_unity}{Unity} ships with your application. To control how \doxylink{namespace_burst}{Burst} compiles AOT, use the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8burst_0d1_88_87_2_documentation_0i_2building-aot-settings}{Player Settings window}{0}. Depending on the platform you want to build for, AOT compilation might require access to linker tools.
