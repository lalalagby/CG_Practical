Touch support is divided into\+:
\begin{DoxyItemize}
\item low-\/level support implemented in the \`{}\+Touchscreen\`{} class.
\item high-\/level support implemented in the \`{}\+Enhanced\+Touch.Touch\`{} class.
\end{DoxyItemize}

\texorpdfstring{$>$}{>}{\bfseries{Note}}\+: You should not use \`{}\+Touchscreen\`{} for polling. If you want to read out touches similar to \href{https://docs.unity3d.com/ScriptReference/Input-touches.html}{\texttt{ {\ttfamily Unity\+Engine.\+Input.\+touches}}}, see \`{}\+Enhanced\+Touch\`{}. If you read out touch state from \`{}\+Touchscreen\`{} directly inside of the {\ttfamily Update} or {\ttfamily Fixed\+Update} methods, your app will miss changes in touch state.

Touch input is supported on Android, i\+OS, Windows, and the Universal Windows Platform (UWP).

\texorpdfstring{$>$}{>}{\bfseries{Note}}\+: To test your app on i\+OS or Android in the editor with touch input from your mobile device, you can use the \doxylink{namespace_unity}{Unity} Remote as described \href{Debugging.md\#unity-remote}{\texttt{ here}}.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_touch_autotoc_md1580}{}\doxysection{\texorpdfstring{{\ttfamily Touchscreen} Device}{{\ttfamily Touchscreen} Device}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_touch_autotoc_md1580}

A touch screen Device consists of multiple \href{../api/UnityEngine.InputSystem.Controls.TouchControl.html}{\texttt{ {\ttfamily Touch\+Controls}}}. Each of these represents a potential finger touching the Device. The \href{../api/UnityEngine.InputSystem.Touchscreen.html\#UnityEngine_InputSystem_Touchscreen_primaryTouch}{\texttt{ {\ttfamily primary\+Touch}}} Control represents the touch which is currently driving the \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_pointers}{{\ttfamily Pointer}}{0} representation, and which should be used to interact with the UI. This is usually the first finger that touches the screen.

\href{../api/UnityEngine.InputSystem.Touchscreen.html\#UnityEngine_InputSystem_Touchscreen_primaryTouch}{\texttt{ {\ttfamily primary\+Touch}}} is always identical to one of the entries in the \href{../api/UnityEngine.InputSystem.Touchscreen.html\#UnityEngine_InputSystem_Touchscreen_touches}{\texttt{ {\ttfamily touches}}} array. The \href{../api/UnityEngine.InputSystem.Touchscreen.html\#UnityEngine_InputSystem_Touchscreen_touches}{\texttt{ {\ttfamily touches}}} array contains all the touches that the system can track. This array has a fixed size, regardless of how many touches are currently active. If you need an API that only represents active touches, see the higher-\/level \`{}\+Enhanced\+Touch.Touch\`{} class.

You can use touch input with Actions, like any other \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_pointers}{{\ttfamily Pointer}}{0} Device. To do this, \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_action_bindings}{bind}{0} to the \href{Pointers.md\#controls}{\texttt{ pointer Controls}}, like {\ttfamily \texorpdfstring{$<$}{<}Pointer\texorpdfstring{$>$}{>}/press} or {\ttfamily \texorpdfstring{$<$}{<}Pointer\texorpdfstring{$>$}{>}/delta}. This gets input from the primary touch, and any other non-\/touch pointer Devices.

However, if you want to get input from multiple touches in your Action, you can bind to individual touches by using Bindings like {\ttfamily \texorpdfstring{$<$}{<}Touchscreen\texorpdfstring{$>$}{>}/touch3/press}. Alternatively, use a wildcard Binding to bind one Action to all touches\+: {\ttfamily \texorpdfstring{$<$}{<}Touchscreen\texorpdfstring{$>$}{>}/touch\texorpdfstring{$\ast$}{*}/press}.

If you bind a single Action to input from multiple touches, you should set the Action type to \href{Actions.md\#pass-through}{\texttt{ pass-\/through}} so the Action gets callbacks for each touch, instead of just one.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_touch_autotoc_md1583}{}\doxysection{\texorpdfstring{{\ttfamily Enhanced\+Touch.\+Touch} Class}{{\ttfamily Enhanced\+Touch.\+Touch} Class}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_touch_autotoc_md1583}
The \href{../api/UnityEngine.InputSystem.EnhancedTouch.Touch.html}{\texttt{ {\ttfamily Enhanced\+Touch.\+Touch}}} class provides a polling API for touches similar to \href{https://docs.unity3d.com/ScriptReference/Input-touches.html}{\texttt{ {\ttfamily Unity\+Engine.\+Input.\+touches}}}. You can use it to query touches on a frame-\/by-\/frame basis.

Because the API comes with a certain overhead due to having to record touches as they happen, you must explicitly enable it. To do this, call \href{../api/UnityEngine.InputSystem.EnhancedTouch.EnhancedTouchSupport.html\#UnityEngine_InputSystem_EnhancedTouch_EnhancedTouchSupport_Enable}{\texttt{ {\ttfamily Enhanced\+Touch\+Support.\+Enable()}}}\+:

\texorpdfstring{$>$}{>}{\bfseries{Note}}\+: \href{../api/UnityEngine.InputSystem.Touchscreen.html}{\texttt{ {\ttfamily Touchscreen}}} does not require \href{../api/UnityEngine.InputSystem.EnhancedTouch.EnhancedTouchSupport.html}{\texttt{ {\ttfamily Enhanced\+Touch\+Support}}} to be enabled. The \href{../api/UnityEngine.InputSystem.EnhancedTouch.Touch.html}{\texttt{ {\ttfamily Enhanced\+Touch.\+Touch}}} API is designed to provide access to touch information along two dimensions\+:


\begin{DoxyEnumerate}
\item By finger\+: Each finger is defined as the Nth contact source on a \href{../api/UnityEngine.InputSystem.Touchscreen.html}{\texttt{ {\ttfamily Touchscreen}}}. You can use \href{../api/UnityEngine.InputSystem.EnhancedTouch.Touch.html\#UnityEngine_InputSystem_EnhancedTouch_Touch_activeFingers}{\texttt{ Touch.\+active\+Fingers}} to get an array of all currently active fingers.
\item By touch\+: Each touch is a single finger contact with at least a beginning point (\href{../api/UnityEngine.InputSystem.TouchPhase.html}{\texttt{ {\ttfamily Pointer\+Phase.\+Began}}}) and an endpoint (\href{../api/UnityEngine.InputSystem.TouchPhase.html}{\texttt{ {\ttfamily Pointer\+Phase.\+Ended}}} or \href{../api/UnityEngine.InputSystem.TouchPhase.html}{\texttt{ {\ttfamily Pointer\+Phase.\+Cancelled}}}). Between those two points, an arbitrary number of \href{../api/UnityEngine.InputSystem.TouchPhase.html}{\texttt{ {\ttfamily Pointer\+Phase.\+Moved}}} and/or \href{../api/UnityEngine.InputSystem.TouchPhase.html}{\texttt{ {\ttfamily Pointer\+Phase.\+Stationary}}} records exist. All records in a touch have the same \href{../api/UnityEngine.InputSystem.Controls.TouchControl.html\#UnityEngine_InputSystem_Controls_TouchControl_touchId}{\texttt{ {\ttfamily touch\+Id}}}. You can use \href{../api/UnityEngine.InputSystem.EnhancedTouch.Touch.html\#UnityEngine_InputSystem_EnhancedTouch_Touch_activeTouches}{\texttt{ Touch.\+active\+Touches}} to get an array of all currently active touches. This lets you track how a specific touch moves over the screen, which is useful if you want to implement recognition of specific gestures.
\end{DoxyEnumerate}

See \href{../api/UnityEngine.InputSystem.EnhancedTouch.Touch.html}{\texttt{ {\ttfamily Enhanced\+Touch.\+Touch} API documentation}} for more details.

\texorpdfstring{$>$}{>}{\bfseries{Note}}\+: The \href{../api/UnityEngine.InputSystem.EnhancedTouch.Touch.html}{\texttt{ {\ttfamily Touch}}} and \href{../api/UnityEngine.InputSystem.EnhancedTouch.Finger.html}{\texttt{ {\ttfamily Finger}}} APIs don\textquotesingle{}t generate GC garbage. The bulk of the data is stored in unmanaged memory that is indexed by wrapper structs. All arrays are pre-\/allocated.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_touch_autotoc_md1584}{}\doxysection{\texorpdfstring{Touch Simulation}{Touch Simulation}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_touch_autotoc_md1584}
Touch input can be simulated from input on other kinds of \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_pointers}{Pointer}{0} devices such as \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_mouse}{Mouse}{0} and \doxysectlink{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8inputsystem_0d1_86_83_2_documentation_0i_2_pen}{Pen}{0} devices. To enable this, you can either add the \href{../api/UnityEngine.InputSystem.EnhancedTouch.TouchSimulation.html}{\texttt{ {\ttfamily Touch\+Simulation}}} {\ttfamily Mono\+Behaviour} to a {\ttfamily Game\+Object} in your scene or simply call \href{../api/UnityEngine.InputSystem.EnhancedTouch.TouchSimulation.html\#UnityEngine_InputSystem_EnhancedTouch_TouchSimulation_Enable}{\texttt{ {\ttfamily Touch\+Simulation.\+Enable}}} somewhere in your startup code.
