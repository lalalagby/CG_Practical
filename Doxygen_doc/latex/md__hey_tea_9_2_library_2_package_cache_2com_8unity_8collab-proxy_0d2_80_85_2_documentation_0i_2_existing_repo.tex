 Suppose you want to start working on a \doxylink{namespace_unity}{Unity} project in an existing \doxylink{namespace_unity}{Unity} version control repository and already have a \doxylink{namespace_unity}{Unity} version control account linked to your \doxylink{namespace_unity}{Unity} ID. In that case, you will be able to open the project straight from the {\bfseries{\doxylink{namespace_unity}{Unity} Hub}}. A workspace will automatically be created for your project on your machine.


\begin{DoxyEnumerate}
\item In the \doxylink{namespace_unity}{Unity} Hub v3 Beta, click {\bfseries{Open}} \texorpdfstring{$>$}{>} {\bfseries{Open remote project}} to see the list of your \doxylink{namespace_unity}{Unity} version control repositories that contain a \doxylink{namespace_unity}{Unity} project.
\item Click the project and click {\bfseries{Next}}.
\item Click the Editor version and platform and click the {\bfseries{change version}} button.
\item In the Editor pop-\/up, click the {\bfseries{Migrate}} button to migrate your local workspace to a \doxylink{namespace_unity}{Unity} version control workspace
\item Once the migration is completed, click the {\bfseries{Open \doxylink{namespace_unity}{Unity} version control}} button.
\end{DoxyEnumerate}
