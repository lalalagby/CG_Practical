 Custom Material Inspectors enable you to define how \doxylink{namespace_unity}{Unity} displays properties in the Material Inspector for a particular shader. This is useful if a shader includes a lot of properties and you want to organize them in the Inspector. The Universal Render Pipeline (URP) and High Definition Render Pipeline (HDRP) both support custom Material Inspectors, but the method to create them is slightly different.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8core_0d14_80_88_2_documend4ef7be8cf3d55e7e81de74a330e3ef4_autotoc_md1785}{}\doxysection{\texorpdfstring{Creating a custom Material Inspector}{Creating a custom Material Inspector}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8core_0d14_80_88_2_documend4ef7be8cf3d55e7e81de74a330e3ef4_autotoc_md1785}
The implementation for custom Material Inspectors differs between URP and HDRP. For example, for compatibility purposes, every custom Material Inspector in HDRP must inherit from {\ttfamily HDShader\+GUI} which does not exist in URP. For information on how to create custom Material Inspectors for the respective render pipelines, see\+:


\begin{DoxyItemize}
\item {\bfseries{HDRP}}\+: \href{https://docs.unity3d.com/Packages/com.unity.render-pipelines.high-definition@latest?subfolder=/manual/hdrp-custom-material-inspector.html}{\texttt{ HDRP custom Material Inspectors}}.
\item {\bfseries{URP}}\+: \href{https://docs.unity3d.com/Manual/SL-CustomShaderGUI.html}{\texttt{ Unity Custom Shader GUI}}.
\end{DoxyItemize}\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8core_0d14_80_88_2_documend4ef7be8cf3d55e7e81de74a330e3ef4_autotoc_md1786}{}\doxysection{\texorpdfstring{Assigning a custom Material Inspector}{Assigning a custom Material Inspector}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8core_0d14_80_88_2_documend4ef7be8cf3d55e7e81de74a330e3ef4_autotoc_md1786}
When you create a shader, either hand-\/written or using Shader Graph, both URP and HDRP provide a default editor for it to use. To override this default and provide your own custom Material Inspector, the method differs depending on whether you hand-\/wrote the shader or used Shader Graph.\hypertarget{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8core_0d14_80_88_2_documend4ef7be8cf3d55e7e81de74a330e3ef4_autotoc_md1787}{}\doxysubsection{\texorpdfstring{Using hand-\/written shaders}{Using hand-\/written shaders}}\label{md__hey_tea_9_2_library_2_package_cache_2com_8unity_8render-pipelines_8core_0d14_80_88_2_documend4ef7be8cf3d55e7e81de74a330e3ef4_autotoc_md1787}
To set a custom Material Inspector for a hand-\/written shader\+:


\begin{DoxyEnumerate}
\item Open the shader source file.
\item Assign a string that contains the class name of the custom Material Inspector to the {\bfseries{Custom\+Editor}} shader instruction.
\end{DoxyEnumerate}

This is the same method as for the Built-\/in Renderer\textquotesingle{}s \href{https://docs.unity3d.com/Manual/SL-CustomShaderGUI.html}{\texttt{ custom shader GUI}}.

To set a custom Material Inspector for a Shader Graph shader\+:


\begin{DoxyEnumerate}
\item Open the Shader Graph.
\item In the \href{https://docs.unity3d.com/Packages/com.unity.shadergraph@latest?subfolder=/manual/Internal-Inspector.html}{\texttt{ Graph Inspector}}, open the Graph Settings tab.
\item If {\bfseries{Active Targets}} does not include the render pipeline your project uses, click the {\bfseries{plus}} button then, in the drop-\/down, click the render pipeline.
\item In the render pipeline section ({\bfseries{HDRP}} or {\bfseries{URP}} depending on the render pipeline your project uses) find the {\bfseries{Custom Editor GUI}} property and provide it the name of the custom Material Inspector. 
\end{DoxyEnumerate}
