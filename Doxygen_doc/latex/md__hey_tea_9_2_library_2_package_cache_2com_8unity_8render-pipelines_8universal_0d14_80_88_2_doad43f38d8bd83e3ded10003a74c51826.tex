\&\#160;\&\#160;{\bfseries{Depth\&\#160;Priming\&\#160;Mode}}   &This property determines when \doxylink{namespace_unity}{Unity} performs depth priming.~\newline
Depth Priming can improve GPU frame timings by reducing the number of pixel shader executions. The performance improvement depends on the amount of overlapping pixels in the opaque pass and the complexity of the pixel shaders that \doxylink{namespace_unity}{Unity} can skip by using depth priming.~\newline
The feature has an upfront memory and performance cost. The feature uses a depth prepass to determine which pixel shader invocations \doxylink{namespace_unity}{Unity} can skip, and the feature adds the depth prepass if it\textquotesingle{}s not available yet.~\newline
The options are\+:
\begin{DoxyItemize}
\item {\bfseries{Disabled}}\+: \doxylink{namespace_unity}{Unity} does not perform depth priming.
\item {\bfseries{Auto}}\+: If there is a Render Pass that requires a depth prepass, \doxylink{namespace_unity}{Unity} performs the depth prepass and depth priming.
\item {\bfseries{Forced}}\+: \doxylink{namespace_unity}{Unity} always performs depth priming. To do this, \doxylink{namespace_unity}{Unity} also performs a depth prepass for every render pass. {\bfseries{Note}}\+: Depth priming is disabled at runtime on certain hardware (Tile Based Deferred Rendering) regardless of this setting.
\end{DoxyItemize}On Android, i\+OS, and Apple TV, \doxylink{namespace_unity}{Unity} performs depth priming only in the Forced mode. On tiled GPUs, which are common to those platforms, depth priming might reduce performance when combined with MSAA.~\newline
~\newline
This property is available only if {\bfseries{Rendering Path}} is set to {\bfseries{Forward}}    \\\cline{1-2}
\&\#160;\&\#160;{\bfseries{Accurate G-\/buffer normals}}   &Indicates whether to use a more resource-\/intensive normal encoding/decoding method to improve visual quality.~\newline
~\newline
This property is available only if {\bfseries{Rendering Path}} is set to {\bfseries{Deferred}}.    \\\cline{1-2}
{\bfseries{Copy\&\#160;Depth\&\#160;Mode}}   &Specifies the stage in the render pipeline at which to copy the scene depth to a depth texture. The options are\+:
\begin{DoxyItemize}
\item {\bfseries{After Opaques}}\+: URP copies the scene depth after the opaques render pass.
\item {\bfseries{After Transparents}}\+: URP copies the scene depth after the transparents render pass.
\item {\bfseries{Force Prepass}}\+: URP does a depth prepass to generate the scene depth texture.
\end{DoxyItemize}{\bfseries{Note}}\+: On mobile devices, the {\bfseries{After Transparents}} option can lead to a significant improvement in memory bandwidth. This is because Copy Depth causes a switch in render target between the Opaque pass and the Transparents pass. When this occurs, \doxylink{namespace_unity}{Unity} stores the contents of the Color Buffer in main memory then loads it again once the Copy Depth is complete. The impact increases significantly when MSAA is enabled as \doxylink{namespace_unity}{Unity} must also store and load the MSAA data alongside the Color Buffer.   \\\cline{1-2}
